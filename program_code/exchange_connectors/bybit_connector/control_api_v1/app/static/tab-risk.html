📊 倉位控制與曝險 / Position Sizing & Exposure

Per-Trade Risk Budget / 單筆風險預算
每筆交易投入的最大餘額比例（可調軟參數，非 P1 硬限制）。越大允許單筆越重，越小則單筆更轻。
關鍵影響：BTC/ETH 等高價資產需要至少 5-10% 才能產生最小手數
%
Max Single Position / 單筆最大持倉
單個倉位占賬戶餘額的最大比例。防止單一倉位過於集中
過高 → 單筆虧損可能致命 | 過低 → 高價資產無法開倉
%
Max Total Exposure / 總曝險上限
所有持倉市值總和占賬戶餘額的最大比例。超過則拒絕新開倉
100% = 滿倉上限 | 200% = 允許 2x 槓桿曝險
%
Max Correlated Exposure / 關联曝險上限
高度相關持倉的總曝險上限（空 = 不限制）。用於控制關联品種集中風險
過低 → 限制對冲策略 | 過高 → 關联風險失控
%
Allowed Categories / 允許品類
允許交易的品類，逗號分隔（例：linear,spot）。Rust 引擎直接過滤
留空 = 不過滤 | linear = 僅永續合約
Max Same-Direction / 同向最大持倉數
同一方向（全做多/全做空）的最大持倉數量。Guardian 超限則拒絕
過高 → 方向性集中風險 | 過低 → 限制策略多樣性
個
Preferred Margin Mode / 保證金模式
Rust RiskConfig.limits.margin_mode。isolated = 單倉獨立保證金 | cross = 全倉共享保證金
isolated 爆倉只損單倉 | cross 資本效率高但爆倉牵連全帐戶
Preferred Position Mode / 持倉模式
Rust RiskConfig.limits.position_mode。one_way = 單向持倉 | hedge = 對冲双向持倉
one_way 同幣種僅單方向 | hedge 可同時做多做空（需對應策略）
