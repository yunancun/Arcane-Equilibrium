🎛️ Autonomy Posture / 自主程度設定 AMD v2

系統要多自主？— 保守自主（門檻較高）／ 標準自主（門檻較低）切換
這個設定決定「系統哪些動作可以自己做、哪些一定要你點頭」。能力本身不變， 切到「標準自主」只是把允許自動執行的範圍放寬；切到「保守自主」則更多動作要你親自核准。 最關鍵的安全閘門（如上實盤、換交易所）兩個等級都永遠需要你手動核准。
了解更多 / 兩個等級差在哪
保守自主（Level 1 · Conservative）：較多受保護動作需要你手動核准 （含 5 道安全閘門 + 換交易所，永遠上鎖）。
標準自主（Level 2 · Standard）：較多動作可自動執行，但仍受失效保護機制兜底 （5 道安全閘門 + 換交易所兩個等級都永遠需手動核准）。
切換需要：Operator 身份 + 兩步驟驗證（2FA TOTP）+ 打字確認「CONFIRM SWITCH」+ 至少 30 字理由 + 24 小時冷卻期 + 三路通知（Slack + Email + Console 橫幅）。
技術備註：自主程度與 LAL 0-4（每筆決策核准深度）+ Stage 0R-4（每策略升級階段）三維度正交； Level 只動 path 集合邊界，不動 §Decision 2 的 5 條 fail-safe hard requirements。
