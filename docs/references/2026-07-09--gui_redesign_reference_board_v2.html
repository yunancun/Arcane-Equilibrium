設計主張:玄衡儀
——從「抄哪家交易所」到「造一台自己的儀器」

第二版。引入 Anthropic 官方 frontend-design skill 重審第一版,並把參考面從交易所擴展到 Linear、teenage engineering、Berkeley Mono、iA 這些跨領域標杆。第一版的五方向圖鑑仍可在本頁版本記錄裡回看(initial-five-directions)。

一、skill 對第一版的三條校準

frontend-design skill 的核心立場:設計要「不可能被誤認為別家」,而不是在成熟風格裡挑一個。用它回看第一版,有一條直接打臉:

自我批判:第一版頭號推薦撞上了「AI 默認審美」

skill 點名了三種 AI 生成設計的俗套,第二種是「近黑底 + 單一亮色 accent」——這正是第一版方向 B(Hyperliquid 薄荷綠)的底層公式。而且照著 Hyperliquid 做,本質是穿別人的制服:它的薄荷綠是它的身分,不是你的。深色高密度的結論仍然成立(那是品類規律),但 accent 與氣質必須是自己選的,不能是「深色終端的默認出廠色」。

校準二:字體要有主張,不能用「安全臉」

skill 明確反對「任何項目都能用的那套字體」。跨界調研恰好給出答案:U.S. Graphics 用一套 Berkeley Mono 撐起整個「terminal 美學做成高級品」的品牌;iA 證明 tabular-nums + 嚴格對齊本身就是設計。字體(尤其等寬字型的選擇)應該是這次改版裡最有辨識度的單一決策。

校準三:要有一個 signature 元素,而且只要一個

「把大膽花在一處,其餘保持安靜」。第一版給了五種風格但零個記憶點。第二版的全部大膽預算,花在下面這一個元素上:衡樑。

二、設計主張:玄衡儀

這個系統自己的名字就是最好的設計題目:玄衡 · Arcane Equilibrium。

「玄」在傳統色譜裡不是純黑,是「黑而有赤」——帶一絲暗紅底溫的深黑。「衡」是秤樑,也是「璇璣玉衡」——古代觀測天象的儀器。順著名字走,這個 GUI 的正確身分不是「另一個交易所界面」,而是一台觀測市場的儀器:玄漆為底、青銅刻度、朱砂鈐印。而「印」甚至不是強加的比喻——系統詞彙裡本來就有 seal(proof packet 封存、hidden-OOS sealer、seal-lineage):被封存驗證的東西蓋一方小朱印,是把既有工程概念可視化,不是裝飾。

操作者是工程師、無人工下單、治理概念(gate / lease / envelope / audit)是界面的主角——儀器美學(teenage engineering 的規格表排版、Berkeley Mono 的等寬網格)比交易所美學更貼合這個產品的真實。

Signature
衡樑:頂欄常駐的一根青銅髮絲線秤樑,支點居中,左右兩端 = 風控包絡已用/剩餘(或多/空敞口)。健康時水平微光,越過閾值時緩緩傾斜、支點變色。它同時是:系統名的直譯、operator 最關心的單一治理指標、和沒有任何交易所有過的記憶點。全站唯一的環境動效預算都給它。
Motion
數據區微動效 ≤220ms、只確認數值變化、永不遮數字(Hyperliquid 紀律);衡樑允許 500ms 級緩動(它是環境儀表,不是數據)。prefers-reduced-motion 全部靜止。

四、跨界借鑒(交易所之外,已逐一驗證)

按「借什麼」分組;每張卡一個具體手法,不是泛泛的「設計很好」:

儀器美學 —— 玄衡儀的直系血親

teenage engineering網站即儀器

絲印(silkscreen)排版:小號等寬全大寫標籤+細規則線構成規格表——玄衡儀「儀表態」的字階範式直接源於此。

teenage.engineering
U.S. Graphics / Berkeley Mono原產地

「terminal 美學做成奢侈品」的原型:MIL-SPEC 圖紙風、全站單一等寬字型、型號化命名(TX-02)。策略/lane/gate 賦予規格編號的想法源於此。官網擋爬蟲,瀏覽器直接開。

usgraphics.com
Warp / Ghostty

Warp 的 output-block:每條命令+輸出是一個可折疊、可複製、帶時間戳的卡片——event log 的正確形制。Ghostty 證明純排版即可傳達工程可信度。

warp.dev · ghostty.org

產品工藝 —— 深色界面的手感從哪來

Linear深色標杆

深色「海拔」不靠陰影靠亮度階:邊框/分隔/hover 全用同一組 alpha-white token,層次一致不髒。玄衡儀的分層機制照此辦理。

linear.app
Stripe

色彩克制紀律:漸變只做品牌點綴、絕不進數據區;docs 的雙欄同步滾動可搬到策略配置說明頁。

stripe.com
Raycast

玻璃(backdrop-blur)只用於浮層,主面板純色;快捷鍵用「鍵帽」元件可視化——hotkey 提示直接照抄。

raycast.com
Resend

深色底上的「微光層」:極低透明度 radial glow 標記焦點區,替代高飽和高亮引導注意力——衡樑的「健康微光」用這個手法。

resend.com
Supabase可讀源碼

dark-first 開發者 dashboard,前端開源:深色表格、行 hover、密集 filter bar 的實現可以直接讀代碼,不止看截圖。

supabase.com
Ableton Learning Music

「可互動處才有色彩」:灰階為底,只有可操作控件與數據語義著色——數據密集界面防過載的第一原則。

learningmusic.ableton.com
