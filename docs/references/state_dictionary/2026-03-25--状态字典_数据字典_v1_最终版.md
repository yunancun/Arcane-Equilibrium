## 1. 总体合同

### 1.1 通用命名

- 时间统一：`*_ts_ms`
- 原因列表统一：`*_reason_codes`
- 门槛判断统一：`*_gate_state`
- 控制配置统一：`*_switch`
- 只读摘要统一：`*_summary`
- 直接运行事实统一：`*_fact`
- 派生字段统一只读
- 审计字段统一只读

### 1.2 根对象合同

状态字典根对象不再重复承载 envelope 元信息；元信息只在 `meta` 下 canonical 保存。

```json
{
  "meta": {},
  "global_runtime": {},
  "chapter_status": {},
  "product_family_status": {},
  "control_plane": {},
  "capability_matrix": {},
  "business_metrics": {},
  "health_telemetry": {},
  "learning_state": {},
  "audit_context": {}
}
```

### 1.3 写入权限类型

| 标记 | 含义 |
| --- | --- |
| `RO-fact` | 只读事实；来源 runtime / exchange / telemetry / recheck |
| `CFG` | 只允许配置接口写入 |
| `ACT` | 只允许动作处理器写入；禁止通用配置直写 |
| `DRV` | 只读派生 |
| `AUD` | 只读审计 |

### 1.4 真相源规则

同一语义只允许一个 canonical 路径。  
镜像、摘要、控制中心视图、GUI 视图一律非 canonical。

### 1.5 禁止项

- 禁止直接 patch 任意 `ACT` 字段
- 禁止直接 patch 任意 `DRV` / `AUD` 字段
- 禁止把 `summary` 字段当写入入口
- 禁止出现双 canonical
- 禁止用未注册枚举值

## 14. 推导规则

### 14.1 `global_runtime.derived.global_mode_state`

按 `global_runtime.facts.system_mode_fact` 直接映射：

| `system_mode_fact` | `global_mode_state` |
| --- | --- |
| `observe_only` | `observe_only` |
| `shadow_only` | `shadow_only` |
| `design_only` | `design_only` |
| `demo_reserved` | `demo_reserved` |
| `live_reserved` | `live_reserved` |

### 14.2 `global_runtime.derived.global_stage_label`

按以下优先级自上而下命中第一条：

1. 若 `global_runtime.controls.global_execution_mode_switch = live_reserved`，则 `future_live_reserved`
2. 若 `chapter_status.K.current_phase_ready = true` 且 `chapter_status.K.readiness_scope = design_only_gate_closed`，则 `design_only_gate_closed`
3. 若 `chapter_status.J.current_phase_ready = true` 且 `chapter_status.J.readiness_scope = shadow_closeout`，则 `shadow_closeout_ready`
4. 若 `chapter_status.J.chapter_state` 为 `partial` / `implemented` / `canonical_open`，则 `shadow_closeout_partial`
5. 其余为 `observer_baseline`

### 14.3 `global_runtime.derived.global_capability_state`

按以下优先级：

1. 若 `global_stage_label = design_only_gate_closed`，则 `shadow_control_ready`
2. 若 `global_stage_label = shadow_closeout_ready`，则 `shadow_operational_visibility`
3. 若 `global_stage_label = future_live_reserved`，则 `live_candidate_reserved`
4. 其余为 `minimal_visibility`

### 14.4 `global_runtime.derived.global_execution_authority_state`

按以下优先级：

1. 若 `global_runtime.controls.global_execution_mode_switch = disabled`，则 `disabled`
2. 若 `global_runtime.controls.global_execution_mode_switch = live_reserved`，则 `live_blocked`
3. 若 `global_runtime.controls.global_execution_mode_switch = demo_reserved` 且 `control_plane.demo_control.demo_state_switch != demo_enabled`，则 `demo_blocked`
4. 若 `global_runtime.controls.global_execution_mode_switch = demo_reserved` 且 `control_plane.demo_control.demo_state_switch = demo_enabled` 且 `health_telemetry.gates.health_gates_overall_state != failed` 且 `control_plane.risk_envelope.risk_envelope_state != blocking`，则 `demo_enabled`
5. 否则为 `demo_blocked`

### 14.5 `global_runtime.derived.runtime_still_protected`

以下条件同时满足时为 `true`：

- `global_runtime.derived.global_execution_authority_state` 不为 `demo_enabled`
- `global_runtime.controls.global_execution_mode_switch != live_reserved`

否则为 `false`。

### 14.6 `product_family_status.<pf>.derived.capability_state`

按以下优先级：

1. 若 `visibility_switch = false`，则 `unavailable`
2. 若 `exchange_permission_fact = unavailable` 或 `account_permission_fact = unavailable`，则 `unavailable`
3. 若 `enabled_switch = false`，则 `visible_only`
4. 若 `mode_switch = observe_only`，则 `visible_only`
5. 若 `mode_switch = shadow_only` 且 `capability_matrix.product_families.<pf>.control_plane_capability_state = implemented`，则 `shadow_control_ready`
6. 若 `mode_switch = shadow_only`，则 `shadow_visible`
7. 其余为 `reserved`

### 14.7 `product_family_status.<pf>.derived.execution_authority_state`

按以下优先级：

1. 若 `enabled_switch = false`，则 `disabled`
2. 若 `global_runtime.derived.global_execution_authority_state = disabled`，则 `disabled`
3. 若 `global_runtime.derived.global_execution_authority_state != demo_enabled`，则 `blocked`
4. 若 `mode_switch != shadow_only`，则 `blocked`
5. 若该产品族存在任一 `effective_*_allowed_state = allowed`，则 `guarded`
6. 其余为 `blocked`

### 14.8 `product_family_status.<pf>.derived.product_family_summary`

按以下优先级：

1. 若 `visibility_switch = false`，则 `hidden`
2. 若 `enabled_switch = false`，则 `visible_but_disabled`
3. 若 `mode_switch = disabled`，则 `visible_but_not_enabled`
4. 若 `derived.capability_state = shadow_control_ready`，则 `shadow_control_ready`
5. 若 `mode_switch in {observe_only, shadow_only}`，则 `shadow_visible_only`
6. 其余为 `reserved`

### 14.9 `control_plane.action_permissions.*.effective_*_allowed_state`

对每个动作类型独立计算，按以下优先级命中第一条：

1. 若对应 `configured_*_switch = false`，则 `disabled`，并追加 `configured_switch_disabled`
2. 若对应产品族 `enabled_switch = false`，则 `blocked`，并追加 `product_family_disabled`
3. 若对应产品族 `visibility_switch = false`，则 `blocked`，并追加 `product_family_not_visible`
4. 若对应产品族 `mode_switch != shadow_only`，则 `blocked`，并追加 `product_family_mode_blocked`
5. 若 `global_runtime.controls.global_execution_mode_switch = disabled`，则 `blocked`，并追加 `global_execution_blocked`
6. 若 `control_plane.demo_control.demo_state_switch != demo_enabled`，则 `blocked`，并追加 `demo_not_enabled`
7. 若 `control_plane.risk_envelope.risk_envelope_state = blocking`，则 `blocked`，并追加 `risk_scope_blocked`
8. 否则为 `allowed`

全局权限对象按全局条件计算。  
产品族权限对象按“全局条件 + 产品族条件”共同计算。

  - `demo_arm_gate_state`
  - `demo_arm_reason_codes`
  - `demo_arm_last_evaluated_ts_ms`
  - `demo_enable_gate_state`
  - `demo_enable_reason_codes`
  - `demo_enable_last_evaluated_ts_ms`
  - `demo_relock_gate_state`
  - `demo_relock_reason_codes`
  - `demo_relock_last_evaluated_ts_ms`

### 16.2 `arm`

- `expected_previous_state` 必填
- 允许前态：
  - `closed`
  - `relocked`
- 进入条件：
  - `demo_prerequisites_gate_state = passed`
  - `global_runtime.controls.global_execution_mode_switch = demo_reserved`
- 成功后必须写入：
  - `demo_state_switch = armed_but_closed`
  - `demo_last_action_type = arm`
  - `demo_last_action_result = success`
- 失败 / 阻断 reason code 仅允许：
  - `previous_state_mismatch`
  - `prerequisites_not_passed`
  - `execution_mode_disabled`
  - `live_mode_reserved_only`
  - `state_revision_mismatch`
  - `replayed_request`
  - `idempotency_conflict`

### 16.3 `enable`

- `expected_previous_state` 必填，且必须为 `armed_but_closed`
- `payload.acknowledged` 必填，且必须为 `true`
- 进入条件：
  - `demo_enable_gate_state = passed`
  - `demo_operator_ack_required = false`，或 `payload.acknowledged = true`
  - `demo_cooldown_state != active`
- 成功后必须写入：
  - `demo_state_switch = demo_enabled`
  - `demo_operator_ack_completed = true`
  - `demo_last_action_type = enable`
  - `demo_last_action_result = success`
- 失败 / 阻断 reason code 仅允许：
  - `previous_state_mismatch`
  - `not_armed`
  - `operator_ack_required`
  - `cooldown_active`
  - `health_gate_blocked`
  - `risk_envelope_blocked`
  - `state_revision_mismatch`
  - `replayed_request`
  - `idempotency_conflict`

### 16.4 `relock`

- `expected_previous_state` 必填
- 允许前态：
  - `armed_but_closed`
  - `demo_enabled`
  - `relocked`
- 成功后必须写入：
  - `demo_state_switch = relocked`
  - `demo_cooldown_state = active`
  - `demo_cooldown_until_ts_ms = client_ts_ms + cooldown_policy_ms`
  - `demo_last_action_type = relock`
  - `demo_last_action_result = success`
- 失败 / 阻断 reason code 仅允许：
  - `previous_state_mismatch`
  - `state_revision_mismatch`
  - `replayed_request`
  - `idempotency_conflict`

## 19. V1 实现硬约束

- 顶层块名不得改
- `meta.*` 为根元信息唯一 canonical
- `global_runtime.controls.*` 为全局运行配置唯一 canonical
- `chapter_status.<chapter>.*` 为章节状态唯一 canonical
- `capability_matrix.<chapter>.*` 为章节 recheck / closeout 结果唯一 canonical
- `demo_state_switch` 为唯一正式 demo 主状态机字段
- `ACT` 字段只能通过对应 action handler 写入
- `chapter_ready` 禁用；只允许 `current_phase_ready + readiness_scope`
- `summary` 字段不得作为写入源
- 任意控制动作必须带：
  - `request_id`
  - `idempotency_key`
  - `expected_state_revision`
- `learning_state` 不得直接放权到执行层
- `product_family_status` 采用固定块结构；不接受散乱 `*_map` 作为主真相
- `control_plane.action_permissions` 必须同时区分：
  - `configured_*_switch`
  - `effective_*_allowed_state`
- 未在注册表中的枚举值一律非法
