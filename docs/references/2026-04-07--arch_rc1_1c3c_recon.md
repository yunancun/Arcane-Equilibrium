## 4. Hidden Coupling Risks

### 4.1 `/unhalt-session` bypasses RiskManager

```python
def unhalt_session(actor: ...):
    from .paper_trading_routes import PAPER_STORE
    def mutator(state):
        state["session"]["session_halted"] = False
        state["session"]["session_halt_reason"] = None
        return state
    PAPER_STORE.mutate(mutator)
    # then sends IPC resume to Rust
```

**Two systems need to agree:** Python `PAPER_STORE` + Rust `pipeline.session_halted`.

**1C-3-C decision required:**
- (a) Add `RiskViewClient.unhalt_session()` that calls Rust `resume_paper` IPC and lets Rust be the truth
- (b) Keep `PAPER_STORE.mutate` as-is (out of `RiskViewClient` scope)

**Recommendation:** (a) â€” let Rust be the truth. Python `PAPER_STORE.session_halted` should be derived from Rust snapshot, not a parallel write.

### 4.2 `update_global_config` / `agent_adjust` best-effort IPC push

Lines 162â€“207 and 332â€“350 already do:
```python
async def _push_risk():
    client = EngineIPCClient()
    try:
        await client.connect()
        await client.update_risk_config(**ipc_kwargs)
        await client.disconnect()
    except Exception:
        pass  # best-effort
```

**Problem:** Python wrote to disk, Rust may have failed silently â†’ state divergence.

**1C-3-C:** route should `await` `client.update_global_config()` (which goes through `patch_risk_config` IPC, the new write path) and **fail with HTTP 500** if Rust unreachable. No more best-effort.

### 4.3 Rust snapshot blending in `get_risk_status()` (lines 254â€“267)

Currently overlays Rust fields onto Python's `rm.get_status()`. Post-migration, the Rust-native `get_risk_runtime_status` IPC already returns the full Rust state, so this blending step disappears. GUI binds directly to the new shape.
