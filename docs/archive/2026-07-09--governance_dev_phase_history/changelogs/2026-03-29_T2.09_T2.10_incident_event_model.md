### 實現內容 / Implementation Details

**1. 正式事件對象 Event（GAP-M1）**
- 唯一 event_id、event_type、severity、source
- triggered_at_ms、triggered_by、affected_objects
- reason_code、reason_detail、metadata
- audit_chain_ref、parent_event_id（因果鏈）
- 工廠方法 create_event() 和 from_reconciliation_report()

**2. 5 級嚴重度層級（DOC-07 §3）**
- NOTICE(0)：可觀察，非關鍵
- ANOMALY(1)：偏差，需關注
- NEAR_MISS(2)：險些事故
- INCIDENT(3)：真實影響
- CRITICAL_INCIDENT(4)：威脅賬戶安全

**3. 事故記錄 IncidentRecord（DOC-07 §8）**
- 影響範圍：symbols、strategies、objects
- 根因分類（7 類）：state_conflict、execution_failure、risk_gap、audit_gap、auth_gap、logic_error、external_failure
- 完整性評估：truth_source_integrity、audit_integrity
- 恢復流程：recovery_approved_by、resolved_at_ms

**4. 嚴重度→動作映射（DOC-07 §3-§5）**
| 嚴重度 | 觸發動作 |
|--------|----------|
| NOTICE | RECORD_ONLY |
| ANOMALY | RECORD + MONITORING + RISK_CAUTIOUS |
| NEAR_MISS | AUTH_RESTRICT + RISK_REDUCED + ALERT |
| INCIDENT | AUTH_FREEZE + RISK_DEFENSIVE + MANUAL_REVIEW + ALERT |
| CRITICAL | AUTH_FREEZE + RISK_CIRCUIT_BREAKER + TRADING_FREEZE + REVIEW + ALERT |

**5. IncidentPolicy 引擎**
- 事件處理：自動根據嚴重度觸發動作
- 狀態機集成回調：on_auth_action、on_risk_action、on_operator_alert
- NEAR_MISS 以上自動創建 IncidentRecord
- 恢復審批流程（operator-only）
- 對賬報告→事件轉換
- 查詢：open_incidents、by_severity、stats
