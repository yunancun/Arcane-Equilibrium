### 實現內容 / Implementation Details

**核心組件：**

1. **LeaseState** — 9 個狀態（Enum）
   - DRAFT, REGISTERED, ACTIVE, BRIDGED, FROZEN
   - REVOKED, EXPIRED, REJECTED, CONSUMED
   - 終態：REVOKED, EXPIRED, REJECTED, CONSUMED

2. **LeaseTransitionRule** — 18 條有效轉換規則
   - DRAFT → REGISTERED, REJECTED
   - REGISTERED → ACTIVE, FROZEN, REVOKED, EXPIRED, REJECTED
   - ACTIVE → BRIDGED, FROZEN, REVOKED, EXPIRED, REJECTED
   - FROZEN → REGISTERED, ACTIVE, REVOKED, EXPIRED
   - BRIDGED → CONSUMED, REVOKED

3. **FORBIDDEN_TRANSITIONS** — 12 條禁止轉換
   - DRAFT 不可直接 → ACTIVE / BRIDGED / CONSUMED
   - REGISTERED 不可 → BRIDGED / CONSUMED
   - REVOKED / EXPIRED / REJECTED / CONSUMED 不可重新啟動

4. **DecisionLeaseObject** — 租約數據類
   - intent（目的描述）、time_window_ms（有效期窗口）
   - governance_metadata（治理鏈引用）
   - 完整轉換歷史記錄

5. **DecisionLeaseStateMachine** — 狀態機引擎
   - 線程安全（threading.Lock）
   - 7 種發起者類型（OPERATOR, I_CONTROL_PLANE, RISK_GOVERNOR 等）
   - 審批守衛（revoke/unfreeze 需要審批）
   - 過期守衛（自動到期檢查）
   - 審計回調接口
   - 持久化導出/導入

6. **便捷方法** — register, reject, activate, bridge, consume, freeze, revoke, unfreeze_to_registered, unfreeze_to_active

7. **查詢方法** — get_live_leases, get_bridgeable_leases, status_summary
