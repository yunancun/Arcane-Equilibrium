### 實現內容 / Implementation Details

**8 個正式狀態（SM-01 §3）：**
DRAFT → PENDING_APPROVAL → ACTIVE → RESTRICTED → FROZEN → REVOKED → EXPIRED → REJECTED

**16 條合法遷移（SM-01 §6-7）：**
- §7.1 草案與審批：DRAFT→PENDING, DRAFT→REJECTED, PENDING→ACTIVE, PENDING→REJECTED
- §7.2 生效後治理：ACTIVE→RESTRICTED, ACTIVE→FROZEN, ACTIVE→REVOKED, ACTIVE→EXPIRED
- §7.3 收縮/凍結後恢復與終止：RESTRICTED→ACTIVE/FROZEN/REVOKED/EXPIRED, FROZEN→RESTRICTED/ACTIVE/REVOKED/EXPIRED

**7 條明確禁止遷移（SM-01 §8）：**
- 終態不可回流：REVOKED→ACTIVE/RESTRICTED, EXPIRED→ACTIVE/RESTRICTED, REJECTED→ACTIVE/PENDING
- 跳過審批：DRAFT→ACTIVE

**守衛條件：**
- 擴權（scope expansion）必須有 approved_by
- 發起者身份驗證（5 類發起者）
- 終態不可遷出

**其他功能：**
- 過期守護（ExpiryGuardian）：定期掃描並自動觸發 EXPIRED
- 審計軌跡：每次遷移生成 authorization_transition 審計對象（含 14 個欄位）
- 線程安全：threading.Lock 原子操作
- 持久化：export_state / import_state JSON 序列化
