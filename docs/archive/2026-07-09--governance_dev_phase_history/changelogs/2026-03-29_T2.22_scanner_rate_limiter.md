#### ScannerRateLimiter Engine
Thread-safe rate limiter with the following capabilities:

**Core Methods:**
- `can_scan() → (bool, str)`: Check if scan can start now; returns reason if blocked
- `record_scan_start() → bool`: Mark scan as starting; returns False if rate limited
- `record_scan_complete() → None`: Mark scan as finished; clears error cooldown
- `record_scan_error() → None`: Mark scan as failed; activates error cooldown
- `get_next_scan_time_ms() → Optional[int]`: When next scan can start (epoch milliseconds)
- `time_until_next_scan_seconds() → float`: Seconds to wait before next scan allowed
- `get_scan_stats() → ScanStats`: Current scan statistics (counts, averages, timestamps)

**Features:**
- ✓ Minimum interval enforcement (300 seconds default)

- ✓ Audit callback support for compliance logging
- ✓ Thread-safe with RLock synchronization
- ✓ Statistics tracking (totals, errors, average intervals, last times)

## Technical Design / 技術設計

### Rate Limiting Strategy
1. **Minimum Interval Enforcement**: Tracks `_last_scan_complete_ms`. Scan rejected if elapsed time since completion < 300 seconds.
2. **Error Cooldown**: Tracks `_last_error_time_ms`. On error, activates 600-second cooldown. Scan rejected during cooldown.
3. **Concurrent Limits**: Tracks `_active_scans`. Rejected if active scans ≥ `max_concurrent_scans`.
4. **Priority**: Error cooldown checked first (highest priority), then concurrent limits, then interval.

### Thread Safety
- Uses `threading.RLock` for all state mutations
- Lock acquired in `can_scan()`, `record_scan_*()`, `get_next_scan_time_ms()`, and `get_scan_stats()`
- Audit callbacks executed within lock to ensure consistent state reporting

### Audit Trail
Optional audit callback invoked for:
- `scan_start`: Scan beginning, active count
- `scan_start_rejected`: Rejected scan with reason
- `scan_complete`: Successful completion with duration and totals
- `scan_error`: Failed scan with error count and cooldown duration
- `scan_complete_no_start`: Completion called without matching start

### Statistics Tracking
- `total_scans`: Incremented only on `record_scan_complete()`
- `total_errors`: Incremented on `record_scan_error()`
- `total_successes`: Calculated as `total_scans - total_errors`
- `average_interval_seconds`: Rolling average of scan intervals (max 100 samples)
- `last_scan_time_ms`, `last_error_time_ms`: Timestamp tracking

## Example Usage / 使用範例

```python
from app.scanner_rate_limiter import ScannerConfig, ScannerRateLimiter

# Initialize with default 5-minute interval
limiter = ScannerRateLimiter()

# Check if scan is allowed
can_scan, reason = limiter.can_scan()
if not can_scan:
    print(f"Scan blocked: {reason}")  # e.g., "min_interval_not_reached (remaining 4.3s)"

# If allowed, start scan
if limiter.record_scan_start():
    try:
        # ... perform market scan ...
        limiter.record_scan_complete()
        print(f"Scan complete. Next scan in {limiter.time_until_next_scan_seconds():.1f}s")
    except Exception as e:
        limiter.record_scan_error()
        print(f"Scan error. Cooldown activated (remaining {limiter.time_until_next_scan_seconds():.1f}s)")

# View statistics
stats = limiter.get_scan_stats()
print(f"Total scans: {stats.total_scans}, Errors: {stats.total_errors}, Avg interval: {stats.average_interval_seconds:.1f}s")
```

## Notes / 附註

### Implementation Notes
1. **Zero Startup Cost**: No market data fetch or external calls on initialization
2. **Sub-millisecond Checks**: `can_scan()` is pure in-memory; complies with H0 < 1ms requirement
3. **Error State Clearing**: Successful scan automatically clears error cooldown, allowing recovery without manual intervention
4. **Interval Tracking**: Rolling window of 100 most recent intervals for accurate average calculation
5. **Lock Granularity**: RLock allows recursive calls; suitable for nested governance checks
