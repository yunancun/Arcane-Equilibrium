## 概述 / Overview

实现了交易归因引擎，将已完成的交易分解为多个成因因子，严格区分"判断"（skill）和"偶然"（luck）。本模块直接支持根原则 §5.8（每笔交易必须可解释、可追踪）和根原则 §5.12（系统必须能自动归因交易结果）。

Implemented the Trade Attribution Engine to decompose completed trades into attribution factors that strictly distinguish skill (judgment) from luck (random/unexplained). This module directly supports Root Principle §5.8 (every trade must be interpretable and traceable) and Root Principle §5.12 (system must auto-attribute trade results).

#### Core Engine / 核心引擎

**TradeAttributionEngine**

*Key Methods:*

- **attribute_trade()** — Decompose a completed trade
  - Input: trade metadata (entry/exit prices, times, quantities, costs)
  - Input: optional counterfactuals (expected_timing_pnl, expected_sizing_volatility, expected_execution_slippage)
  - Output: TradeAttributionResult with 6 attribution factors
  - Returns: cached result (thread-safe)

- **Factor Calculation Methods** (static)
  - `_calculate_alpha_score()`: Direction correctness
    - Positive PnL → positive score (0 to 1.0)
    - Negative PnL → negative score (-1.0 to 0)
    - Contribution: 100% (all PnL comes from direction choice)

  - `_calculate_timing_score()`: Entry/exit timing
    - Compares actual PnL vs expected_timing_pnl
    - Positive if actual > expected, negative if worse
    - Contribution: |improvement| / |gross_pnl|

  - `_calculate_sizing_score()`: Position sizing quality
    - Evaluates size/volatility ratio (ideal ~1.0)
    - Good range [0.5, 2.0] → positive score
    - Out of range → negative score
    - Contribution: 0.1–0.3 (sizing less important than direction)

  - `_calculate_execution_score()`: Fill quality
    - Better than expected slippage → positive
    - Worse than expected → negative
    - Contribution: |actual_vs_expected| / |gross_pnl|

  - `_calculate_cost_score()`: Fee optimization
    - cost_drag = total_cost / |gross_pnl|
    - Excellent (<5%) → 0.8–1.0
    - Good (5–20%) → 0.8 declining to -0.8
    - Poor (>20%) → -0.8 to -1.0
    - Contribution: cost_drag percentage

  - `_calculate_luck_score()`: Residual
    - Unexplained component = 1.0 - sum(other_contributions)
    - Score: 0.0–0.5 (neutral, not explanatory)

- **Aggregation Methods**

  - **aggregate_attribution()** — Strategy-level summary
    - Input: strategy name, period_start, period_end
    - Output: StrategyAttributionSummary (or None)
    - Computes: win_rate, avg_skill_pct, skill_consistency, total costs
    - Attribution by category: average contribution across all trades

  - **get_strategy_skill_ratio()** — Long-term skill tracking
    - Input: strategy name
    - Output: StrategySkillRatio (or None)
    - Computes: skill_level, confidence based on trade count
    - Tracks: trades_positive_skill (>0.50), trades_negative_skill (<0.40)

- **Utility Methods**
  - `get_trade_attribution(trade_id)`: Retrieve cached result
  - `list_strategy_summaries()`: Dict of all summaries
  - `list_strategy_skill_ratios()`: Dict of all skill ratios
  - `to_dict()`, `from_dict()`: Serialization (JSON-compatible)

*Design Features:*
- **Thread-safe**: Uses RLock for all operations
- **Contribution Normalization**: All contribution_pct sum to ~1.0 (enforced during attribution)
- **Caching**: Trades cached by trade_id for fast lookup
- **Serialization**: All dataclasses JSON-serializable with proper rounding
- **Skill vs Luck**: Automatically aggregated from 6-factor decomposition

## 核心特性 / Key Features

### 1. Six-Factor Attribution / 六因子归因

Each trade is decomposed into:
- **ALPHA** (direction): Was the direction right? By how much?
- **TIMING** (entry/exit): Did entry/exit improve PnL vs counterfactual?
- **SIZING** (position size): Was size appropriate for volatility?
- **EXECUTION** (fills): Did we get good prices vs expected slippage?
- **COST** (fees): Did we optimize maker/taker/VIP level?
- **LUCK** (residual): What's unexplained after other factors?

### 2. Skill vs Luck Quantification / 判断vs偶然量化

- skill_pct: Sum of (ALPHA + TIMING + SIZING + EXECUTION + COST) contributions
- luck_pct: LUCK contribution
- Normalized to sum ~1.0
- Enables distinction between "good decision, bad execution" vs "lucky trade"

### 3. Strategy-Level Aggregation / 策略级汇总

- win_rate: % of trades with positive net PnL
- avg_skill_pct: Average skill across all trades
- skill_consistency: Std dev (lower = more repeatable)
- attribution_by_category: Which factors matter most?
- Supports performance analysis at strategy level

### 4. Long-Term Skill Tracking / 长期 Skill 追踪

- StrategySkillRatio: Classify as HIGH/MODERATE/LOW skill
- Confidence: Increases with sample size (30+ → 90%, 100+ → 95%)
- Tracks: Trades with positive vs negative skill
- Enables: Identifying strategies that are truly profitable vs lucky

### 5. Thread-Safe, Serializable / 线程安全，可序列化

- RLock protection on all mutable state
- JSON-compatible serialization
- Supports system snapshot/restore
- Append-only attribution cache

## 使用例 / Example Usage

```python
from app.trade_attribution import TradeAttributionEngine, AttributionCategory
import datetime

engine = TradeAttributionEngine()
now = datetime.datetime.utcnow()

# Attribute a single trade
result = engine.attribute_trade(
    trade_id="trade_123",
    symbol="BTCUSDT",
    strategy="MA_CROSSOVER",
    entry_price=40000.0,
    exit_price=41000.0,
    quantity=1.0,
    entry_timestamp=now,
    exit_timestamp=now + datetime.timedelta(hours=1),
    market_prices_at_entry={"BTCUSDT": 40000.0},
    market_prices_at_exit={"BTCUSDT": 41000.0},
    fees_paid=10.0,
    slippage=5.0,
    expected_timing_pnl=500.0,
)

print(f"Trade PnL: {result.pnl_net:.2f}")
print(f"Skill: {result.skill_pct:.1%}, Luck: {result.luck_pct:.1%}")

# Print attribution breakdown
for score in result.attribution_scores:
    print(f"  {score.category.value}: {score.score:.2f} ({score.contribution_pct:.1%})")

# Aggregate across strategy
summary = engine.aggregate_attribution(
    strategy="MA_CROSSOVER",
    period_start=now,
    period_end=now + datetime.timedelta(days=1),
)

if summary:
    print(f"Strategy stats: {summary.trade_count} trades, {summary.win_rate:.1%} win rate")
    print(f"Avg skill: {summary.avg_skill_pct:.1%}")

# Get long-term skill classification
ratio = engine.get_strategy_skill_ratio("MA_CROSSOVER")
if ratio:
    print(f"Skill level: {ratio.skill_level.value} (confidence: {ratio.confidence:.0%})")
```
