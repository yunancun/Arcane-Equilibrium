## Problem Statement / 问题陈述

### Historical Issue / 历史问题

SM-02 规范 §12 明确规定：

> "Decision Lease ACTIVE 状态最多 30 秒，然后自动过期"
>
> "Decision Lease BRIDGED 状态最多 60 秒，然后自动过期"

However, historical code sometimes had ACTIVE TTL set to 60 seconds, creating a spec drift.

However, historical code sometimes had ACTIVE TTL set to 60 seconds, creating a spec drift.

### Root Cause / 根本原因

- TTL 值分散在多个文件中（无单一真实来源）
- 缺乏形式化的 TTL 配置模块
- 缺乏自动化验证机制确保规范对齐
- 配置变更缺乏审计跟踪

### Solution / 解决方案

创建专用的 TTL 配置模块，实现：
1. **单一真实来源 (SSOT)**: 所有 TTL 值集中在一个模块
2. **形式化验证**: LeaseTTLValidator 确保值在规范边界内
3. **合规报告**: SpecComplianceReport 生成可读的合规性报告
4. **管理器模式**: LeaseTTLConfigManager 作为单例提供对配置的安全访问
5. **审计跟踪**: 所有配置变更都被记录

### Key Design Invariants / 关键设计不变量

1. **Immutability**: LeaseTTLConfig is frozen (dataclass(frozen=True))
   - 防止意外修改
   - 确保配置的一致性

2. **Spec Alignment**: All default values sourced from SM-02 §12
   ```python
   SPEC_TTL_VALUES = {
       "ACTIVE_DEFAULT_TTL_SECONDS": 30,        # SM-02 §12
       "BRIDGED_DEFAULT_TTL_SECONDS": 60,       # SM-02 §12
       "FROZEN_MAX_DURATION_SECONDS": 300,      # SM-02 §7.4
       "REGISTERED_TO_ACTIVE_MAX_WAIT_SECONDS": 3600,  # SM-02 §7.2
   }
   ```

3. **Override Restrictions**: ACTIVE & BRIDGED TTLs cannot be overridden
   ```python
   active_ttl_override_authority = TTLOverrideAuthority.NEVER
   bridged_ttl_override_authority = TTLOverrideAuthority.NEVER
   ```

4. **Thread-Safe**: LeaseTTLConfigManager uses locks for concurrent access
   - get_config() returns immutable copy
   - Singleton pattern ensures single instance
   - All access is lock-protected

5. **Audit Trail**: All config changes are logged with timestamps
   - export_change_log() provides audit trail
   - Every configuration operation is traceable

### Validator Boundary Checks / 验证器边界检查

```python
class LeaseTTLValidator:
    MIN_ACTIVE_TTL_SECONDS = 10
    MAX_ACTIVE_TTL_SECONDS = 60      # spec says 30

    MIN_BRIDGED_TTL_SECONDS = 30
    MAX_BRIDGED_TTL_SECONDS = 300    # spec says 60

    MIN_FROZEN_TTL_SECONDS = 60
    MAX_FROZEN_TTL_SECONDS = 1800

    @staticmethod
    def validate_ttl_config(config: LeaseTTLConfig) → TTLValidationResult
```

Validation checks:
- ✓ ACTIVE TTL must equal spec value (30s)
- ✓ BRIDGED TTL must equal spec value (60s)
- ✓ All values within absolute bounds
- ✓ ACTIVE override authority must be NEVER
- ✓ BRIDGED override authority must be NEVER
