## 三、Phase 1 計劃：Gap Analysis / Phase 1 Plan

### 3.1 前置條件（Phase 1 啟動前必須解決）

| # | 項目 | 解決方案 | 責任 |
|---|------|----------|------|
| P-1 | SSHFS 文件權限修復 | Ubuntu 端 `chmod -R o+r` 或重新掛載 SSHFS | Operator |
| P-2 | 確認 Git 物件可讀 | 修復後測試 `git show HEAD:path` | PM |

### 3.2 任務分解

#### T1.1 — R1 文件審計員：治理文件全文閱讀與結構化（2-3 sessions）

```
輸入：01_source_documents/ 中 22 份 .docx 文件
輸出：02_audit_reports/T1.1_GOVERNANCE_DOC_STRUCTURED_SUMMARY.md

工作內容：
  1. 逐一閱讀 22 份治理文件
  2. 提取每份文件的：
     - 核心要求清單（checklist 化）
     - 約束條件
     - 合規標準
     - 可驗證的指標
  3. 按 D1-D8 維度（AUDIT_CHECKLIST.md）分類整理
  4. 產出結構化摘要（可供後續 gap 對比）

D1-D8 審計維度：
  D1: 目的與一致性
  D2: 範圍與適用性
  D3: 角色與職責
  D4: 流程完整性
  D5: 控制措施
  D6: 報告與記錄
  D7: 合規要求
  D8: 持續改進
```

#### T1.2 — E2+AI-E：代碼深度讀取（需 SSHFS 修復後，1-2 sessions）

```
輸入：program_code/ 全部源碼（修復權限後）
輸出：02_audit_reports/T1.2_CODE_DEEP_READ_SUPPLEMENT.md

工作內容：
  1. 直接讀取 T0.2 無法訪問的核心文件：
     - main.py（FastAPI 入口）
     - pipeline_bridge.py（管線核心）
     - risk_manager.py（風控引擎）
     - paper_trading_engine.py（Paper 引擎）
     - layer2_engine.py + layer2_types.py（AI 推理）
  2. 讀取 ai_agents/bybit_thought_gate/ H1-H5 實現
  3. 補充 T0.2 和 T0.3 中的「推斷」為「確認」
  4. 更新架構依賴圖
```

#### T1.3 — R1+E2：逐條 Gap Analysis（核心任務，2-4 sessions）

```
輸入：T1.1 治理文件結構化摘要 + T0.2/T1.2 代碼架構
輸出：02_audit_reports/T1.3_GAP_ANALYSIS_REPORT.md

工作內容：
  1. 對每份治理文件的每條要求：
     ✅ 已實現（代碼中有對應且通過測試）
     ⚠️ 部分實現（框架存在，細節不符或未測試）
     ❌ 未實現（代碼中無對應）
     🔀 偏離（代碼實現與文件要求不一致）
  2. 按嚴重度分級：Critical / High / Medium / Low
  3. 標註每個 gap 的修復難度估算：
     - S（小，< 1 session）
     - M（中，1-2 sessions）
     - L（大，3+ sessions）
     - XL（超大，需架構重構）
```

#### T1.4 — CC：代碼清潔度驗證（可與 T1.3 並行，1 session）

```
輸入：T0.1 中的問題清單（scripts/ 重複、.orig 文件、secret_files/）
輸出：02_audit_reports/T1.4_CODE_CLEANLINESS_REPORT.md

工作內容：
  1. scripts/ 目錄逐一掃描：標記 duplicate / unique / deprecated
  2. .orig 文件清單 → 確認可安全刪除
  3. settings/secret_files/ 安全審計
  4. main_legacy.py / main_snapshot_stable.py 用途確認
```

#### T1.5 — PM：Gap 優先級排序與 Phase 2 計劃（1 session）

```
輸入：T1.3 Gap Analysis + T1.4 清潔度報告
輸出：02_audit_reports/T1.5_GAP_PRIORITY_AND_PHASE2_PLAN.md

工作內容：
  1. 所有 gap 按 嚴重度 × 修復難度 排序
  2. 分為：Phase 2 必修 / Phase 2 建議 / Phase 3 延後
  3. 制定 Phase 2 任務清單（帶角色分配和 session 估算）
  4. 識別 Phase 2 的依賴鏈和關鍵路徑
```
