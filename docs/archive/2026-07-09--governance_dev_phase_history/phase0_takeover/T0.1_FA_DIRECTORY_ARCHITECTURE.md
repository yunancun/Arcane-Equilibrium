## 4. 代碼重複觀察 / Code Duplication Observation

⚠️ **重大發現：scripts/ 是其他目錄的鏡像複製**

```
program_code/exchange_connectors/bybit_connector/scripts/    ← 約 250 files
```

此目錄幾乎完全複製了以下模塊的文件：
- `ai_agents/bybit_thought_gate/` 的所有文件
- `readonly_observer_pipeline/` 的所有文件
- `risk_control/bybit_local_models_and_risk/` 的所有文件
- `trade_executor/bybit_decision_lease/` 的所有文件
- `trading_strategy/bybit_event_driven/` 的所有文件
- `market_data_processor/bybit_business_events/` 的所有文件
- `io_and_persistence/` 的所有文件
- `misc_tools/` 的所有文件

**影響：** 619 個 .py 中，估計有 ~250 個是重複文件。實際獨立代碼量約 370 個 .py 文件。
**建議：** Phase 1 由 CC 確認 scripts/ 中的文件是否與源模塊完全一致。如一致，應在整理時移除，保留單一真相源。

## 7. 關鍵發現與建議 / Key Findings & Recommendations

### 🔴 高優先級

1. **scripts/ 大量重複文件（~250 files）**
   - 同一個 .py 文件同時存在於功能目錄和 scripts/ 目錄
   - 風險：修改一處忘記同步另一處 → 行為不一致
   - 建議：CC 在 Phase 1 確認是否完全一致；FA 在 Phase 2 提議移除重複

2. **settings/secret_files/ 在 Git 中**
   - SKILL.md 明確說明密鑰應在 srv/ 外的 secrets/ 目錄
   - 需 E3 安全審核確認此目錄是否包含真實密鑰
   - 如包含真實密鑰 = **P0 安全事件**

### 🟡 中優先級

3. **4 個空目錄**（dashboard/, daily_reports/, database_setup/, market_data_collector/）
   - 可能是規劃但未實現的模塊
   - 建議：確認是否在路線圖中，否則清理

4. **docker_projects/ 大部分只有 README**
   - 5/7 子目錄只有 README.md，無實際 Docker 配置
   - 僅 trading_services/ 和 monitoring_services/ 有實質內容

5. **Control API 代碼集中度過高**
   - main.py 單文件包含 113 路由 — 需確認是否已按功能拆分為路由模塊
   - paper_trading_engine.py 和 risk_manager.py 可能是大型單文件

### 🟢 低優先級

6. **日誌和文檔體量大**
   - docs/ 有 194 個文檔文件（md + txt），歷史價值高
   - helper_scripts/ 有 115 個 shell 腳本，大部分是歷史修復腳本
