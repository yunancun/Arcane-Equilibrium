## 二、Critical 級別 Gap（必須在 Phase 1 前修復）

### GAP-C1：對賬層完全缺失（EX-04）
- **要求：** 對賬正式邊界定義 — 訂單/成交/持倉一致性判斷、事故管理
- **現狀：** ❌ 整個模組未實現
- **影響：** Paper 引擎持倉與交易所（Demo）持倉可能失步，無法發現
- **修復：** 建立 ReconciliationEngine（對比 paper_state vs demo_state，標記差異，觸發事故）
- **工作量：** XL（3-4 sessions）

### GAP-C2：授權狀態機完全缺失（SM-01）
- **要求：** 8 個正式狀態（DRAFT→PENDING_APPROVAL→ACTIVE→RESTRICTED→FROZEN→REVOKED→EXPIRED→REJECTED）
- **現狀：** ❌ 僅有 execution_authority 二元標誌，無正式狀態機
- **影響：** 無法追蹤授權生命週期，無法凍結/恢復授權，無審批鏈
- **修復：** 建立 AuthorizationStateMachine 類，實現完整轉換和守衛條件
- **工作量：** XL（3-4 sessions）

### GAP-C3：風控狀態機僅二元（SM-04）
- **要求：** 6 狀態（NORMAL→CAUTIOUS→REDUCED→DEFENSIVE→CIRCUIT_BREAKER→MANUAL_REVIEW）
- **現狀：** 🔀 僅有 blocking/normal 二元判斷
- **影響：** 無漸進降級，直接從正常跳到熔斷，恢復也無需審批
- **修復：** 擴展 RiskManager 為 6 狀態機，加入轉換守衛和逐級降級邏輯
- **工作量：** L（2-3 sessions）

### GAP-C4：決策租約未強制執行（SM-02）
- **要求：** 9 狀態 + 時間門 + 過期強制
- **現狀：** 🔀 Schema 已定義但硬編碼為不發射模式
- **影響：** Phase 1 啟用時無生命週期管理，租約可能永不過期
- **修復：** 實現 LeaseStateMachine + 過期守護線程 + 強制撤銷機制
- **工作量：** L（2-3 sessions）

## 三、High 級別 Gap（Phase 1 期間修復）

### GAP-H1：OMS 執行狀態機不完整（SM-03）
- **要求：** 11 狀態（含 PENDING, APPROVED, RECONCILING, COMPLETED, ABORTED）
- **現狀：** ⚠️ 有 7 狀態（缺 PENDING, APPROVED, RECONCILING, COMPLETED）
- **影響：** 無 RECONCILING 安全閘門，持倉同步風險
- **工作量：** L（2 sessions）

### GAP-H2：多 Agent 協調缺失（EX-06, DOC-04）
- **要求：** Scout/Strategist/Guardian/Analyst/Executor + OpenClaw Conductor
- **現狀：** ⚠️ Guardian(RiskManager) + Executor(PaperEngine) + Strategist(Layer2) 存在，但 Scout/Conductor/正式 Analyst 缺失
- **影響：** 無正式的 Agent 間通訊協議和衝突仲裁
- **工作量：** L（2-3 sessions）

### GAP-H3：審計軌跡不完整（DOC-07, cross-cutting）
- **要求：** 不可變審計日誌 + transition_id + trigger_event_id + approved_by
- **現狀：** ⚠️ PaperEngine 有 _audit()，但僅記憶體中，無持久化不可變日誌
- **影響：** 系統重啟後審計丟失，無法事後追溯
- **工作量：** M（1-2 sessions）

### GAP-H4：Portfolio 級別風控缺失（EX-01, DOC-01 §5.16）
- **要求：** 相關性矩陣計算 + 行業集中度限制 + 儲備金規則
- **現狀：** ⚠️ max_correlated_exposure_pct 有配置但無運行時計算
- **影響：** 多倉位同方向暴露風險未被限制
- **工作量：** M（1-2 sessions）

### GAP-H5：事故→狀態機集成缺失（DOC-07 + SM cross-cutting）
- **要求：** 事故事件觸發狀態機轉換（如 incident→FROZEN）
- **現狀：** ❌ 無事故事件模型，無事故→狀態機聯動
- **工作量：** M（1-2 sessions）
