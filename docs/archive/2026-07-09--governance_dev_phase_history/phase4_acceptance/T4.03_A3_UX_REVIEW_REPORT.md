# Phase 4 Acceptance: A3 UX Audit — Governance State Exposure in Console

**Date**: 2026-03-30
**Reviewer**: A3 (UX Auditor)
**Scope**: 10-Tab Trading Console (control_api_v1/app/static)
**Objective**: Verify if governance states from GovernanceHub are properly exposed to the operator in the GUI

---

## Executive Summary

**Finding**: The 10-tab console has **minimal governance state exposure**. Critical governance states (authorization, risk level, decision leases, reconciliation status) are **NOT displayed** in any tab, despite the GovernanceHub API being fully implemented (`/api/v1/governance/*`).

**Risk Level**: **HIGH** for operator oversight. Operators cannot visually inspect:
- Current authorization state (ACTIVE/RESTRICTED/FROZEN/PENDING_APPROVAL)
- Risk governor level (NORMAL → MANUAL_REVIEW, 0-5 scale)
- Decision lease tracking (active count, total tracked)
- Reconciliation health (last check, consistency status)
- Cross-SM governance mode (NORMAL/RESTRICTED/FROZEN/MANUAL_REVIEW)

**Recommendation**: Create a dedicated **"Governance"** tab (11th tab) + add governance status badges to the **System** tab header.

## Governance Status Object (Not Displayed)

From `governance_hub.py`, the `GovernanceStatus` dataclass exposes:

```python
@dataclass
class GovernanceStatus:
    # What the API provides but GUI doesn't display:
    enabled: bool

    # Authorization (SM-01)
    auth_state: str           # ACTIVE | RESTRICTED | FROZEN | PENDING_APPROVAL | NONE
    auth_expires_at_ms: int
    auth_scope: dict
    auth_pending_approval: bool

    # Risk Governor (SM-04)
    risk_level: int           # 0-5 (NORMAL to MANUAL_REVIEW)
    risk_level_name: str
    risk_escalation_reason: str

    # Decision Lease (SM-02)
    active_leases_count: int
    total_leases_tracked: int

    # Reconciliation (EX-04)
    last_reconciliation_ms: int
    last_reconciliation_result: str
    is_consistent: bool

    # Audit
    incident_count: int
    callback_errors: int
```

**None of these fields are displayed in the console.**

## Operator Pain Points

**Scenario 1: Risk Escalation**
- System escalates from NORMAL → DEFENSIVE due to drawdown
- Operator sees risk pressure metric move, but **does NOT see**:
  - Which risk level (3) was triggered
  - Why authorization didn't restrict (auth state unknown)
  - If cross-SM cascading happened

**Scenario 2: Authorization Expiration**
- Auth is set to expire in 1 hour
- Operator has **no countdown timer**, no expiration alert
- System might freeze without warning

**Scenario 3: Reconciliation Failure**
- Paper/Demo account diverges
- Operator has **no visibility** into:
  - When last recon ran
  - Is system consistent?
  - What discrepancies exist?

**Scenario 4: Incident Investigation**
- Operator wants to audit what happened
- Tab shows current state but **no incident count**, **no governance audit trail**

## Conclusion

The OpenClaw 10-tab console provides **excellent operational visibility** for trading metrics, positions, and risk management, but **completely lacks governance state exposure**. This is a critical gap for Phase 4 acceptance, as operators cannot visually monitor:

1. Authorization validity and scope
2. Risk governor escalation levels
3. Decision lease tracking
4. Reconciliation consistency

**Recommendation**:
- **Create an 11th "Governance" tab** with unified dashboard for all 4 SMs (Auth, Risk Governor, Decision Lease, Reconciliation)
- **Add governance badges** to System tab header for at-a-glance status
- **Integrate governance API** endpoints into console

This would enable operators to fully oversee the governance system, approve pending actions, and respond to incidents in real-time.
