四、项目总目标与绝对不能忘的根原则
============================================================

------------------------------------------------------------
4.1 项目总目标
------------------------------------------------------------

本项目不是一个普通自动下单程序，也不是单纯的量化脚本集合。

真正要做的是一个：
“以 OpenClaw 为中枢、Bybit 为主交易所、Binance 为辅助交易所的 AI Agent 交易系统。”

它的长期目标不是“尽快实盘”，而是：

1. 风险可控
2. 可审计
3. 可学习
4. 可逐步放权
5. 在成本、延迟、风控、权限治理都受控的前提下，追求长期净收益最大化

------------------------------------------------------------
4.2 四条根原则
------------------------------------------------------------

原则 1：不是看 gross PnL，而是 net PnL  
必须计入：
- AI API 成本
- 手续费
- 电费
- 设备折旧
- 基础设施成本
- 后续维护成本

原则 2：AI 输出不能当成即时命令  
AI 有延迟，市场有时效。
因此 AI 更应输出 decision lease（决策租约），而不是“现在立刻买卖”的直接命令。
本地层必须继续做：
- freshness 校验
- timing 校验
- risk 校验
- permission / legality 校验
- execution gate

原则 3：权限不能按时间自动升级，必须按表现赢得  
不能因为系统运行久了就自动放权。
必须在已经证明有效的子场景中局部放权。

原则 4：最终形态不是机械 bot，而是长期进化型交易个体  
它要会：
- 观察
- 判断
- 审计
- 学习
- 自我观测
- 产出经营报告
- 在严格边界下逐步争取更高权限

十、H0 / H / I 的最终设计理解（这是新对话最容易丢的部分）
============================================================

------------------------------------------------------------
10.1 H0：Local Deterministic Judgment Core
------------------------------------------------------------

H0 的正式角色是：
本地确定性判断内核

它不直接下单，不替代 AI，不把系统拉回人工半自动。
它的职责是：

- 以最低延迟、最低成本、最高确定性
- 先完成本地应当完成的判断
- 把显然不该做、显然不值得问 AI 的情况全部挡掉

H0 应回答的不是“买还是卖”，而是：
1. 现在是否具备交易资格？
2. 如果不具备，阻塞原因是什么？
3. 如果具备，属于哪种级别的资格？
4. 是否值得升级到 AI 治理层？
5. 扣除现实摩擦后，是否还存在净可交易边际？

------------------------------------------------------------
10.2 H：AI Governance
------------------------------------------------------------

H 不是“让 AI 直接决定交易”，而是让 AI 成为受治理的认知资源。

H 解决的问题是：
- 该不该想
- 值不值得问
- 问谁最划算
- 调用多少预算
- 在何种情况下必须禁止调用

H 的关键定位：
- 只读
- 不放权
- 可审计
- 预算受控
- 可被本地层前置筛掉

------------------------------------------------------------
10.3 I：Decision Lease / Execution Authorization Control Plane
------------------------------------------------------------

I 不是“给 AI 开执行权限”，而是把 AI 输出变成一个：
可审计、可衰减、可撤销、可 supersede 的控制平面。

关键理念：
1. lease 不是执行许可本体
2. lease 必须可审计、可撤销、可过期、可重放
3. 影子控制平面阶段必须维持：
   - execution_authority = not_granted
   - decision_lease_emitted = false
   - live_operator_ack_enabled = false
   - runtime_still_protected = true

一句话：
H 解决“AI 怎么看”，
I 解决“谁有资格把看法变成权力”。

十四、当前最关键的概念词典
============================================================

- readonly observer  
  当前主系统真实角色：只读观察，不允许执行。

- healthy_no_business_events_yet  
  业务事件侧健康，但当前还没有看到真实 wallet / order / execution / position 业务事件。
  这是健康空态，不是故障。

- healthy_but_empty  
  event-driven scaffold 当前高层语义：结构健康，但为空，不能推进到交易态。

- observe_only_retained  
  系统合理继续保持 observe-only，而不是因为出错。

- decision lease  
  未来 AI 输出的受治理决策租约，不是即时执行命令。

- thought_gate  
  本地先判断“是否值得问 AI”的门。

- query_budget  
  控制 AI 调用预算，防止无限询问。

- model_router  
  根据任务价值、复杂度、成本路由不同模型。

- structural hardening  
  不是正式业务完工，而是先把结构层、测试层、审计层补强到可维护状态。

- chapter consistency  
  章节级一致性检查，确认某章的 latest / contract / summary / handoff / final audit 语义一致。

- master regression  
  跨章节总回归，确认多章结果能同时成立且不污染主 runtime。
