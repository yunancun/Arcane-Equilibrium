A. Shared semantic helper to add to all step-1 scripts
------------------------------------------------------------------

Add a helper equivalent to:

    def _norm_tier(value):
        if value is None:
            return "none"
        return str(value).strip().lower()

    def is_legal_no_call(route_plan, selected_ai_tier, should_call_ai):
        return (
            str(route_plan).strip().lower() == "route_skip"
            and _norm_tier(selected_ai_tier) == "none"
            and should_call_ai is False
        )

Also add a stricter sanity helper:

    def is_inconsistent_no_call(route_plan, selected_ai_tier, should_call_ai):
        route_skip = str(route_plan).strip().lower() == "route_skip"
        tier_none = _norm_tier(selected_ai_tier) == "none"
        return len({route_skip, tier_none, should_call_ai is False}) not in (1, 3)

Rule:
- all three aligned => legal no-call
- partial mismatch => semantic inconsistency => fail hard

- response_state = response_not_applicable_legal_no_call
- overall_ok = true
- no longer require invocation/provider response/parsed json
- but still preserve a hard error if should_call_ai=true and invocation was skipped

- selected_ai_tier
- should_call_ai

Recommended output fields
-------------------------

- overall_ok

- no_call_terminal_accepted      # bool
- invocation_required            # bool
- invocation_attempted
- provider_response_present
- parsed_json_present
- hard_errors                    # list[str]
- soft_warnings                  # list[str]
- semantic_flags                 # list[str]
- route_plan
- selected_ai_tier
- should_call_ai

Decision matrix
---------------
Case 1: legal no-call
- route_plan=route_skip
- selected_ai_tier=none
- should_call_ai=false
Expected result:
- overall_ok = true
- terminal_mode = legal_no_call
- response_state = response_not_applicable_legal_no_call
- invocation_required = false
- no_call_terminal_accepted = true
- hard_errors = []
- semantic_flags include:
  - legal_no_call_terminal_state
  - provider_response_not_applicable
  - parsed_json_not_applicable
- DO NOT assert invocation_attempted/provider_response_present/parsed_json_present are true

Case 2: actual invocation path
- should_call_ai=true
Expected result:
- preserve current strict contract checks
- invocation/provider response/parsed json still required

Case 3: inconsistent semantic state
Examples:
- should_call_ai=false but selected_ai_tier=light
- route_skip but should_call_ai=true
- should_call_ai=true but tier=none
Expected result:
- overall_ok = false
- response_state = response_semantic_inconsistency
- hard_errors include semantic mismatch details

Case 4: call required but skipped
- should_call_ai=true and invocation_attempted=false
Expected result:
- overall_ok = false
- hard_errors include invocation_missing_when_call_required

Suggested pseudo-implementation
-------------------------------
1. Read upstream H1-C / H1-R / H1-F latest payloads.
2. Derive:
       legal_no_call = is_legal_no_call(...)
       inconsistent = is_inconsistent_no_call(...)
3. Branch:
   a) inconsistent => fail hard
   b) legal_no_call => success terminal no-call
   c) should_call_ai=true => run existing strict invocation contract
   d) else => fail hard because semantics are under-specified

G. Idle-account downgrade rule
------------------------------------------------------------------

Add or reuse a helper wherever account activity is audited:

    def is_expected_idle_account(system_mode, execution_state, position_count, order_count, execution_count):
        return (
            str(system_mode).strip().lower() == "read_only"
            and str(execution_state).strip().lower() == "disabled"
            and int(position_count or 0) == 0
            and int(order_count or 0) == 0
            and int(execution_count or 0) == 0
        )

When true, downgrade these to info/idle_flags, not blocker/failure:
- no_open_positions
- no recent order history
- no recent execution history
- control-only private WS traffic with successful auth/subscribe

Do NOT downgrade when:
- execution enabled
- any position/order/execution count > 0
- missing private source while account is active
