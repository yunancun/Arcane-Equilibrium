### 7. Lock Acquisition Failure (High Concurrency) ✅

**Code Path:** `governance_hub.py:354-365`

```python
try:
    with self._lock:
        # ... check logic ...
        self._cached_auth_state = (result, now_ms)

        logger.debug(f"Error in is_authorized: {e}")
```

**Analysis:**

- External thread lock contention → exception handler returns False
- Cache update is atomic (tuple assignment in Python is atomic)

**Result:** ✅ **PASS** — Exception handler fails-closed.

### 8. Cache Expiry Race Condition ✅

**Code Path:** `governance_hub.py:342-346`

```python
now_ms = int(time.time() * 1000)
if self._cached_auth_state is not None:
    cached_result, cached_ts_ms = self._cached_auth_state
    if now_ms - cached_ts_ms < self._cache_ttl_ms:
        return cached_result
```

**Analysis:**
- Cache is **lock-free for reads** (no writes during reads)
- Write path (line 364) is protected by lock
- Timestamp comparison cannot race: tuple unpacking is atomic
- Worst case: stale cache returns for ≤100ms after expiry (TTL = 100ms)
- No authorization state change can occur without cache invalidation (line 325)

**Invariant:** `_invalidate_auth_cache()` called on all state transitions:
- `_on_risk_escalation()` → calls `_invalidate_auth_cache()`
- `_on_reconciliation_mismatch()` → calls `_invalidate_auth_cache()`
- Authorization SM state change → invalidates cache

**Result:** ✅ **PASS** — Race condition prevented by cache invalidation.

**Test Coverage:**
- `test_is_authorized_cache_expiry_race` (NEW — added in test_governance_hub.py)

**Risk Level:** NONE (invariant enforced)
