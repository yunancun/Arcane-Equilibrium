### 實現內容 / Implementation Details

**1. 閘門條件配置（PaperLiveGateConfig）**

根據 DOC-08 §11 定義的條件：

| 條件 | 默認值 | 說明 |
|------|--------|------|
| min_paper_duration_weeks | 4 | 最少紙盤運行時長 |
| min_trades | 500 | 最少完成的往返交易 |
| min_win_rate_percent | 30.0 | 最少胜率（含手續費後） |
| min_net_pnl_threshold | 0.0 | 淨 PnL 必須為正 |
| min_sharpe_ratio | 0.5 | 最少夏普比 |
| max_drawdown_percent | 100.0 | 最大回撤（當前階段無硬上限） |
| min_profit_factor | 1.2 | 最少利潤因子（平均赢/平均損） |
| min_audit_trail_completeness_percent | 99.0 | 審計鏈完整性 |
| max_reconciliation_mismatch_percent | 0.1 | 對賬不匹配率 |
| max_consecutive_losses | 10 | 連續虧損自動暫停閾值 |
| require_no_major_incidents | True | 要求無重大風險事件 |

**2. 閘門評估（PaperLiveGate.evaluate_gate()）**

- 同時評估所有 11 項條件
- 返回 GateCheckResult，包含：
  - `passed`: bool — 所有條件是否均通過
  - `gate_status`: GateStatus — 閘門狀態
  - `criteria_results`: dict — 各項條件詳細檢查結果
  - `blocking_reasons`: list — 未通過的原因清單
  - `timestamp_ms`: int — 評估時間戳

**3. 單項條件檢查（11 個獨立方法）**

| 方法 | 檢查內容 |
|------|---------|
| `_check_duration()` | 紙盤最少 4 週（EX-05 §4.1 TABLE 3 #4） |
| `_check_trade_count()` | 最少 500 筆完成的往返交易 |
| `_check_win_rate()` | 胜率 > 30%（含手續費後） |
| `_check_net_pnl()` | 淨 PnL > 0 |
| `_check_sharpe()` | 夏普比 >= 0.5 |
| `_check_drawdown()` | 最大回撤 <= 配置上限 |
| `_check_profit_factor()` | 利潤因子 >= 1.2 |
| `_check_audit_trail()` | 審計鏈完整性 >= 99% |
| `_check_reconciliation()` | 對賬不匹配 <= 0.1% |
| `_check_consecutive_losses()` | 連續虧損 < 10（自動暫停閾值） |
| `_check_major_incidents()` | 無重大事件 |

**4. 閘門狀態轉移（GateStatus 枚舉）**

```
NOT_EVALUATED
    ↓
IN_PROGRESS
    ↓
GATE_PASSED  or  GATE_FAILED
    ↓
OPERATOR_APPROVAL_PENDING  (if GATE_PASSED)
    ↓
OPERATOR_APPROVED  or  OPERATOR_REJECTED
```

關鍵設計：
- 閘門通過是必要條件，不充分；Operator 批准是充要條件
- EX-05 §4.1 規範：第一次新策略類型需要 Operator 批准；同類型後續變體自動晉升（若條件達成）
- 本實現涵蓋評估層，與 Strategy Incubation Pipeline 的後續邏輯分離

**5. Operator 批准流程（submit_operator_approval()）**

```python
# 僅在 GATE_PASSED 後可調用
result = gate.submit_operator_approval(
    approved=True,                    # or False
    operator_id="op_nancun_001",      # 審計追蹤
    reason="Metrics solid, risk acceptable"
)
# 返回更新後的 GateCheckResult，gate_status 變為 OPERATOR_APPROVED 或 OPERATOR_REJECTED
```

- Thread-safe（使用 Lock）
- 拒絕不需要條件重新評估
- 審計回調記錄所有批准/拒絕事件

**6. 狀態查詢 API**

| 方法 | 功能 |
|------|------|
| `get_gate_status()` | 獲取最後一次評估結果 |
| `get_remaining_requirements()` | 若未通過，返回尚未滿足的條件及缺口 |

Example:
```python
remaining = gate.get_remaining_requirements()
# {
#   "win_rate": {"actual": 28.5, "required": 30.0, "reason": "..."},
#   "trade_count": {"actual": 450, "required": 500, ...}
# }
```

**7. 線程安全與審計**

- 所有狀態修改被 `threading.Lock` 保護
- `audit_callback` 集成（簽名 `(event_type: str, event_data: dict) -> None`）
  - 事件：`gate_evaluated`, `operator_approval_submitted`
  - 回調異常被捕獲，不影響主流程
- 支持序列化：`to_dict()`, `to_json()`, `from_dict()`

**8. CriterionCheckResult 詳細結果**

每個條件檢查返回：
```python
@dataclass
class CriterionCheckResult:
    criterion_name: str              # 條件名稱
    status: CheckStatus              # PASSED / FAILED / NOT_APPLICABLE
    actual_value: Any                # 實際值
    required_value: Any              # 要求值
    passed: bool                      # 是否通過
    reason: Optional[str]            # 詳細原因
    timestamp_ms: int                # 檢查時間戳
```

## 後續集成點 / Integration Points

1. **Paper Trading Engine**
   - 在紙盤交易結束時調用 `gate.evaluate_gate()`
   - 傳入實時的交易指標（時長、交易量、胜率、Sharpe 等）

2. **Operator Dashboard**
   - 顯示 `get_remaining_requirements()` 提示 Operator 進度
   - 若 GATE_PASSED，顯示批准/拒絕按鈕，調用 `submit_operator_approval()`

3. **Strategy Incubation Pipeline**
   - 在 Live Promotion 流程中檢查 gate_status == OPERATOR_APPROVED
   - 第一次新策略類型已由 Operator 批准，後續同類型自動晉升

4. **Learning Analyst (EX-05 L2+)**
   - 可讀取 gate 評估結果以優化策略參數
   - 不能繞過或修改閘門規則

## 使用示例 / Usage Example

```python
from app.paper_live_gate import PaperLiveGate, PaperLiveGateConfig

# 初始化閘門引擎
config = PaperLiveGateConfig(
    min_paper_duration_weeks=4,
    min_trades=500,
    min_win_rate_percent=30.0,
)

def audit_callback(event_type: str, event_data: dict) -> None:
    print(f"[AUDIT] {event_type}: {event_data}")

gate = PaperLiveGate(config=config, audit_callback=audit_callback)

# 評估閘門（在紙盤運行週期結束時）
result = gate.evaluate_gate(
    paper_start_time_ms=1700000000000,  # 4 週前
    total_trades=650,
    win_rate_percent=32.5,
    net_pnl=2500.0,
    sharpe_ratio=0.75,
    max_drawdown_percent=18.0,
    profit_factor=1.4,
    audit_trail_completeness_percent=99.8,
    reconciliation_mismatch_percent=0.05,
    consecutive_losses=5,
    has_major_incidents=False,
)

print(f"Gate Passed: {result.passed}")
print(f"Status: {result.gate_status.value}")

if result.passed:
    # Operator 批准
    approved = gate.submit_operator_approval(
        approved=True,
        operator_id="op_nancun_001",
        reason="All metrics solid, ready for supervised live",
    )
    print(f"Operator Status: {approved.operator_approval_status.value}")
else:
    # 顯示缺口
    remaining = gate.get_remaining_requirements()
    for criterion, info in remaining.items():
        print(f"  {criterion}: {info['actual']} vs {info['required']}")
```
