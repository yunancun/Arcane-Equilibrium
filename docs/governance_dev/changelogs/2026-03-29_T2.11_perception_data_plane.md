### 實現內容 / Implementation Details

**1. 認知級別標記（EX-07 §1 / DOC-01 §5.10）**
- CognitiveLevel 枚舉：fact / inference / hypothesis
- 核心原則：交易所 API 數據 = fact，AI/搜索數據 = inference
- 未標記的 inference 不得進入決策鏈
- 8 種數據源類型各有預設認知級別

**2. 數據鮮度追蹤（EX-07 §2.1 TABLE 2）**
- FRESH (< 5min)、RECENT (5-30min)、STALE (30min-2h)、EXPIRED (> 2h)
- 自動計算鮮度
- EXPIRED 數據不可進入決策鏈

**3. 數據質量評估（EX-07 §2.2）**
- 四維度：completeness、consistency、latency、source_reliability
- 加權綜合評分
- 低質量數據阻止進入決策鏈

**4. 數據質量驅動的風險降級（EX-07 §2.3）**
- STALE 價格 → 不開新倉
- EXPIRED 價格 → CAUTIOUS 模式
- WebSocket 斷連 > 30s → NO_NEW_ENTRY
- WebSocket 斷連 > 5min → REDUCED
- REST 連續 3 次失敗 → DEFENSIVE

**5. Agent 數據訪問控制（EX-07 §6 TABLE 5）**
- 5 個 Agent × 8 個數據類別 = 40 條訪問規則
- Scout 不可訪問賬戶/持倉（防止持倉偏見）
- Executor 不可訪問搜索結果（只執行不判斷）
- P0/P1 參數所有 Agent 只讀
- Guardian 可寫 P2 風控參數

**6. 漂移防護（EX-07 §7）**
- 搜索數據標記為 fact → 自動糾正為 inference + 嚴重漂移警告
- 漂移警告記錄和查詢
- Perplexity/DuckDuckGo/Ollama 輸出強制為 inference

**7. 感知平面引擎 PerceptionPlane**
- register_data()：註冊數據，自動應用認知級別和鮮度
- get_data()：帶 Agent 訪問控制的數據檢索
- get_decision_eligible_data()：只返回可進入決策鏈的數據
- validate_for_decision()：決策前驗證
- assess_degradation()：數據質量→風險降級評估
- 審計回調集成
