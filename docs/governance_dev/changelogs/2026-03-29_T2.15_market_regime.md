This task implements the complete formal dataclass system for market regime specification in DOC-03 § 6.3, including:
- **MarketRegime Enum**: 9 regime types (TRENDING_UP, TRENDING_DOWN, RANGING, SQUEEZE, HIGH_VOLATILITY, LOW_VOLATILITY, BREAKOUT, REVERSAL, UNKNOWN)
- **RegimeConfidence Enum**: HIGH/MEDIUM/LOW mapped to float 0-1 range
- **RegimeTimeframe Enum**: M5, M15, H1, H4, D1 (multi-timeframe support)
- **MarketRegimeSnapshot Dataclass**: Core regime object with symbol, regime, confidence, timeframe, detected_at_ms, volatility_percentile, atr_value, volume_profile, supporting_indicators, metadata
- **RegimeTransition Dataclass**: Records regime changes with from_regime, to_regime, transition_at_ms, trigger_reason
- **MarketRegimeTracker Class**: Multi-timeframe regime tracking, cross-timeframe conflict detection (EX-06 §6.4), history management, thread-safe, JSON serialization

### Core Features

1. **Multi-Timeframe Regime Detection**
   - Track regimes simultaneously across M5, M15, H1, H4, D1 for same symbol
   - Automatic conflict detection across timeframes (EX-06 §6.4)
   - Two conflict types: directional_divergence (opposite trends), trend_range_divergence (trending vs ranging)

2. **Confidence Grading**
   - HIGH: confidence > 0.75
   - MEDIUM: 0.5 ≤ confidence ≤ 0.75
   - LOW: confidence < 0.5
   - Float values auto-mapped to enum levels

3. **Regime Transition Tracking**
   - Auto-generated transition record on every regime change with timestamp & reason
   - Configurable max history length per symbol/timeframe

4. **Serialization Support**
   - JSON roundtrip: to_json/from_json preserves all semantics
   - Dict serialization: to_dict/from_dict with enum value preservation
   - Tracker full state serialization: can persist & restore complete state

5. **Thread Safety**
   - RLock protects all read/write operations
   - Concurrent updates/reads safe (tested with 5 threads)
   - Suitable for multi-agent parallel access

### Key Classes

#### 1. MarketRegimeSnapshot (Dataclass)
Core regime state object per DOC-03 § 6.

**Fields:**
- `symbol` (str): Trading pair e.g. "BTCUSDT"
- `regime` (MarketRegime): Current regime enum
- `confidence` (float): 0.0-1.0 confidence score
- `timeframe` (RegimeTimeframe): M5/M15/H1/H4/D1
- `detected_at_ms` (int): UTC epoch_ms timestamp (per DOC-03 § 1.6)
- `volatility_percentile` (float): 0-100 historical distribution
- `atr_value` (float): Average True Range absolute value
- `volume_profile` (Dict): e.g. {"high": 0.6, "normal": 0.4}
- `supporting_indicators` (Dict): Technical indicators used (SMA, RSI, Bollinger, etc.)
- `metadata` (Dict): Algorithm-specific context

**Methods:**
- `confidence_level()` → RegimeConfidence: Maps float to enum
- `is_high_confidence()` → bool
- `is_medium_confidence()` → bool
- `is_low_confidence()` → bool
- `to_dict()` → Dict: Serialization with enum names
- `to_json()` → str: JSON serialization
- `from_dict(d)` → MarketRegimeSnapshot: Deserialization
- `from_json(s)` → MarketRegimeSnapshot: JSON deserialization

#### 2. RegimeTransition (Dataclass)
Records every regime change event.

**Fields:**
- `transition_id` (str): Auto-generated "rt:xxxxx"
- `from_regime` (MarketRegime)
- `to_regime` (MarketRegime)
- `transition_at_ms` (int): UTC epoch_ms
- `timeframe` (RegimeTimeframe)
- `symbol` (str): Trading pair
- `trigger_reason` (str): e.g. "volatility_spike", "breakout_detected"
- `trigger_confidence` (float): Confidence of the new regime
- `metadata` (Dict): Additional context

#### 3. MarketRegimeTracker (Class)
Central regime tracking system with multi-timeframe awareness.

**Core Operations:**

| Operation | Signature | Returns | Purpose |
|-----------|-----------|---------|---------|
| `update_regime()` | symbol, regime, confidence, timeframe, ... | (is_transition, RegimeTransition?) | Update regime, detect transition |
| `get_current_regime()` | symbol, timeframe | MarketRegimeSnapshot? | Query current regime |
| `get_all_current_regimes()` | symbol | Dict[timeframe → snapshot] | All timeframes for symbol |
| `get_regime_history()` | symbol, timeframe, limit | List[RegimeTransition] | Recent transitions |
| `detect_multi_timeframe_conflict()` | symbol, confidence_threshold | Dict? | Check for cross-TF conflicts |
| `get_stats()` | — | Dict | Internal statistics |
| `reset_stats()` | — | None | Clear statistics |

**Multi-Timeframe Conflict Detection (EX-06 §6.4):**
```
Conflict Type A: DIRECTIONAL_DIVERGENCE
  Scenario: H4 TRENDING_UP + M5 TRENDING_DOWN (both high conf)
  Severity: HIGH
  Action: Alert, recalibrate strategy parameters

Conflict Type B: TREND_RANGE_DIVERGENCE
  Scenario: H1 TRENDING_UP + H4 RANGING (both high conf)
  Severity: MEDIUM
  Action: Caution zone, reduce position size, tighter stops
```

## Usage Examples / 使用示例

### Example 1: Single-Symbol Multi-Timeframe Tracking (简单多时间框架追踪)

```python
from app.market_regime import (
    MarketRegime, RegimeTimeframe, MarketRegimeTracker
)

tracker = MarketRegimeTracker()

# M5 shows breakout
is_trans, trans = tracker.update_regime(
    symbol="BTCUSDT",
    regime=MarketRegime.BREAKOUT,
    confidence=0.76,
    timeframe=RegimeTimeframe.M5,
    atr_value=450.0,
    volatility_percentile=85,
    trigger_reason="resistance_broken",
    supporting_indicators={"bb_width": 280, "rsi": 72}
)

# H1 confirms trend
tracker.update_regime(
    symbol="BTCUSDT",
    regime=MarketRegime.TRENDING_UP,
    confidence=0.82,
    timeframe=RegimeTimeframe.H1,
    atr_value=480.0
)

# Check for conflicts
conflict = tracker.detect_multi_timeframe_conflict("BTCUSDT")
print(f"Conflict detected: {conflict}")  # None if aligned

# Get current state
h1_regime = tracker.get_current_regime("BTCUSDT", RegimeTimeframe.H1)
print(f"H1 Regime: {h1_regime.regime.value}, Confidence: {h1_regime.confidence_level}")
```

### Example 2: Conflict Detection & Alert (冲突检测与警报)

```python
# H4 trending up, M5 trending down → conflict
tracker.update_regime(
    "BTCUSDT", MarketRegime.TRENDING_UP, 0.80, RegimeTimeframe.H4
)
tracker.update_regime(
    "BTCUSDT", MarketRegime.TRENDING_DOWN, 0.78, RegimeTimeframe.M5
)

conflict = tracker.detect_multi_timeframe_conflict("BTCUSDT", confidence_threshold=0.75)
if conflict:
    if conflict["severity"] == "high":
        # Pause new entries, tighten risk management
        logger.warning(f"High-severity conflict: {conflict}")
```

### Example 3: Serialization for Persistence (序列化保存状态)

```python
# Save tracker state
json_state = tracker.to_json()
with open("regime_state.json", "w") as f:
    f.write(json_state)

# Restore tracker state
restored = MarketRegimeTracker.from_json(json_state)
snapshot = restored.get_current_regime("BTCUSDT", RegimeTimeframe.H1)
```

### Example 4: History Query (历史查询)

```python
# Get recent regime transitions for a symbol/timeframe
history = tracker.get_regime_history(
    "BTCUSDT",
    RegimeTimeframe.H1,
    limit=10  # Last 10 transitions
)

for trans in history:  # Most recent first
    print(f"{trans.from_regime.value} → {trans.to_regime.value} at {trans.transition_at_ms}")
    print(f"  Reason: {trans.trigger_reason}")
```
