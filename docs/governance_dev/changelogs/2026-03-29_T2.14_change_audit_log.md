### Key Classes and Methods / 關鍵類和方法

#### ChangeAuditLog Engine
```python
class ChangeAuditLog:
    # Record a change
    def record_change(
        change_type: ChangeType,
        who: str,
        what: str,
        reason: str,
        old_value: Optional[Any] = None,
        new_value: Optional[Any] = None,
        affected_components: Optional[List[str]] = None,
        auto_approve: bool = False,
    ) -> ChangeRecord

    # Record an emergency change (bypasses approval)
    def record_emergency_change(
        change_type: ChangeType,
        who: str,
        what: str,
        reason: str,
        old_value: Optional[Any] = None,
        new_value: Optional[Any] = None,
        affected_components: Optional[List[str]] = None,
    ) -> ChangeRecord

    # Approve a pending change
    def approve_change(
        change_id: str,
        approved_by: str,
        approval_reason: Optional[str] = None,
    ) -> Optional[ChangeRecord]

    # Reject a pending change
    def reject_change(
        change_id: str,
        rejected_by: str,
        rejection_reason: str,
    ) -> Optional[ChangeRecord]

    # Query change history with filters
    def get_change_history(
        start_time: Optional[float] = None,
        end_time: Optional[float] = None,
        change_type: Optional[ChangeType] = None,
        initiator: Optional[str] = None,
        approval_status: Optional[ChangeApprovalStatus] = None,
    ) -> List[ChangeRecord]

    # Get all changes awaiting approval
    def get_pending_approvals() -> List[ChangeRecord]

    # Get emergency changes older than 24 hours
    def get_emergency_changes_pending_review() -> List[ChangeRecord]

    # Export to JSON format
    def export_to_json(pretty: bool = False) -> str
```

### ChangeRecord Immutability / ChangeRecord 不可變性
```python
@dataclass(frozen=True)
class ChangeRecord:
    """Immutable change record with full audit trail."""
    change_id: str                           # Unique identifier (e.g., chg:00000001)
    change_type: ChangeType                  # Type of change
    who: str                                 # Initiator name
    when: float                              # Timestamp (seconds since epoch)
    when_ms: int                             # Timestamp (milliseconds since epoch)
    what: str                                # Human-readable description
    reason: str                              # Reason for change
    old_value: Optional[str]                 # Previous value (JSON serialized)
    new_value: Optional[str]                 # New value (JSON serialized)
    affected_components: List[str]           # List of affected components
    approval_status: ChangeApprovalStatus    # Current approval status
    approved_by: Optional[str]               # Who approved (if applicable)
    approval_timestamp: Optional[float]      # When approved
    approval_reason: Optional[str]           # Reason for approval/rejection
    rollback_info: Optional[Dict[str, Any]]  # Rollback details if applicable
    recorded_at_ms: int                      # When recorded
```

### Change Recording Flow / 變更記錄流

**Normal Change (YELLOW - Requires Approval):**
```
1. Agent/System calls record_change() with auto_approve=False
   → Status: PENDING
2. Operator receives notification of pending change
3. Operator calls approve_change() or reject_change()
   → Status: APPROVED or REJECTED
4. Change is immutable from this point forward
```

**Auto-Approved Change (GREEN):**
```
1. Agent/System calls record_change() with auto_approve=True
   → Status: AUTO_APPROVED
   → Automatically approved by system
2. Change is immediately effective
```

**Emergency Change (Requires Post-Review):**
```
1. System calls record_emergency_change()
   → Status: EMERGENCY_BYPASSED
   → Change takes effect immediately
2. Change appears in get_emergency_changes_pending_review() queue
3. Operator must review and approve within 24 hours
4. After approval: Status changes to APPROVED
```

## Design Decisions / 設計決策

### 1. Immutable ChangeRecord / 不可變的 ChangeRecord
**Decision:** Use frozen dataclass
**Rationale:**
- Safety: Prevents accidental modification of audit records
- Compliance: Matches DOC-06 §5 requirement for append-only immutability
- Traceability: Maintains clear historical record

### 2. Append-Only Semantics / 仅追加語義
**Decision:** Store records in list, replace entire records for status changes
**Rationale:**
- Safety: Never loses historical state
- Auditability: Each approval/rejection creates new record snapshot
- Simplicity: Clear before/after states for all transitions

### 3. Thread Safety / 線程安全
**Decision:** Use threading.RLock() for all operations
**Rationale:**
- Concurrency: Multiple agents/operators may update simultaneously
- Consistency: Prevents race conditions on approval operations
- Compliance: Matches patterns used in other system modules

### 4. Emergency Bypass Mechanism / 緊急繞過機制
**Decision:** EMERGENCY_BYPASSED status with 24-hour review requirement
**Rationale:**
- Safety: Allows immediate response to critical situations
- Governance: Enforces mandatory post-action review
- Traceability: Clear marker for unusual activity

### 5. JSON Serialization / JSON 序列化
**Decision:** Serialize complex values (old_value, new_value) as JSON strings
**Rationale:**
- Flexibility: Supports any JSON-serializable type
- Searchability: String fields are easily indexed/searchable
- Compatibility: Works with external audit systems
