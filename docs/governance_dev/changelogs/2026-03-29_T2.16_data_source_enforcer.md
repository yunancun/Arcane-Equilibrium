## Core Implementation | 核心实现

### 1. DataSourceTag (Immutable Marking)

```python
@dataclass(frozen=True)
class DataSourceTag:
    """Immutable mark applied to external data."""
    tag_id: str
    source_url_or_api: str
    fetched_at_ms: int
    cognitive_level: CognitiveLevel
    confidence: float  # 0.0-1.0
    tagged_by: str
    tag_reason: str
    is_external: bool = True
```

**Key Properties:**
- **Frozen (immutable):** Cannot be modified after creation
- **Unique ID:** `tag_id` for full audit trail
- **Source tracking:** URL/API endpoint + fetch timestamp
- **Cognitive level:** Explicitly assigned (fact/inference/hypothesis)
- **Confidence:** 0.0-1.0 confidence in the marking
- **Reason:** Explanation for why this level was assigned

### 2. DataSourceClassifier (Automatic Classification)

Implements source-based auto-classification rules:

**By Source Type (DataSourceType):**
- Bybit EXCHANGE_REST → FACT (confidence 0.99)
- Bybit EXCHANGE_WS → FACT (confidence 0.99)
- SEARCH_PERPLEXITY → INFERENCE (confidence 0.75)
- SEARCH_WEB → INFERENCE (confidence 0.65)
- LOCAL_OLLAMA → INFERENCE (confidence 0.70)
- LOCAL_INDICATOR → FACT (confidence 0.95)
- EVENT_CALENDAR → INFERENCE (confidence 0.75)
- LEARNING_HISTORY → INFERENCE (confidence 0.80)

**By URL Pattern:**
- `api.bybit.com`, `stream.bybit.com` → FACT
- `perplexity` → INFERENCE
- `duckduckgo`, `google`, `bing` → INFERENCE
- News sites (reuters, bloomberg, etc.) → INFERENCE

### 3. DataSourceEnforcer (Main Enforcement)

**Core Responsibilities:**
1. **Validate & Tag:** `validate_and_tag()` auto-classifies and creates immutable tags
2. **Enforce:** `reject_untagged_data()` blocks unmarked data from decision chain
3. **Track Stats:** `tagged_count`, `rejected_count`, breakdown by level/source
4. **Thread-Safe:** Protected by `threading.Lock` for concurrent access
5. **Audit Trail:** Optional `audit_callback` for all tagging decisions

**Pipeline Integration Points:**
```python
# Exchange data (FACT)
tag, response = enforcer.wrap_exchange_response(
    data_id="price_001",
```

## Governance Alignment | 治理对齐

### DOC-01 §5.10: Cognitive Honesty Principle

> "系统的所有结论必须明确区分事实、推断与假设。不得将推断伪装为事实，不得将假设当作确定性结论。外部数据源（新闻、情绪信号、搜索结果）默认为推断级别，不得等同于交易所返回的事实数据。"

**Implementation:**
- Every external data piece receives an immutable `DataSourceTag`
- Exchange API data automatically classified as FACT
- Search/AI output automatically classified as INFERENCE
- Untagged data is explicitly rejected before entering decision chain
- Audit trail captures all classification decisions

### EX-07 §1: Unmarked Inference Cannot Enter

**Core Principle:** "任何 inference 不得在未标记的情况下进入决策链"

**Implementation:**
- `reject_untagged_data()` enforces this at decision entry points
- Raises `ValueError` if data lacks cognitive level marking
- Strict mode enabled by default
- Statistics track rejection rate for monitoring

## Design Decisions | 设计决策

### 1. Frozen DataSourceTag

**Decision:** Use `@dataclass(frozen=True)` for immutability

**Rationale:**
- Once a tag is created, it cannot be modified (prevents drift)
- Enforces immutable tagging discipline
- Enables safe concurrent read access
- Aligns with DOC-01 §5.10 immutability requirement

### 2. Auto-Classification Before Manual Override

**Decision:** Classifier runs first; explicit overrides accepted

**Rationale:**
- Default behavior enforces cognitive honesty
- Allows flexibility for edge cases
- Maintains audit trail showing classification decision
- Prevents accidental data misclassification

### 3. Rejection vs. Warning Modes

**Decision:** Provide both strict (raise) and permissive (warn) modes

**Rationale:**
- Strict mode (default): enforces hard boundary in production
- Permissive mode: allows testing/debugging without crashes
- Both modes track rejections for monitoring
- Enables gradual rollout with metrics

### 4. Thread-Safe Statistics

**Decision:** Use lock-protected counter updates

**Rationale:**
- Multi-agent system requires concurrent data processing
- Statistics remain consistent under concurrent access
- No performance impact on single-threaded usage

### 5. Optional Audit Callback

**Decision:** Audit callback is optional; exceptions are caught

**Rationale:**
- Audit system is optional (may not be deployed initially)
- Audit failures don't block core enforcement
- Graceful degradation if audit subsystem fails

```python

    raise_on_untagged=True  # Enforce in production

    # Safe to use: data is properly marked

    # Reject: untagged data cannot influence decisions
```

## Performance Characteristics | 性能特征

- **Tag creation:** O(1) with unique ID generation
- **Tag lookup:** O(1) hash table access
- **Rejection check:** O(1) hash table lookup
- **Statistics update:** O(1) counter increment (lock duration ~1µs)
- **Memory:** ~200 bytes per tag (tag_id, URLs, metadata)

**Estimated overhead per data item:**
- CPU: <100µs (includes lock acquisition, hashing)
- Memory: ~250 bytes (DataSourceTag + dict entry)
- For 1,000,000 daily data items: ~250MB peak memory, <100s CPU time
