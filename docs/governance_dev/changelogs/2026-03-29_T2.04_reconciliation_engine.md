### 實現內容 / Implementation Details

**核心組件：**

1. **ReconciliationEngine** — 對賬引擎
   - 訂單對賬：狀態比對、缺失檢測、Bybit API 狀態映射
   - 持倉對賬：數量、方向、缺失（含零倉位過濾）
   - 成交對賬：筆數、數量、加權平均價格偏差
   - 餘額對賬：漂移檢測（絕對容差）
   - 數據新鮮度檢查
   - 線程安全（threading.Lock）

2. **Discrepancy 分類系統**
   - 9 種差異類型：ORDER_STATE/MISSING、POSITION_SIZE/SIDE/MISSING、FILL_COUNT/QUANTITY/PRICE、BALANCE_DRIFT
   - 4 級嚴重度：INFO → WARNING → CRITICAL → FATAL
   - 5 種動作：LOG_ONLY、ALERT、FREEZE_TRADING、MANUAL_REVIEW、AUTO_CORRECT

3. **ReconciliationReport** — 對賬報告
   - 唯一 report_id、時間戳、差異列表
   - 整體結果判定：MATCH / MISMATCH_MINOR / MISMATCH_MAJOR / ERROR
   - is_consistent 屬性、critical_count 統計
   - 完整 to_dict() 序列化

4. **事件觸發機制**
   - FATAL → 自動凍結 + 人工審核
   - CRITICAL + auto_freeze_on_critical → 凍結
   - 超過 max_discrepancies_before_freeze → 凍結
   - incident_callback 接口與外部系統集成

5. **ScheduledReconciler** — 定時對賬運行器
   - 可配置間隔（預設 30 秒）
   - 守護線程、優雅關閉
   - 自動調用 paper_state_fn / remote_state_fn

6. **審計集成**
   - audit_callback 接口（與 AuditPipeline 集成）
   - 每次對賬發送完整審計記錄

**配置項（ReconciliationConfig）：**
- price_tolerance_pct: 0.5%（價格容差）
- qty_tolerance_pct: 0.1%（數量容差）
- balance_tolerance_abs: 1.0 USDT（餘額容差）
- max_data_age_ms: 60s（數據新鮮度）
- auto_freeze_on_critical: True
- max_discrepancies_before_freeze: 5
