### EX-05 §8 Analyst Autonomy Boundaries / Analyst 自主权边界

**Analyst Can Autonomously** (per EX-05 §8.1):
- Record observations (fully automatic)
- Extract lessons from observations (auto-scan enabled)
- Generate hypotheses from patterns (L2+ required)
- Deploy experiments to Paper Trading (no Operator pre-approval)
- Evaluate experiment results and propose verdicts
- Recommend strategy parameter adjustments
- Auto-deploy validated strategy variants to paper
- Track and report learning pipeline metrics

**Analyst Cannot Autonomously** (per EX-05 §8.2):
- Modify live strategy parameters directly
- Promote strategies to live without gate criteria
- Disable or modify P0/P1 risk controls
- Delete or modify historical observations (append-only)
- Override Guardian's risk-based rejections
- Access execution systems directly
- Modify its own evolution level

**Requires Operator Approval** (per EX-05 §8.3):
- First-ever live deployment of a new strategy type
- Changes to Paper→Live gate criteria
- Analyst L5 meta-learning proposals that change pipeline
- Cross-product-family strategy transfers
- Increasing AI compute budget ceiling

## Architecture & Design / 架构与设计

### Thread Safety / 线程安全

```python
# All state mutations protected by threading.Lock()
self._lock = threading.Lock()
self._state = TierState()

# Properties are thread-safe reads
@property
def current_tier(self) -> LearningTier:
    with self._lock:
        return self._state.current_tier

# Metrics updates are atomic
def update_metrics(self, observation_count: int | None = None, ...) -> None:
    with self._lock:
        if observation_count is not None:
            self._state.observation_count = observation_count
```

### Promotion Logic / 晋升逻辑

Promotions follow structured eligibility gates:

1. **Check Eligibility**: Verify gate conditions (metrics thresholds, time requirements)
2. **Build Audit Record**: Capture state snapshot BEFORE updating (for correct previous_tier)
3. **Update State**: Advance tier, increment version, track promotion
4. **Emit Audit Callback**: Send promotion record to audit system

```python
def promote_tier(self, target_tier: LearningTier, ...) -> bool:
    with self._lock:
        # Check eligibility gates
        eligible, reasons = self._check_eligibility_unsafe(target_tier)

        if not eligible and initiator != "Operator":
            return False

        # Build audit record BEFORE state update
        record = _build_promotion_record(
            self._state,  # Current state before update
            target_tier,
            event,
            initiator_enum,
            ...
        )

        # Update state
        self._state.current_tier = target_tier
        ...

        # Emit audit callback
        if self._audit_callback:
            self._audit_callback(record)

        return True
```

### Serialization / 序列化

Full state can be exported and imported for persistence:

```python
def export_state(self) -> dict[str, Any]:
    """Export tier state for persistence / storage"""
    with self._lock:
        return {
            "current_tier": self._state.current_tier.name,
            "observation_count": self._state.observation_count,
            "confirmed_patterns": self._state.confirmed_patterns,
            ...
            "promotions": self._state.promotions,
            "version": self._state.version,
        }

def import_state(self, exported: dict[str, Any]) -> bool:
    """Import tier state from exported data"""
    try:
        with self._lock:
            tier_name = exported.get("current_tier", "L1")
            self._state.current_tier = LearningTier[tier_name]
            ...
        return True
    except (KeyError, ValueError, TypeError) as e:
        logger.error(f"Failed to import: {e}")
        return False
```

## Example Usage / 使用示例

### Basic Flow / 基本流程

```python
from app.learning_tier_gate import LearningTierGate, LearningTier

# Create gate with optional audit callback
audit_log = []
def audit_callback(record):
    audit_log.append(record)
    print(f"Promotion: {record['previous_tier']} → {record['next_tier']}")

gate = LearningTierGate(audit_callback=audit_callback)

# L1 is automatic at startup
assert gate.current_tier == LearningTier.L1
assert gate.can_record_observations() is True
assert gate.can_discover_patterns() is False

# L1 → L2: When 500+ observations and win_rate > 20%
gate.update_metrics(observation_count=500, win_rate=0.21)
eligible, reasons = gate.check_tier_eligibility(LearningTier.L2)
assert eligible is True

gate.promote_tier(LearningTier.L2, reason="Pattern discovery unlocked")
assert gate.current_tier == LearningTier.L2
assert gate.can_discover_patterns() is True

# L2 → L3: When L2 running 2+ weeks + 3+ confirmed patterns
gate._l2_start_time_ms = int(time.time() * 1000) - (15 * 24 * 60 * 60 * 1000)
gate.update_metrics(confirmed_patterns=3)
gate.promote_tier(LearningTier.L3)
assert gate.current_tier == LearningTier.L3

# L3 → L4: When 3+ validated hypotheses + positive ROI
gate.update_metrics(validated_hypotheses=3, experiment_roi=0.1)
gate.promote_tier(LearningTier.L4)
assert gate.current_tier == LearningTier.L4

# L4 → L5: Requires 6+ months + sustained positive + Operator approval
gate.update_operational_time(180)
gate.update_metrics(sustained_positive_live_days=30)
gate.promote_tier(
    LearningTier.L5,
    initiator="Operator",
    approved_by="nancun@openclaw.dev"
)
assert gate.current_tier == LearningTier.L5

# Verify audit trail
assert len(audit_log) == 4
print(f"Full evolution tracked with {len(audit_log)} promotion events")
```
