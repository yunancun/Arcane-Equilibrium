## 清理原因 / Reason for Cleanup

### 背景 / Background

在 2026-03-28 Session 6 的工作中，发现 `bybit_private_*_check.py.orig` 文件缺失导致 cron 任务每 5 分钟崩溃。
作为应急修复，创建了 3 个 `.orig` stub 文件，只在 API Key 未配置时返回 `not_configured` 状态。

During Session 6 (2026-03-28), missing `.orig` files caused cron tasks to crash every 5 minutes.
As an emergency fix, 3 `.orig` stub files were created that only return `not_configured` status when API keys are missing.

### 为何可以安全删除 / Why Safe to Remove

1. **临时补丁性质 (Temporary Patch Nature):**
   - 这些 `.orig` 文件从未包含真实业务逻辑，只是空壳
   - These `.orig` files never contained real business logic, just shell stubs

2. **不被代码引用 (Not Referenced in Code):**
   - 虽然 `io_and_persistence/*.py` 中引用了 `.orig` 的路径，但这些引用应该指向真实实现
   - While `io_and_persistence/*.py` files reference `.orig` paths, these references should point to real implementations
   - 删除后，系统应切换到正式的 API 实现方案
   - After removal, system should switch to proper API implementation approach

3. **代码清洁标准 (Code Cleanliness Standard):**
   - 根据 T1.4 Code Cleanliness findings，这类临时文件应被识别为 GAP-L4 技术债务
   - Per T1.4 Code Cleanliness findings, temporary files like these should be identified as GAP-L4 technical debt
   - 清理后提高代码库的整洁度和可维护性
   - Cleanup improves codebase cleanliness and maintainability
