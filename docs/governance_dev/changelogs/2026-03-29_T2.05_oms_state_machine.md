### 實現內容 / Implementation Details

**核心變更：7 → 11 狀態生命週期**

新增 4 個關鍵狀態：

1. **PENDING** — 等待授權審核（與 AuthorizationStateMachine 協調）
2. **APPROVED** — 授權通過，等待提交執行
3. **RECONCILING** — 關鍵安全閘門：paper↔demo 持倉一致性驗證
4. **COMPLETED** — 通過對賬驗證的最終狀態

**完整狀態轉換圖：**
```
CREATED → PENDING → APPROVED → SUBMITTED → WORKING → FILLED → RECONCILING → COMPLETED
                  ↘ REJECTED   ↘ CANCELED   ↘ REJECTED  ↘ PARTIALLY_FILLED ↘ REJECTED
                                              ↘ CANCELED   ↘ CANCELED
                                                            ↘ FILLED
```

**組件：**

1. **OrderState** — 11 個狀態（Enum）
2. **OrderEvent** — 13 個事件類型
3. **OrderInitiator** — 7 種發起者（Operator、AIAgent、System、ExecutionVenue、AuthorizationSM、ReconciliationEngine、RiskGovernor）
4. **OMS_TRANSITION_RULES** — 16 條有效轉換規則
5. **FORBIDDEN_TRANSITIONS** — 12 條禁止轉換
   - 不可跳過授權（CREATED → SUBMITTED/WORKING/APPROVED）
   - 不可跳過對賬（FILLED → COMPLETED）
   - 終態不可退出
   - 執行中不可倒退
6. **OMSOrder** — 擴展訂單物件（含 approved_by、reconciliation_result）
7. **OMSStateMachine** — 狀態機引擎
   - 線程安全、守衛條件、審計回調、持久化
   - 便捷方法：submit_for_approval、approve、reject、send_to_venue、acknowledge、partial_fill、fill、cancel、begin_reconciliation、reconciliation_pass、reconciliation_fail
   - 查詢：get_active_orders、get_reconciling_orders、get_pending_approval、status_summary
   - Paper Engine 雙向狀態映射（map_from_paper_state / map_to_paper_state）
