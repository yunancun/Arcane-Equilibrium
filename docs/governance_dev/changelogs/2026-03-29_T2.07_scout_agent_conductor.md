### 實現內容 / Implementation Details

**1. 結構化 Agent 角色（EX-06 §1）**
- AgentRole 枚舉：Scout、Strategist、Guardian、Analyst、Executor + Conductor
- AgentState 生命週期：INITIALIZING → RUNNING → DEGRADED → PAUSED → STOPPED
- ResourcePriority 計算優先級：Guardian > Scout(urgent) > Strategist > Analyst > Scout(routine)

**2. 結構化消息協議（EX-06 §8）**
- 11 種消息類型：intel_object、event_alert、trade_intent、risk_verdict、approved_intent、execution_report、round_trip_complete、pattern_insight、risk_pattern、strategy_proposal、system_directive
- AgentMessage 基礎對象：sender/receiver/timestamp/type/priority
- IntelObject：情報對象（source、freshness、data_quality、sentiment、relevance_score）
- EventAlert：事件告警（event_type、severity、lead_time_hours）
- TradeIntent：交易意圖（symbol、strategy、direction、size、confidence、thesis）
- RiskVerdict：風控裁決（approved / rejected / modified）

**3. 消息總線 MessageBus（EX-06 §8.1）**
- 14 條有效路由（基於 TABLE 3）
- 路由驗證：只允許規定的 sender→receiver 組合
- 訂閱/發布模式
- 審計回調集成
- 線程安全（Lock）

**4. Scout Agent（EX-06 §3）**
- produce_intel()：生成情報對象，自動路由至 Strategist
- produce_event_alert()：生成事件告警，自動路由至 Guardian
- 數據質量標記（§3.4）：fact / inference / hypothesis
- 相關性閾值過濾（低相關性情報不發送）
- 高嚴重度告警自動提升優先級
- 約束遵守：不生成交易信號、不修改風控參數、不直接調用交易所 API

**5. 衝突仲裁（EX-06 §9）**
- 5 種衝突場景裁決
- 核心原則：Guardian 的風控結論永遠優先於 Strategist 的交易意圖
- open_vs_tighten → Guardian 獲勝
- scout_bearish + strategist_bullish → Guardian 介入
- analyst_vs_strategist → Strategist 自主權
- executor_anomaly → Guardian 可觸發 CIRCUIT_BREAKER
- resource_contention → Conductor 按優先級分配

**6. Conductor 編排器（EX-06 §2）**
- Agent 生命週期管理：register、set_state、heartbeat
- 任務分發：根據事件類型路由至相關 Agent
- 資源分配：AI 計算預算按優先級分配（$2/day ceiling）
- 系統指令廣播：system_directive 至全部/指定 Agent
- 交易意圖管線：Strategist → Guardian 審查 → Executor
- Conductor 約束遵守：不直接下單、不覆蓋 Guardian 否決、不修改 P0/P1
