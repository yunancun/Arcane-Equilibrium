### 實現內容 / Implementation Details

**核心組件：**

1. **AuditFileWriter** — 追加寫 JSON Lines 審計文件
   - 線程安全寫入
   - 每次寫入立即刷盤（fsync）
   - 日期自動輪轉
   - 文件大小輪轉（預設 50MB）
   - 記錄數輪轉（預設 50 萬條）
   - 批量寫入支持

2. **AuditFileReader** — 審計記錄查詢
   - 按時間範圍查詢
   - 按來源（source）過濾
   - 按事件類型（event_type）過濾
   - 關鍵字搜索
   - 損壞行自動跳過
   - 按日期統計
   - 每日摘要生成（EX-01 §8.2）

3. **AuditPipeline** — 中央審計管道
   - `make_callback(source)` — 生成狀態機回調
   - 內存緩衝（最近 10,000 條，快速查詢）
   - 訂閱者通知機制
   - 與 AuthorizationStateMachine / RiskGovernorStateMachine 集成

**安全不變量：**
- 追加寫模式：記錄寫入後永不刪除或修改
- 每次寫入立即 fsync：崩潰不丟數據
- 文件輪轉創建新文件但不刪除舊文件
- 損壞行讀取時跳過，寫入時不靜默丟棄

### 集成方式 / Integration

```python
# 使用示例 / Usage example
pipeline = AuditPipeline(AuditPersistenceConfig(base_dir="data/audit"))
auth_sm = AuthorizationStateMachine(audit_callback=pipeline.make_callback("authorization_sm"))
risk_gov = RiskGovernorStateMachine(audit_callback=pipeline.make_callback("risk_governor"))
```
