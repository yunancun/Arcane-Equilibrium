## 4. 技術設計概要 / Technical Design Overview

### 4.1 governance.js — API 調用層

```
governance.js
├── govApi                    // 共享 fetch wrapper
│   ├── govGetStatus()        // GET /governance/status
│   ├── govGetAuthStatus()    // GET /governance/auth/status
│   ├── govPostApprove(note)  // POST /governance/auth/approve
│   ├── govGetRiskLevel()     // GET /governance/risk/level
│   ├── govPostOverride(lvl, reason)  // POST /governance/risk/override
│   ├── govPostReconcile(paper, demo, reason)  // POST /governance/reconcile
│   ├── govGetLeases()        // GET /governance/leases
│   └── govPostHealthCheck()  // POST /governance/health-check
├── govRender                 // 共享渲染工具
│   ├── govAuthBadge(state)   // → ocChip HTML for auth state
│   ├── govRiskBadge(level)   // → ocChip HTML for risk level
│   ├── govModeBadge(mode)    // → ocChip HTML for gov mode
│   ├── govExpiryCountdown(ms)  // → "2h 15m" or "EXPIRED"
│   └── govConsistencyIcon(bool) // → "✓ Consistent" or "⚠ Diverged"
└── govConst                  // 常量
    ├── AUTH_STATES           // { ACTIVE: 'good', RESTRICTED: 'warn', FROZEN: 'bad', ... }
    ├── RISK_LEVELS           // { 0: 'NORMAL', 1: 'CAUTIOUS', ... }
    └── GOV_MODES             // { NORMAL: 'good', RESTRICTED: 'warn', ... }
```

### 4.5 Modal 設計（共用模式）

三個操作 modal 共用相同結構：

| Modal | 觸發 | 輸入欄位 | API |
|-------|------|----------|-----|
| Approve Auth | [APPROVE] 按鈕 | approval_note (textarea, 1-500 chars) | POST /auth/approve |
| Override Risk | [OVERRIDE] 按鈕 | target_level (dropdown), reason (textarea) | POST /risk/override |
| Trigger Recon | [RECONCILE] 按鈕 | reason (textarea) | POST /reconcile |

Modal 模板：
```html
<div class="oc-modal" id="modal-{name}" style="display:none">
  <div class="oc-modal-content">
    <h3>{title}</h3>
    <div class="oc-modal-body">{fields}</div>
    <div class="oc-modal-footer">
      <button class="oc-btn" onclick="submit{Name}()">Submit / 提交</button>
      <button class="oc-btn oc-btn-secondary" onclick="closeModal('{name}')">Cancel / 取消</button>
    </div>
  </div>
</div>
```

## 5. 開發規範 / Development Standards

### 5.1 UI 規範

- **雙語：** 所有標籤使用 `英文 / 繁體中文` 格式
- **配色：** 沿用 common.js 的 `--green`, `--red`, `--yellow`, `--blue` CSS 變數
- **Badge 映射：**
  - Auth: ACTIVE → `good`(綠), RESTRICTED → `warn`(黃), FROZEN → `bad`(紅), PENDING → `info`(藍)
  - Risk: 0-1 → `good`, 2-3 → `warn`, 4-5 → `bad`
  - Mode: NORMAL → `good`, RESTRICTED → `warn`, FROZEN → `bad`, MANUAL_REVIEW → `info`
- **Explainers：** 每個 Card 使用 `ocExplain()` 提供中英雙語說明

### 5.2 API 規範

- 全部使用 `ocApi()` / `ocPost()` 共享 wrapper
- 錯誤處理使用 `ocToast(msg, 'error')`
- 操作成功使用 `ocToast(msg, 'success')`
- Auto-refresh 15 秒間隔

### 5.3 安全規範

- 所有用戶輸入使用 `ocEsc()` 防止 XSS
- Approval/Override modal 輸入限制 1-500 字元
- Risk Override 只允許降級（`target_level < current_level`），UI 端過濾
- 操作前顯示確認提示

## 8. 風險評估 / Risk Assessment

| 風險 | 可能性 | 影響 | 緩解 |
|------|--------|------|------|
| GovernanceHub 未初始化 | Medium | API 返回 503 | UI 顯示 "Governance not available" 提示，不崩潰 |
| SM 未注入（測試環境） | Medium | 部分數據為 null | 所有 render 函數處理 null/undefined |
| Approval 時 Auth 非 PENDING | Low | API 返回 400 | Modal 提交後刷新狀態，toast 顯示錯誤 |
| 瀏覽器兼容性 | Low | CSS 差異 | 使用已驗證的 common.js 共享組件 |
