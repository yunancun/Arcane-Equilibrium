  shadow_decision_builder.py → lease schema + revoke + expiry

[Phase 2 本地策略]

## 3. 數據流分析 / Data Flow Analysis

### 3.1 主交易管線（Paper Trading 模式）

```
1. bybit_public_ws_listener
   → 接收 Bybit WS 推送（ticker/trade/kline）
   → 發送到 market_data_dispatcher

2. market_data_dispatcher
   → fan-out 到各消費者：
     - kline_manager（聚合 K線）
     - pipeline_bridge.on_tick_result()（tick 路徑事件）
     - risk_manager（tick 級風控檢查）

3. kline_manager
   → tick 聚合為 1m/5m/15m/1h/4h K線
   → 觸發 indicator_engine 計算
   → 觸發 signal_generator 生成信號

4. strategy_orchestrator
   → 協調 kline + indicator + signal + strategies
   → 各策略生成 OrderIntent

5. pipeline_bridge（核心橋接）
   → Intent → Paper Order 提交
   → tick 路徑：風控自動平倉 / 時間止損 / 軟止損
   → 成交回調 → E1 觀察記錄
   → _emit_round_trip()（記錄每輪交易結果）
   → stop_manager 追蹤（ATR 動態止損）

6. paper_trading_engine
   → 訂單狀態管理（7 狀態機）
   → 成交模擬（market=含滑點, limit=價格穿越）
   → PnL 計算（realized + unrealized - fees - ai_cost）

7. risk_manager
   → P0 硬止損 / P1 全局限制 / P2 Agent 自適應
   → regime 感知乘數（volatile=1.5×, squeeze=0.6×）
   → check_positions_on_tick()（逐 tick 檢查）

8. bybit_demo_sync（可選雙重執行）
   → Paper Engine 訂單 → 同步到 Bybit sandbox
```

### 3.2 掃描→部署管線

```
market_scanner（每 5 分鐘）
  → 掃描 650+ 永續合約符號
  → 評分 0-100（funding_arb/grid/trend/breakout/reversion）
  → 過濾：volume > $5M，24h change < 40%

strategy_auto_deployer
  → 接收掃描結果
  → 選擇最佳策略類型
  → 部署到 strategy_orchestrator
  → 風險控制：1% per trade, max 5 symbols, 10 連虧暫停

strategy_orchestrator
  → 管理所有活躍策略實例
  → Thread-safe 協調
  → 分發 tick/kline 到各策略
```

### 3.3 學習管線

```
pipeline_bridge._emit_round_trip()
  → 每輪交易完成後寫入 observation
  → 包含：entry/exit 價格、PnL、regime、持倉時間

learning 自動掃描（路由觸發）
  → observations → lessons → hypotheses → experiments
  → review_queue → Operator 審批
```

## 4. 關鍵設計模式 / Key Design Patterns

### 4.1 門控鏈（Gate Chain）
系統採用嚴格的門控鏈設計，每個門必須通過才能進入下一步：
- H0: freshness → health → eligibility → risk_envelope → final_audit
- H1-H5: thought_gate → budget → model_router → governor → cost_logging
- I: Decision Lease（shadow-only，帶時效 + 可撤銷）

**原則：任何門失敗 = fail-closed，不猜測、不跳過。**

### 4.2 影子模式（Shadow Mode）
所有涉及執行權限的模組都以「影子模式」運行：
- Decision Lease 只生成不發射
- Paper Trading 模擬所有訂單
- Bybit Demo 可選同步但不影響真實資金
- 硬邊界：`system_mode=read_only`, `execution_state=disabled`, `execution_authority=not_granted`

### 4.3 成本感知（Cost-Aware）
每個決策路徑都追蹤成本：
- 手續費：taker 0.055%, maker 0.02%
- AI 成本：per-call 記賬，納入 net_pnl
- 注意力稅：持倉期間 AI 監控成本 → cost_edge_ratio 超閾值建議平倉
- 滑點：market order 模擬 0.05% 滑點

### 4.4 Regime 感知
信號與風控都根據市場 regime 調整：
- `trending`: 標準參數
- `volatile`: 止損 1.5×、止盈 1.5×、時間止損 1.5×
- `squeeze`: 止損 0.6×、止盈 0.7×、時間止損 0.3×（~14h vs 默認 48h）
- `unknown`: 禁止入場（冷啟動保護）

### 4.5 止損隱身（Stealth Stops）
永遠不在交易所放 stop order：
- 本地 tick() 觸發所有止損
- 防止交易所/做市商看到止損位並獵殺
- ATR 動態計算 + 隨機偏移

## 5. API 架構 / API Architecture

### 5.1 路由分組（113 條）

| 分組 | 路由數 | 前綴 | 職責 |

| Paper | ~24 | `/paper/` | Paper Trading 全生命週期 |

### 5.2 安全加固
- API Token：自動生成 + `hmac.compare_digest()`（常數時間比較）
- 速率限制：slowapi 120/min/IP
- CORS：可配置 origins
- 文本長度限制：title≤200, detail≤2000, reason≤500
- 幂等緩存：24h TTL + 500 條上限
- 狀態文件權限：`chmod 0o600`
