### 2.2 Atomic patch commit 內容（單一 commit）

cohort 拍板後，**同 commit** atomic patch `risk_config_demo.toml`：


**Acceptance**（per ARCH-04 §3.1 Rust schema enforcement）：
- Rust schema validate `shadow_mode = false` 配 `canary_stage = 0` → fail-closed reject + log（per AMD-03 §4.4）

- `stage_entered_at_ms` 與 `observation_period_ms` 必正整數，與 stage 規格對齊（Stage 1 = 7d = 604800000 ms）

## 4. Risk Acceptance

### 4.1 為何 B-later 不會「永遠 later」

風險：B-later 變成 indefinite postpone。Mitigation：

1. **Sprint N+1 W3 dispatch dependency**：W-AUDIT-9 IMPL land + cohort 拍板
   是 W-AUDIT-9 standalone milestone exit criteria（per TODO v19 §5.1
   invariant 4），不可跳；Sprint N+1 sign-off 必驗
2. **PA 拍板 cohort 屬 Sprint N+1 W3 dispatch SOP**（per §2.1 步驟 1）；
   不在拍板 = N+1 sign-off blocker
3. **TODO v19 §0 Sprint Banner N+1 milestone「Stage 1 cohort 7d 觀察」**
   explicit 寫入；逃 = N+1 milestone fail

### 4.2 為何 Sprint N+0 不修

CC 提的 alternative B（Sprint N+0 內就修）：

> 同 commit atomic patch `risk_config_demo.toml` shadow_mode = false +
> canary_stage = 1 + cohort 字段；W-AUDIT-9 IMPL land 後即 Stage 1 active

棄理由：
1. **W-AUDIT-9 IMPL 還沒 land 完整 7 sub-task**（Sprint N+0 W1-W2 才剛開始派發 per
   TODO v19 §6 Day 0-3）；TOML patch 在 IMPL 完成前等同盲推
2. **Stage 1 cohort 選擇不是技術 task 而是 operator 業務拍板**（per
   AMD-03 §2.4「cohort 由 operator 在 Settings tab 顯式選擇」）；W-AUDIT-9 T5
   GUI surface 沒 land 前 operator 沒有合適界面
3. **Sprint N+0 capacity 緊**（5 active + 1 stand-by E1，per TODO v19 §0
   Sprint Banner）；推 cohort decision 至 N+0 = capacity 撞牆風險增
