## 3. ADR-0026 v2 Validation Tighten（per reviewer critique #4）

### 3.1 三件套替換 manual review

Track A Direct Exploit 仍 bypass CPCV + DSR（因 event-driven + 無歷史樣本），但替換 gate 從 v1「manual backtest review」改為三件套：

**1. Pre-Registration**（Open Science Framework style）：
- `learning.hypothesis_preregistration` 表（V102 內含）lock 所有決策閾值
- Modify = 新 row + chain-link 既有 row（audit trail）
- 防 p-hacking + garden-of-forking-paths

**2. Event-Study**（Brown & Warner 1985 / MacKinlay 1997）：
- Estimation window 30+ days + Event window per spec + Post-event window
- Cumulative Abnormal Return (CAR) + t-stat ≥ 2.0
- Wilcoxon signed-rank robustness check p < 0.05
- n_events ≥ pre-registered min

**3. Replay Match**（既有 Stage 0R）：
- Replay 30d historical data
- Match rate ≥ 80%
- CAR consistency ±20% vs event-study

**Gate sequence**: DRAFT → PREREGISTERED → REPLAY_OK → EVENT_STUDY_OK → PA+QC 5d operational review (only veto on operations, not statistics) → STAGE_0_SHADOW → ... → STAGE_1_DEMO_CANARY.

### 3.2 統計嚴格性

三件套合計提供：
- **i.i.d. 假設不需要**（event-study 直接做 abnormal return tests）
- **Multiple-testing control**（pre-registration lock 閾值，防 p-hacking）
- **Reproducibility**（replay match rate 確保策略行為可重現）
- **Operational sanity**（5d window review 看 fee/latency/exchange edge）

→ 統計嚴格性 ≥ CPCV+DSR 在 event-driven data 上的水準。

### 4.1 Track A W8 kill ladder（v4.1）

| Phase | Threshold | Action |
|---|---|---|
| W2 | LCS event-study CAR t-stat < 1.5 OR pre-registration miss > 2σ | **KILL LCS，all-in NLE shadow** |
