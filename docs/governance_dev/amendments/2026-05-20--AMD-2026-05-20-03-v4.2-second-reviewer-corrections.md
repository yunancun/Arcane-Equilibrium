## 3. ADR-0026 v3 修正

### 3.1 Pre-registration schema +8 fields

v2 7 fields → v3 15 fields（per reviewer #4.2）：

| 既有 v2 | 新增 v3 |
|---|---|
| expected_alpha_bps_min/max | code_hash (git SHA) |
| expected_n_events_min | config_hash (risk_config + params hash) |
| expected_sharpe_min | trigger_rule (immutable JSONB) |
| expected_win_rate_min | side_rule ('long_only'/'short_only'/'both') |
| decision_alpha | expected_max_drawdown_pct |
| estimation_window_days | expected_holding_period_sec |
| event_window_seconds | cost_assumption (fee + slippage snapshot) |
|  | dedup_rule ('per_event'/'per_minute'/etc) |
|  | variance_estimator (newey_west/garch_1_1/bootstrap/realized_variance) |
|  | data_window_start_ts + data_window_end_ts |
|  | immutable_trigger_hash (= hash of above) |

`immutable_trigger_hash` 防 p-hacking：任何 modify = supersede 為新 row
+ chain-link audit trail。

### 3.2 Variance estimator 強制聲明

Crypto vol clustering 違反 OLS i.i.d. 假設。v3 強制 prereg `variance_estimator`
CHECK constraint：

- `newey_west` (Newey & West 1987 HAC)
- `garch_1_1` (Bollerslev 1986)
- `bootstrap` (block bootstrap)
- `realized_variance` (Andersen et al 2003)

t-stat 計算用 HAC-corrected SE，**禁用 default OLS**。

### 3.3 Replay match function DEFER 到 Phase 1.5

reviewer #4.3 指出 replay match function 尚未實作。v3 處理：

```
Phase 1 (N+1-N+3) gate sequence:
  DRAFT → PREREGISTERED → EVENT_STUDY_OK → STAGE_0_SHADOW → ...

Phase 1.5 (N+4+ when replay match function lands):
  DRAFT → PREREGISTERED → EVENT_STUDY_OK → REPLAY_MATCH_OK → STAGE_0_SHADOW → ...
```

Phase 1 用 2-instrument gate（prereg + event-study）；Phase 1.5 升 3-instrument
gate。replay match function land 後 mandatory。

### 3.4 LCS strategy thesis v3

per reviewer #5: 純速度戰必輸；改 **isolated cluster + book recovery +
maker entry**：

```
Trigger:
  isolated_liq_cluster (|Σ liq_value_60s| > $1M tier_A / > $300k tier_B)
  AND no_follow_on_liq_within_30s
  AND spread <= 50 bps
  AND orderbook_bid_depth_5lvl >= 80% pre-cluster level

Entry:
  PostOnly maker on opposite side
  Offset 2-5 bps inside best bid/ask
  Timeout 60s

Hold 30-180s:
  TP: +25 bps OR retracement to pre-cluster mid
  SL: -15 bps OR new follow-on liq detected
```

Win rate 預期提升 55-65% → 60-72%；alpha per trade 30-60 bps → 40-80 bps
（含 maker rebate）。
