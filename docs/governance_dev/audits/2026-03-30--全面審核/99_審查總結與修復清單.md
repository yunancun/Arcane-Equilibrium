## 一、問題總覽（按優先級）

### 🔴 P0-A1：MA_Cross 信號永遠被 regime 過濾器阻擋

**影響範圍：** 7 個 MA_Crossover 策略全部失效（PRLUSDT/BIRBUSDT/BSBUSDT/ONTUSDT/BTRUSDT/4USDT/ARIAUSDT）

**根因：**
1. `MACrossoverRule.evaluate()` 創建 `Signal` 時不傳 `metadata` 參數
2. `Signal.__init__` 中 `self.metadata = metadata or {}` → 空字典 `{}`
3. Orchestrator 的注入條件 `if signal.metadata` → `{}` 為 **falsy**，`_regime` 注入被跳過
4. Session 7（2026-03-28）將 `"unknown"` 加入禁止列表 → 策略永遠返回 "unknown" → 永遠不生成 intent

**程式碼位置：**
- `local_model_tools/signal_generator.py:369–376`（Signal 創建無 metadata）
- `app/pipeline_bridge.py:259`（orchestrator._on_signal，falsy 檢查）
- `local_model_tools/strategies/ma_crossover.py:110–116`（regime 過濾）

**修復方案（選擇其一）：**
```python
# 選項 A（建議）：移除 truthiness 依賴，改為 None 檢查
if hasattr(signal, "metadata") and signal.metadata is not None:
    if self._current_regime != "unknown":
        signal.metadata["_regime"] = self._current_regime

# 選項 B：MACrossoverRule 創建 Signal 時傳入佔位符
return Signal(..., metadata={"_regime": "unknown"})
# 同時 orchestrator 的條件改為 `signal.metadata is not None`
```

### 🔴 P0-A2：FundingRate_Arb 策略只接收 BTC/ETH funding rate

**影響範圍：** 3 個 FundingRate_Arb 策略全部錯誤評估（CFGUSDT 107.9bps / ENJUSDT 166.7bps / SENTUSDT 94.0bps）

**根因：**
```python
# pipeline_bridge.py:1432
for symbol in ["BTCUSDT", "ETHUSDT"]:   # 只有 BTC 和 ETH！
    for strategy in self._orch._strategies.values():
        if hasattr(strategy, "evaluate_funding_opportunity"):
            strategy.evaluate_funding_opportunity(funding_rate=btc_or_eth_rate)  # 錯誤！
```
CFGUSDT 策略收到的是 BTCUSDT 的 funding rate（接近 0），永遠不觸發閾值。

**修復方案：**
```python
# 為每個 FundingRate_Arb 策略獲取其自己 symbol 的 rate
for strategy in self._orch._strategies.values():
    if hasattr(strategy, "evaluate_funding_opportunity"):
        symbol = getattr(strategy, '_symbol', None) or getattr(strategy, 'symbol', None)
        if symbol:
            rate = self._fetch_single_funding_rate(symbol)
            if rate is not None:
                strategy.evaluate_funding_opportunity(rate, ...)
```

### 🟡 P1-C1：Session drawdown gate 無強制執行機制

**影響範圍：** 15% 最大 session 回撤 circuit breaker 失效

**根因：**
```python
# risk_manager.py:977-985
if drawdown_pct >= self._config.max_session_drawdown_pct:
    if not sess.get("session_halted"):
        logger.warning("... → halt recommended")
        # ← 只記錄警告，不設置 session_halted = True，不阻止新訂單
```

**修復方案：**
```python
if drawdown_pct >= self._config.max_session_drawdown_pct:
    if not sess.get("session_halted"):
        sess["session_halted"] = True
        logger.warning("SESSION HALTED: drawdown %.1f%% >= %.1f%%", ...)
```
同時在 `paper_trading_engine.submit_order()` 中添加：
```python
if state.get("session", {}).get("session_halted"):
    return {"error": "session_halted", "message": "drawdown circuit breaker triggered"}
```
