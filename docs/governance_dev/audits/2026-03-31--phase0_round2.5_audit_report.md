## 二、P0 阻斷問題（必須立即修復）

### P0-1: MessageBus.subscribe() 參數不匹配 → AnalystAgent 靜默失敗

**位置**: `phase2_strategy_routes.py:475-477`

```python
# 問題代碼（傳入 3 個參數，subscribe() 只接受 2 個）
MESSAGE_BUS.subscribe(_AR.ANALYST, _MT.ROUND_TRIP_COMPLETE, ANALYST_AGENT.on_message)
MESSAGE_BUS.subscribe(_AR.ANALYST, _MT.EXECUTION_REPORT, ANALYST_AGENT.on_message)
MESSAGE_BUS.subscribe(_AR.ANALYST, _MT.SYSTEM_DIRECTIVE, ANALYST_AGENT.on_message)
```

**subscribe() 簽名** (`multi_agent_framework.py:311`):
```python
def subscribe(self, role: AgentRole, callback: Callable) -> None:
```

**影響**:
- 這 3 行 subscribe 調用觸發 TypeError
- 被 `except (ImportError, Exception) as e:` (line 479) 捕獲
- `ANALYST_AGENT` 被設為 None (line 480)
- **AnalystAgent 完全無法運行**：學習回路斷開，L2 指標更新停止

**修復方案**: 移除 MessageType 參數，改為 2 參數調用（與 line 167/211/244/503 一致）

```python
MESSAGE_BUS.subscribe(_AR.ANALYST, ANALYST_AGENT.on_message)
# MessageType 過濾由 Agent.on_message() 內部的 message_type 判斷處理
```

### P0-2: layer2_engine.py L1 triage 自由文本解析邏輯 bug

**位置**: `layer2_engine.py:292`

```python
worth = "true" in text_lower or "yes" in text_lower or "worth" in text_lower
```

**問題**: "not worth investigating" 包含 "worth" → 返回 True（應為 False）

**影響**: L1 triage 可能錯誤放行不值得調查的信號

**修復方案**: 使用更精確的模式匹配

```python
worth = ("true" in text_lower or "yes" in text_lower) and "not" not in text_lower.split("worth")[0][-10:]
# 或更簡潔：直接檢查 "not worth" 否定模式
```
