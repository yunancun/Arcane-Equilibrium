九、今晚过程中的主要问题与修复记录
============================================================

------------------------------------------------------------
9.1 多次出现 shell 粘贴不完整 / 终端输出被误当命令
------------------------------------------------------------

表现包括：
- 命令被截断
- preview/check 脚本末尾变成奇怪的 “PYint(...)”
- 有些路径或文件名在粘贴时被截坏

修复思路：
- 不再迷信一次贴很长命令块；
- 认识到 Ubuntu shell / 终端 / 剪贴板在长粘贴场景下，确实可能出现截断或混贴；
- 后续更适合把大命令拆成多段，一条一条粘。

这不是业务逻辑问题，但会非常干扰施工效率。

------------------------------------------------------------
9.2 某些脚本实际文件名与预期不一致
------------------------------------------------------------

例如：
- bybit_transition_engine_summary_builder.py 并不存在，
  实际文件是 bybit_transition_engine_summary.py
- 早期一次 compile 命令里还找不到 bybit_simulator_adapter_contract_builder.py

修复思路：
- 不强行假设 builder 命名统一；
- 先 ls / preview 确认真实文件名；
- 再按实际存在文件修正检查和注释。

------------------------------------------------------------
9.3 contract checker 缺失或路径写错
------------------------------------------------------------

例如：
- 早前 handoff contract checker 缺失，需要补建；
- 某些 compile 命令因为路径或文件名拼坏而报错。

修复思路：
- 按“主文件 + contract checker + 如有需要则 summary/handoff/final audit”成套补齐；
- 每层完成后都做一次 existence check。

------------------------------------------------------------
9.4 patch helper 出现 SyntaxError
------------------------------------------------------------

表现：
- patch 脚本里全角中文逗号触发 SyntaxError

修复思路：
- 不把 patch helper 的失败，误判为目标脚本损坏；
- 先单独 compile 现有目标脚本确认无损；
- 再用更稳的方法做头注释归位修正。
