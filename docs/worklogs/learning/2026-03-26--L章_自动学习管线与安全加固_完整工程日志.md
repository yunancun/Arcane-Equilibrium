## 一、背景与动机

L 章的 API/GUI 存储层（手动录入 + 查询 + Net PnL）已在此前完成（commit `266ab29`，30 个测试全通过）。但用户明确指出：

> "GUI 手动输入只是备用，Agent 本身应该能自动学习。"

核心约束：
- **Operator 没有很多金融知识** → 审核内容必须简单、易懂
- 每个审核项必须有**明确的后果分析**（批准/拒绝/搁置分别会怎样）
- 提供**询问 AI** 的选项，自动推荐 AI 层级，预生成简短问题
- 参考 I 章 Decision Lease 的紧凑模式来生成 AI 问题

本日志覆盖两个阶段：自动学习管线实现 + 全盘安全审计与加固。

## 二、自动学习管线设计

### 核心概念：审核包 (Review Packet)

灵感来源于 I8 Manual Approval Packet，但面向非金融专业 Operator 大幅简化。

每个审核包包含：
1. **简要说明** — 1-2 句大白话，说清发生了什么
2. **为什么重要** — 后果分析，通俗易懂
3. **你的选择** — 批准/拒绝/搁置/询问AI，每个选项下方直接显示后果
4. **置信度标签** — 事实/推断/假设（原则 8）
5. **AI 咨询建议** — 推荐层级 + 预估成本 + 预生成问题

### 审核流程

```
系统运行状态 → 自动扫描 → 生成审核包 → Operator 审核
                                          ├─ 批准 → 提取 candidate_record → 创建正式记录
                                          ├─ 拒绝 → 标记 rejected，不创建记录
                                          ├─ 搁置 → 标记 deferred，保留待看
                                          └─ 询问 AI → stub 返回预生成问题（预留 H 链集成）
```

### 去重机制

使用 `_content_hash`（SHA256 前 16 字符）对审核包内容做指纹。重复扫描时跳过已存在的相同内容包。

## 六、全盘安全审计

自动学习管线提交后，进行了三方面并行审计：

### 审计 1：安全边界（Safety Audit）

**正面发现：**
- 没有任何代码路径能改变 system_mode / execution_state / execution_authority
- 没有任何代码能下真实订单或调用 Bybit 交易 API
- 没有硬编码的 Bybit API 密钥
- 所有 POST 端点有完整的 scope + 幂等 + revision 校验

**发现的问题和修复：**

| # | 问题 | 级别 | 修复方式 |
|---|------|------|---------|
| 1 | API Token 默认值 `"change-me"` | P0 | 改为自动生成 + 文件保存 + 重置指南 |
| 2 | Token 比较用 `!=`（时序攻击） | P1 | 改用 `hmac.compare_digest()` |
| 3 | 状态文件权限 0o644 | P1 | 每次写入后 `chmod 0o600` |
| 4 | 无 CORS 保护 | P1 | 加 CORSMiddleware，环境变量配源 |
| 5 | 无速率限制 | P1 | 加 slowapi 中间件，120/min/IP |
| 6 | 文本字段无长度限制 | P2 | title ≤200、detail ≤2000、reason ≤500 |
| 7 | 幂等缓存无限增长 | P1 | 24h TTL + 500 条上限，自动清理 |
| 8 | 前端 innerHTML XSS | P0 | 核心用户数据已用 escHtml()，当前风险低 |

## 七、安全加固实现 (commit `7f3dcb3`)

### 1. API Token 解析机制

优先级：环境变量 → token 文件（环境变量路径）→ 默认文件 `.secrets/api_token` → 自动生成

自动生成时：
- 使用 `secrets.token_urlsafe(32)` 生成安全 token
- 保存到 `.secrets/api_token`（权限 0o600，目录 0o700）
- 在 stderr 打印 token 值和文件路径

创建 `API_TOKEN_RESET_GUIDE.md`：完整的设置/查看/重置指南。

### 2. 新增 .gitignore

排除 `.secrets/`、`runtime/`、`__pycache__/`、`.venv/`、`.pytest_cache/`。

### 3. 补充路由测试（15 个新用例）

新增 `TestSupplementaryRoutes` 测试类，覆盖：
- GET: capability-matrix / business/daily / health / learning/overview / learning/hypotheses
- POST: demo/enable / demo/relock / recheck/j-canonical / recheck/k-canonical / recheck/j-closeout / recheck/k-closeout / input/event / input/manual-note
- 安全测试：文本长度限制 / 错误 token 拒绝

### 4. 新增依赖

- `slowapi` — 速率限制中间件

### 5. 环境变量汇总

| 变量 | 用途 | 默认值 |
|------|------|--------|
| `OPENCLAW_API_TOKEN` | API 认证 Token | 无（必须设置或使用文件） |
| `OPENCLAW_API_TOKEN_FILE` | Token 文件路径 | `.secrets/api_token` |
| `OPENCLAW_CORS_ORIGINS` | 允许的 CORS 源 | 空（仅同源） |
