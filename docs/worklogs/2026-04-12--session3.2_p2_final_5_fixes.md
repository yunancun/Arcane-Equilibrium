### FIX-57 — Python/Rust AI 預算 IPC 同步

- **問題**：Python `Layer2CostTracker` 和 Rust `BudgetTracker` 獨立追蹤 AI 成本，無同步機制
- **修復**：
  - **Rust IPC**：新增 `record_ai_usage` 命令（handlers.rs + mod.rs dispatch）
    - 參數：`{ scope, provider, model, tokens_in, tokens_out, purpose?, request_id? }`
    - 回傳：`{ ok: true, cost_usd: f64 }` 或 error
    - Fail-closed：tracker 未初始化或 DB 寫入失敗時回傳錯誤
  - **Python sync**：`Layer2CostTracker._sync_to_rust_budget()` fire-and-forget
    - 每次 `record_claude_cost()` 後在 daemon thread 中 IPC 調用 `record_ai_usage`
    - 非阻塞，失敗不影響本地記錄（Rust 引擎可能未運行）
- **設計決策**：僅同步 Claude API 調用（高成本），不同步 Perplexity/Ollama（零/低成本）
- **文件**：`ipc_server/handlers.rs` · `ipc_server/mod.rs` · `layer2_cost_tracker.py`
