## 技術決策 / Technical Decisions

1. **FIX-21 清理策略**：僅從 `lib.rs` 移除 `pub mod` 聲明，源文件保留在磁盤上（可能有歷史參考價值）。不編譯 = 零維護負擔。

2. **FIX-41 connectButton 替代**：`app-review.js` 中原本 `connectButton.click()` 觸發 → `loadDashboard()`，改為直接 `setTimeout(async () => { await loadDashboard(); ... }, 200)`。保留 200ms 延遲確保 DOM ready。

3. **FIX-44 錯誤處理策略**：選擇 `try { d = await ocApi(...) } catch(e) { d = null; }` 模式而非包裹整個函數體，因為 `ocApi()` 本身已 catch-and-return-null。額外 try/catch 是防禦性措施，主要價值在 null guard 分支改為顯示雙語錯誤文字（原本只是 `return;`）。

4. **FIX-45 刷新頻率**：30s→15s 是保守選擇。更激進的 5-10s 會增加 API 負載，但 Live 監控延遲從 30s 降到 15s 已足夠（position/order 變化通常不會快於 15s）。

5. **FIX-54 補錄粒度**：81 個「缺失」commit 中大部分是 docs/chore 或已被 session 級條目覆蓋。選擇補錄 3 個 session 級合併條目（PNL-FIX + 3E-ARCH 中間修復 + 本次 Batch A+B），而非逐 commit 記錄。
