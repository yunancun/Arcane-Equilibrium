OpenClaw / Bybit
新对话工作方式与带入文件清单（2026-03-25）
============================================================

一、建议在新对话里明确告诉模型的工作方式
------------------------------------------------------------
1. 能直接从 GitHub 仓库读、直接提交 GitHub 主线的，优先直接做。
2. Shell 命令尽量短、稳、分段，不要给特别长的大块 here-doc。
3. 如果需要我贴回输出，结论和需要我看的重点必须 print 在最下面，并清晰标注。
4. 新写和修改的程序必须继续保持中英双语注释，且注释尽量详细。
5. 每完成一段，都按以下结构汇报：
   - 已验证事实
   - 修改内容
   - 风险边界
   - 下一步
6. 绝不能把 shadow-only / skeleton-only / design-only gate closed / closeout ready 误写成 execution 放权。
7. 没核对到的就明确说没核对到，不要编。
8. 不要默认继续深挖 J/K；除非发现 blocker 或真实 bug。
9. 当前建议主线是：
   数据字典 / 状态字典 -> Control API -> GUI MVP -> guarded demo trading
10. L 的前半部分能力应嵌入 GUI / cockpit 设计里预埋，但当前不应先独立重开 L 作为主开发章节。
