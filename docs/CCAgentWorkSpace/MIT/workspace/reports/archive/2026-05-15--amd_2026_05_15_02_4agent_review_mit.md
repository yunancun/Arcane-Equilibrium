## §1 JSONB vs new column trade-off

**Linux PG 實測（ssh trade-core）**：
- `trading.fills` hypertable 35，6 chunks (6.3MB largest)，total 88kB，13,991 rows ts range 2026-04-14 → 2026-05-15
- compression_policy 1007 (compress_after 14 days, columnstore)，retention_policy 1019 (drop_after 365 days)
- `details` JSONB column 當前 **301/13,991 (2.2%) 非空**，全部是 `{"contaminated":true,"contamination_reason":"fa_phantom_1"}` legacy 標記
- INSERT writer `trading_writer.rs:430` 共 23 column，**完全不寫 details**

**新欄位查詢 pattern**（healthcheck [62] 必跑）：
```sql
SELECT count(*) FILTER (WHERE close_maker_attempt) / count(*)::float
  FROM trading.fills
  WHERE engine_mode IN ('demo','live_demo')
    AND exit_reason IS NOT NULL
    AND ts > now() - interval '7 days'
  GROUP BY strategy_name;
```
JSONB path `details->>'close_maker_attempt'`：
- bool cast 必走 `(details->>'close_maker_attempt')::bool` — Postgres planner 無法用 GIN index 快取 bool 結果
- partial index `WHERE (details->>'close_maker_attempt')::bool` 需 functional index，per-row cast 成本
- compressed chunk 對 JSONB path query 解壓開銷 vs new column columnstore native scan：差距 5-20x

**結論**：
- `close_maker_attempt: bool`、`close_maker_fallback_reason: text` → **new columns + partial index**（high-frequency group-by + healthcheck SLA）
- `close_initial_limit_price: double precision`、`close_final_fill_price: double precision` → **可 JSONB**（單筆 audit 讀取，不 group-by）

AMD §4.4 / spec §4.4 沒明文選定，這在 IMPL 時會分叉。**F-MIT-1 MUST**：AMD 必須**明文混合方案**或**統一決策**，否則 E1 IMPL 派工分叉。

## §9 AC-6 統計檢驗

**邏輯陷阱**：
- AC-6「`trading.fills.details` 中 `close_maker_*` 100% non-null」對 race condition / cancel_token shutdown / writer fail-soft 場景**無 NULL 階梯**
- 對照 V083 base：實測 close fill `entry_context_id` 7d 有 demo 8/285 (2.8%) + live_demo 7/204 (3.4%) NULL — 這是 W-AUDIT-4b-M2 已知 fail-soft tail
- 若 close_maker_* AC-6 strict 100%，第一筆 cancel_token 觸發的 fail-soft NULL 會 AC FAIL

**F-MIT-3 MUST**：AMD §11 AC-6 + 對應 healthcheck [63] 應分 NULL 比例階梯：
```
PASS: NULL rate ≤ 0.1% (per-environment 7d)
WARN: 0.1% < NULL rate ≤ 1.0%  
FAIL: NULL rate > 1.0%

排除分類：
- 'not_attempted_safety_path' value 是合法 audit value（不算 NULL），enum 內必含
- cancel_token shutdown 觸發的 audit 必寫 close_maker_attempt=false + close_maker_fallback_reason='engine_shutdown_safety'
```

healthcheck [63] 必加 enum value distribution sub-check：每個 fallback_reason value count + 預期分布範圍（postonly_reject > 0 但 < 5%；timeout_taker 是主要 fallback，30% 是 spec §11 AC-2 邊界）。
