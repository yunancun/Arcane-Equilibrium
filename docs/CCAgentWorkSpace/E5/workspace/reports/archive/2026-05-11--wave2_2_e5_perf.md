## B. Startup latency 影響

### B.1 LG2-T2 wait_for_first_refresh_or_timeout(30s)

**設計**（live_spawn_assert.rs:152-172）：

```rust
pub async fn wait_for_first_refresh_or_timeout(
    account_manager: &AccountManager,
    timeout: Duration,
) -> Result<(), LivePricingBindingError> {
    // Fast path：refresh 已完成 → 立即回 Ok 不 sleep
    if account_manager.last_fee_refresh_ms() > 0 {
        return Ok(());
    }
    let start = std::time::Instant::now();
    let poll = Duration::from_millis(POLL_INTERVAL_MS);  // 200ms
    while start.elapsed() < timeout {
        tokio::time::sleep(poll).await;
        if account_manager.last_fee_refresh_ms() > 0 {
            return Ok(());
        }
    }
    Err(LivePricingBindingError::NoRefresh)
}
```

| 場景 | 結果 | startup latency |
|---|---|---|
| Live spawn（fee refresh 已 await success） | fast path early return | **0ms 增量**（最常見） |
| Live spawn（fee refresh 失敗 OR seed_default 成功） | last_fee_refresh_ms > 0 → fast path | **0ms 增量** |
| Live spawn（fee refresh 與 seed_default 全失敗） | poll 200ms × 150 = 30s | **+30s** worst case |
| Demo / Paper spawn | `if kind == PipelineKind::Live` 守衛跳過 | **0ms 增量** |

**正常啟動路徑（99% 場景）**：
- build_exchange_pipeline 內 `acct.refresh_fee_rates(&*client_arc, "linear").await` 同步成功 → last_fee_refresh_ms 已 stamp
- LG2-T2 wait fast path 立即返回 → 加 ~3 個 atomic load + 1 branch ≈ **<100ns**

**異常啟動路徑**：
- demo/LiveDemo unsupported endpoint → seed_default_fee_rates 同步 stamp last_fee_refresh_ms → fast path
- mainnet 真錯 + seed_default 不適用 → 進入 poll loop → worst 30s timeout → reject spawn（fail-closed 合 §四 hard boundary）

**結論**：✅ **PASS**。fast path 主導；30s 是 defensive net 對 production 啟動時間影響在 worst case 路徑而 worst case 路徑會 reject spawn 反正不啟動。

## C. ArcSwap hot-reload 影響

### C.1 PricingConfig 在 RiskConfig snapshot 路徑

**LG2-T4 設計**（risk_config.rs:`pricing: Option<PricingConfig>`）：

```rust
pub struct RiskConfig {
    // ... existing fields ...
    #[serde(default)]
    pub pricing: Option<openclaw_types::PricingConfig>,
}
```

**Hot-reload path**：
1. TOML 改 `[pricing]` section
2. `RiskConfigManager::reload_from_disk()` 觸發
3. `ConfigStore.swap(new_snapshot)` 走 ArcSwap 原子替換（~50ns）
4. 下個 tick 頂部 `sync_risk_config_if_changed` 發現 version bump
5. `apply_risk_snapshot(new_snapshot)` 拉新 snapshot 給下游派生 copy（H0GateConfig / cost_edge / 其他）

**LG2-T4 self-claim**：「PricingConfig 在 LG2-T4 階段沒有任何下游 owned consumer」— 所以**不需在 apply_risk_snapshot 加 RMW 邏輯**。

**E5 verify**：
- E1 LG2-T2 IMPL 在 `live_auth_watcher::run loop` respawn path 走 `self.risk_live_store.as_ref().and_then(|s| s.load().pricing.clone()).unwrap_or_default()`（live_auth_watcher.rs:732-738）
- main.rs boot 路徑 `risk_stores.live.load().pricing.clone().unwrap_or_default()`（main.rs:330-334）
- **兩個 caller 都是 startup-time / respawn-time，不是 hot path**

**結論**：✅ **PASS**。PricingConfig 純 startup/respawn 讀；ArcSwap snapshot read 50ns + Option<PricingConfig>::clone() 100ns（3 field + 1 Vec<String>）= 可忽略。

### C.2 PricingConfig clone 成本

```rust
pub struct PricingConfig {
```

**Caller frequency**：

**結論**：✅ **PASS**。clone cost 可忽略。
