## §3 對抗性審查（七項自查）

### 3.1 BUG-A 真修了沒？

**問**：rename 後 `_apply_cognitive_modulation` 的外層 `try/except Exception` 是否仍可能掩蓋未來 regression？
**答**：是。但本 fix 不擴外層 except 範圍（保守），新增 `test_get_all_params_does_not_raise()` 為 regression guard：未來若 modulator API 再 rename 走，test 立刻紅。**接受**。

### 3.2 BUG-B 真接到 hot path？

`_handle_intel` line 367-368：
```python
if _intel_count % _COGNITIVE_TICK_INTERVAL == 0:
    _sc_tick_cognitive_modulator(self)
```

放在 `_lock` 取 `_intel_count` 之後（line 354 `_intel_count = self._stats["intel_received"]`，atomic snapshot），早於任何 `if not payload: return` / `if intel.relevance_score < ...: return` 等 5 個 early return。**Unconditional fire 屬實**。

**例外**：emergency_mode 早於增量（line 345-350）— 此時 `_stats["intel_received"]` 不增加 = tick 不 fire。**設計合理**（emergency 期應凍結認知狀態演化）。

整合測試 `test_handle_intel_drives_modulator_at_tick_interval` 正面驗證 update_count 確實隨 N 個 intel 推進。✅

### 3.3 N=10 magic number 合理？

PA RFC §10：intel ~5/min → N=10 = ~2min/update，符合 modulator EMA(α=0.3) ~3-4 cycle 收斂節奏。Magic number 已 **明確命名常量** `_COGNITIVE_TICK_INTERVAL` + 雙語注釋說明 cadence rationale。✅
