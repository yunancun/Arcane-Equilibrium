### **CRITICAL-G05**：原則 #3 核心缺陷 — ExecutorAgent 決策授權斷裂

**違反原則**：#3 AI 輸出 ≠ 命令 + #11 Agent 自主權

**徵狀**：
- `executor_agent.py:482` ExecutorConfig 預設 `_shadow_mode=True`
- `strategy_wiring.py:467` 初始化時未覆蓋 shadow 値
- 結果：ExecutorAgent 產出 shadow intent log，**不發送 SubmitOrder IPC 到 Rust**（設計上避免 Path A/B 倉位衝突）
- Live 場景下 Strategist `shadow=False` 後，intent 來自規則引擎，**非 AI 產出**（AI 輸出被人為禁用）
- Decision Lease 機制完整但物理上無對應訂單提交觸發

**證據**：
- executor_agent.py 第 281 行 `submit_order()` 前未呼 `acquire_lease()`
- Guardian 批准 intent 但 ExecutorAgent `_shadow_mode=True` 拒發
- 無法區別「AI 批准但未執行」vs「AI 未批准」（日誌都有，決策鏈斷裂）

**影響等級**：🔴 **BLOCKER** — 整個 AI→Lease→複核→執行 迴圈在 Executor 層斷裂；任何 live 部署前必修復

**修復路徑**（CLAUDE.md memory.md 2026-03-31 記載）：
```python
# executor_agent.py::_execute_order() 插入
lease_acquired = self._governance_hub.acquire_lease(
    intent_id=intent.intent_id,
    scope=intent.scope,
    expires_ms=self.config.lease_ttl_ms
)
if not lease_acquired:
    # fail-closed REJECT
    await self._emit_execution_report(..., success=False, error="lease_denied")
    return
# 繼續執行 submit_order()
```

**預計工作量**：~2h（含測試補全）；前置必須 Sprint 5a（當前為 demo only）
