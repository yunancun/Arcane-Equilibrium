## 1. Executive Summary

1. **估計對象重定義**：測的不是「bb_reversion maker 化後 net=+0.3bps 嗎」（σ≈8bps 下需 n≈4,400 episodes ≈10 年，不可檢定），而是三個可測分量：(i) 信號條件下無條件 maker fill rate；(ii) 真 post-fill 60s/300s beta-residual markout（現有效 n=1）；(iii) 同 episode 配對 maker-vs-taker 執行成本差 Δcost。
2. **判定式三態**（§6）：SCREEN_PASS（→ bounded probe 候選）/ KILL（附翻案條件）/ INSUFFICIENT（禁方向性結論）。**KILL 的 power 在 R1 僅對 K1/K4 可達**（FIX-1）；K2 的 selection-gap MDE@30≈23bps → K2 kill power 近零，真 selection kill 表現為慢性 INSUFFICIENT 至 2026-08-10 park；K3 需 n_pair≈60（RT-9）。SCREEN_PASS 最早在 n_eff≥30 且 ≥15 post-registration episodes（FIX-3）。
3. **記帳 = intention-to-treat**：per-episode 無條件四態（fill/adverse_through/no_fill/reject 全入分母），fill-only 統計僅作機制分解，不承載判定。
4. **與 maker-nogo 無衝突**（§2.3）：彼裁決殺「做市報價作為 alpha」（無條件雙側掛單，0/172 淨正，break-even maker≤0.4bps）；本研究是「既有信號的執行路由」，收入項=費率替代（taker 5.5→maker 2.0，entry 腿確定性省 3.5bps）±選擇效應。估計對象/分母/機會集合全不同。
5. **誠實前置**：60d gross +3.66bps 下即使 markout=0 maker 化算術也是負（−0.34，evidence F-3）——執行 lever 通過 screen ≠ cell 可盈利；probe 定位=量測儀器（learning-only、loss budget 封頂、`order_authority=NOT_GRANTED`、`promotion_evidence=false`），非 promotion 證據。
6. **預期路徑（誠實申報）**：R1 大概率 INSUFFICIENT（n_eff 25<30、時間簇 14、選擇窗集中）但 KILL-capable（K1/K4）；結構性先驗（p50 持倉 48s < τ=60s：信號壽命≈隊列等待時長，贏家側 fallback 幾乎必然吃掉反轉幅度）+ 60d gross 轉弱 → 大概率終於 KILL。**KILL 本身是 M12 自適應 router 的第一份 strategy-conditional 真 markout 數據，具淨價值。**

## 2. 理論基礎：估計對象（estimand）與邊界

### 2.1 Alpha 來源歸類

本研究**不提出新 alpha**。bb_reversion 既有歸類=類別 6（短期均值回歸）；對象是**執行成本結構**（fee schedule 的 maker/taker 價差是交易所定價結構性事實，非市場低效）。誠實答案：**執行 lever 本身不賺錢，它降低既有信號的成本下限**；cell 是否賺錢仍由信號 gross 決定（目前不足且 60d 轉弱）。

### 2.2 三個分量預算（成本恒等式，bps of notional，成本為正）

以 mid@signal 為基準價，entry 腿單腿成本：

```
c_T(taker)   = hs(t_place) + fee_taker                          [hs=半價差；impact≈0（最小單，聲明）]
c_M(M-fb, τ) = p·(−hs_captured + AS + fee_maker)
             + (1−p)·(drift_τ + hs(t_place+τ) + fee_taker)      [timeout 後 taker fallback]
Δcost_entry  = c_T − c_M                                        [>0 = maker 較便宜]
```

- p=1 時 Δcost = 3.5 + hs_captured − AS → 執行 lever 的 AS 預算 ≈ 3.5 + hs ≈ 4.5-5.5bps/leg（majors hs~1-2bps）。
- 對照 cell 淨值預算（30d gross）：AS ≤ (8.86−4.0)/2 = 2.43bps/leg（100% fill、兩腿 maker）；60d gross 下該預算為負。screen 測前者；後者需 gross 證據另行成立。
- 外部先驗（僅結構類比，不入判定）：DeLise 2024 negative drift 回吞 ~90% 半價差 → hs_captured 有效值可能僅 0.1-0.2bps；fill_sim 兩窗 pooled AS 0.85-2.6bps@15s；bb_rev 單筆 −2.37。**Δcost 符號真不確定 → 實驗值得做**（先驗中性，與 n=1 讀數無關）。

### 2.3 與 maker-nogo 裁決的邊界聲明（為何不重打）

| 維度 | maker-nogo（2026-07-06） | 本研究 |
|---|---|---|
| 估計對象 | 做市報價 NET = hs − AS − 2×maker_fee（自足 alpha） | 執行路由 Δcost = c_T − c_M（既有信號成本差） |
| 分母 | 固定 cadence 無條件 quote-trials（兩側恆掛） | bb_reversion 信號 episodes（單側、事件觸發） |
| 反事實 | 不掛=0（無 taker 對照） | 不 maker=taker 執行（同 episode 配對對照） |
| 費用角色 | +2bps maker fee 是收入端的牆 | 2.0 vs 5.5 是成本端確定性節省 3.5bps/leg |
| 裁決適用 | 0/172 淨正 → 報價業務 NO-GO（不動搖） | 未被覆蓋；maker-nogo 自身把 cost-reduction 列 open fork #3 |

**不重打承諾**：任何結果不得引為「maker-first 翻案」。KILL → 強化 fee 牆敘事；SCREEN_PASS → 僅證「信號條件下執行路由」可行，報價業務仍被 0/172 + break-even≤0.4bps 鎖死。

### 2.4 執行政策空間（凍結）

- **主政策 M-fb**：maker entry（PostOnly join-BBO）+ timeout τ 後 taker fallback。保持交易集不變，把「信號 EV」與「執行成本」正交分解；M-skip 降為次要政策。
- **Entry 腿 only 為主**：exit 72% 是 phys_lock giveback（tick 級動態，不可獨立再生，evidence F-7）→ exit 沿用 realized ts/price（execution-counterfactual）。兩腿全 maker 僅 Lane A 次要欄。此設計取代 stage2「兩腿 maker 8.7bps」算術——主判確定性費率節省=**3.5bps/RT（entry 腿），非 7bps**。

## 3. 重放判準（全部凍結）

### 3.1 Fill 規則帶（三層，主判用中層）

| 帶 | 規則 | 用途 |
|---|---|---|
| 上界 | touch-based（價觸及=成交） | **僅列報，禁入判定**（DeLise 85% vs 真值 60%，PnL 失真）；唯一判定用途=K4 死刑（連上界都輸 taker → 必 KILL） |
| **主判** | fill_sim queue 狀態機（`simulate_symbol` fill_sim.py:152）：掛單時 snapshot 同側 size-at-touch=Q0；同側 aggressor at-or-through 累減；cancel-ahead 只推進不觸發（fill_sim.py:293-301）；三態 FILL / NO_FILL / ADVERSE_THROUGH（fill_sim.py:168-170） | 全部 G/K 判準 |
| 下界 | through-print only（僅價格實穿才 fill） | G5 穩健性檢查（不得災難性翻轉） |

### 3.2 Queue 折扣：主判=back-of-queue（size_ahead_frac=1.0）

理由：①物理（信號掛單永遠入既有隊列尾端）；②不重複計算（resolver 已動態建模 cancel-ahead 推進，靜態 50% 折扣會把撤單利益算兩次）；③偏置方向（MM Dilemma 隊尾 markout −0.775bp vs 隊頭 −0.058bp [外部類比]，用 mid 檔會把 AS 做小）；④可比性（maker-nogo 兩窗即以 back-of-queue 為 default）。front(0.0)/mid(0.5) 保留為劑量反應敏感性欄，不得進判定。

### 3.3 Through-print / ADVERSE_THROUGH 判別

- 隊列耗盡且 best 仍在你的 level → FILL；成交同刻 best 已穿過 → ADVERSE_THROUGH（fill_sim.py:257-265），half_spread 改以 mid@fill 為基準（BUG-2 修正 fill_sim.py:401-411）。
- best 改善離開（未耗盡隊列）→ NO_FILL 入分母；資料尾截斷 → NO_FILL（保守 fill_sim.py:306-307）。
- ADVERSE_THROUGH 記帳上=fill，機制分解上單列（結構性壞 fill 佔比為必報欄）。

### 3.4 Placement 參數（凍結主格）

| 參數 | 主判值 | 敏感性掃描（僅列報，exploratory=true） |
|---|---|---|
| δ_place（信號 ts → 掛單） | **1s**（覆蓋 1m close 計算+IPC+REST，保守） | 250ms / 5s |
| 掛價 | join BBO 同側（OpenLong→bid；details 全 NULL → 價格錨=t_place 的 L1 BBO，evidence F-10） | one-tick improve |
| timeout τ | **60s**（≈1 bar；p50 持倉 0.8min 同量級；承 2026-04-20 timeout<cooldown 教訓） | 30s / 180s |
| side | **OpenLong only 主判**（OpenShort 僅 07-06 後 7 episodes，新行為+down-beta → 單列、demeaned beta 標註，禁入主判定；06-03 鐵則） | — |
| horizons（markout） | **60s + 300s 主判雙窗（AND 邏輯）**；5/15/30s（AS 分量）與 holding-horizon（Lane A 至 realized exit）必列 | — |

### 3.5 PostOnly reject 建模（兩層）

1. **確定性層**：t_place 時意向掛價會立即 cross 對側 best → PostOnly reject（Bybit 機制），計 reject 態入分母；resubmit 一次於 t_place+1s，再 reject → 該 trial 按 no_fill+fallback 政策計價。
2. **隨機敏感帶**：0/1/2/5% 隨機殺單+resubmit 延遲成本，報退化斜率；斜率陡（2% 殺單即翻轉 G4 符號）→ 結論標不穩，SCREEN_PASS 降級 INSUFFICIENT。

### 3.6 資料品質前置（重放第 0 步）

- L1 crossed-row / ts-floor fail-loud 過濾（`--clean-since` ≥ 2026-06-17T14:25+02）；per-symbol L1 gap 審計（episode ±τ 窗內最大 L1 空洞 > 5s → 該 trial 標 censored）；censored 入分母不入檢定，censored_pct >30% → E4 fail。

- **L1 retention 前置（v1.1，FIX-4）**：重放第 0 步即完成 §0.4 piece-3 枚舉複驗 + episode-slice artifact 匯出（凍結斷言錨遷 artifact store）。**06-28 episode L1 於 ~07-19 掉窗** → R1 必本週跑（GO 條件 C2）。

## 4. 記帳與 non-execution bias（凍結）

### 4.1 裁決記帳 = intention-to-treat（per-episode 無條件）

- **觀測單位 = episode**（gap-dedup 30min；同分鐘跨 symbol 齊發合併為 1 個時間簇用於 cluster 計數）。per-quote/per-minute 行數永遠不是 n_eff（F1 偽複製教訓）。
- 每 episode 三軌並列（同一信號集）：**Track T**（taker-at-signal；Lane A 用 realized 成交價、M-arm 錨 realized order-submit ts，Lane B 用 t_place 對側 best 模擬）/ **Track M-fb**（主政策）/ **Track M-skip**（次要）。
- 結局四態全入分母：fill / adverse_through / no_fill / reject。**fill-only 與 unconditional 兩版並列，兩者之差 = selection bias 的直接量測值（本實驗第一輸出）**。
- 漏掉的 fills 計價：M-fb 下=fallback taker 實付（drift_τ + 當時 hs + 5.5）；M-skip 下=0 倉位，機會成本體現在與 Track T 的 episode 級差值（漏掉贏家按 taker 軌淨值扣分）。**判定綁 ITT 量；per-fill 僅作機制分解。**

### 4.2 FIX-1（v1.1 新增，凍結前必修）— M-fb fallback_ts ≥ realized_exit_ts 退化 case 記帳規則

**背景 FACT**：p50 持倉 48s < τ=60s（evidence a.6）→ 對 **≥50% 的 Lane A pairs**，M-fb 的 taker fallback 進場發生在 realized exit **之後**，「maker entry（模擬）+ realized exit」的配對 Δnet 出現負持倉時長。v1 只講了 exit 時點漂移，未定義此退化 case。

**凍結規則（預先指定，禁任何 R1 執行期 ad-hoc）**：
1. 若某 episode M-fb 的 `fallback_ts ≥ realized_exit_ts`（即 τ 窗內未 maker-fill，且 taker fallback 落在原 RT 已平倉之後）⇒ 該 episode 的 **M-arm 記為 `trade_forgone`**，`Δnet_M = −(realized net PnL of that RT)`（把錯過的整筆交易入帳，最保守且與 ITT 一致）。
2. **固定 horizon markout 欄並列**（不依賴 realized exit）：對每個 episode 另計「entry 起 +60s / +300s 固定 horizon」的 M-arm markout，作為不受 exit-timing 漂移污染的次判參照欄（R-3 緩解）。
3. `trade_forgone` 佔比為必報欄（預期 ≥50%）；任何偏離本規則 = deviation §12.2 級 = 強制 v2。
4. **KILL power 措辭限定**：因 K2/K3 在 R1 樣本下 power 不足，**「KILL 的 power 在 R1 可達」僅對 K1（fill 饑餓）與 K4（上界死刑）成立**（此二者不依賴退化 case 記帳、不依賴 day-cluster power）；K2/K3 為慢性累積判定。

## 5. 統計推斷、樣本量與 power（凍結）

### 5.1 推斷框架（沿用 07-10 反事實預註冊 §2-§5）

- **n_eff** = episodes（gap-dedup=非重疊化對應物；Lane A = 配對 RT 數）。
- **Cluster**：CR1 cluster-robust variance by UTC day，`V=[G/(G−1)]·(1/n²)·Σ_g S_g²`，t 檢定 df=G−1，單側。比例類用 day-cluster bootstrap B=1000、seed=20260710，Wilson CI 並列。
- **Eligibility（E 門檻）**：E1 n_eff≥30 episodes（主判 side）；E2 distinct UTC days≥5；**E3 top-day episode-share ≤50%（RT-11 凍結定義=episode-share，非 gross-share；現值 07-08=32% 過門檻）**；E4 censored_pct≤30%；E5 有 fill 的 symbols ≥5。任一 fail → INSUFFICIENT。
- **FIX-3（v1.1 新增，凍結前）**：SCREEN_PASS 的決策樣本須含 **≥15 個 post-registration episodes（ts > 2026-07-10）**，且全部 primary outcomes（fill rate / markout / selection gap / Δcost）必報 **in-selection-window vs post-window 分裂欄**——把決策多數票搬到選擇窗之外（防選擇窗運氣與同窗 AS 量測的機械耦合，RT-8）。
- **多重比較**：主判定=單一預註冊 conjunction（G1-G8 AND、雙 horizon AND）→ 無選擇效應，不需 Bonferroni；全 sweep 欄標 `exploratory=true`，禁入判定/禁候選語言；artifact 登記全部已計算 cells（K 登記，§0.4 piece-3）；從 sweep「發現」更好格子 → 必 v2。
