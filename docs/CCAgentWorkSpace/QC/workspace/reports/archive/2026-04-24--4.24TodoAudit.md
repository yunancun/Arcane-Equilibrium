### 3.1 James-Stein Estimator 數學正確性

**實裝檔案**：`program_code/ml_training/james_stein_estimator.py` (556 lines)

**公式驗證**：
```
理論 (Stein, 1956):
  θ̂_JS = μ + (1 - B) × (x - μ)
  B = (p-2) × Σσ²_i / ||x - μ||²  (positive-part: clamp [0,1])
  
實裝 (line 177-181):
  sq_sum = Σ(x_i - grand_mean)²
  pooled_var = Σσ²_i / p
  B = (p - 2) × pooled_var / sq_sum
  shrunk = grand_mean + (1 - B) × (x - grand_mean)  ✅ 正確
```

**小樣本護衛**：
- p < 3 時 JS 未定義 → 回傳原始值 ✅
- within_vars 計算 (line 252-260)：var = std² / n (SEM 平方) ✅
- single observation fallback = 100 bps² prior ✅（保守）

**多維收縮 (5-01)**：
- win_rate 獨立收縮（line 306-307）：`win_rate × (1-win_rate) / n` ✅
- avg_win/avg_loss 獨立收縮（line 308-311）✅
- combined EV (line 315-319)：`wr_shrunk × aw_shrunk + (1-wr_shrunk) × al_shrunk` ✅

**VERIFIED ✅**：James-Stein 數學無誤。

**但**: 樣本不足 (n_cells=1 < 3) 使 JS 退化為原始值 + 無意義的 grand_mean。

## 4. 數學錯誤 Top 3

### 4.1 Edge_Estimates Grand Mean 計算誤用

**錯誤**：當 n_cells=1 時，grand_mean 定義為「唯一的 raw_bps」，無法代表 universe mean。

**公式濫用**：
```python
# line 261-264
grand_mean = sum(raw_values) / len(raw_values)  # len = 1 時 = raw_values[0]

# line 273
B_global = max(0.0, min(1.0, (p - 2) * pooled_var / sq_sum))
# p=1 時： (1-2) × ... = -1 × ... < 0 → B = 0 (clamped)
# 且 ||raw - mean||² = 0（唯一的值與自身的距離=0）→ B = 0
```

**根本問題**：JS 公式在 p < 3 時未定義，代碼正確回傳原始值，但 `grand_mean_bps` 在此情況下失去信息價值（分母為 0）。

**QC 建議**：
- 當 n_cells < 3 時，grand_mean 應設為 `NaN` 或明確標記 `_meta.is_valid = false`
- 修復代碼 (line 264)：
  ```python
  if len(raw_values) < 3:
      grand_mean = float('nan')
  else:
      grand_mean = sum(raw_values) / len(raw_values)
  ```

**嚴重度**：🟡 **中等** — 不致引發運行錯誤，但污染 JSON 數據，Gate 1 可能信任無效的 grand_mean

### 4.2 Win Rate Shrinkage 方差估計有偏

**錯誤**（line 307）：
```python
lambda es: es.win_rate * (1 - es.win_rate) / max(es.n, 1)
```

**問題**：win_rate 的方差應為 `p(1-p)/n`（Bernoulli），但此公式假設 win_rate 本身是點估計，實則需要樣本方差。

**正確應為**：
```python
# 若 win_rate 是從 n 個樣本計算的比例估計，其樣本方差 = p(1-p)/n
# 但若各組（cell）的 win_rate 本身是隨機變數，需要：
# Var[p_est] = E[p(1-p)] = mean over all cells of p_i(1-p_i)

# 當前實裝實際上是對（given p_i, Var[p] = p(1-p)/n_i），所以有效
```

**QC 評價**：✅ **實際正確** — 代碼邏輯是「對每個 cell 的 win_rate，計算其方差」，公式正確。我的初判有誤。

### 4.3 Donchian Leak 的測量偏診斷不完整

**問題**（line 189-196）：
```
Donchian 包含 current bar ⇒ breach 必然是局部高/低
⇒ mean-revert 傾向 ⇒ forward return 偏低
```

**但缺失的推理**：
- **Measurement bias vs Signal quality** 的區分
  - Bias: 由於定義問題（包含 current bar），導致信號在統計上有偏
  - Quality: 信號本身的預測力
- **leak-free variant 是否等同修復**？
  - leak-free (shift(1)) 排除當前 bar，但仍可能有其他偏差（如 momentum 延遲）

**結論**（基於代碼註解，line 192-193）：
> "leak-free Donchian is the textbook breakout signal; current-bar-inclusive is what the engine actually does"

**QC 評價**：⚠️ **認知清晰但不完整** — 報告應包含：
1. leak-free variant 下，breach 預測力是否高於 current-inclusive？
2. 若高，差異多大？（用 `breach_diff_tstat_leakfree` vs `breach_diff_tstat`）
3. 修復建議：若差異 > 2σ，考慮改 engine 的 Donchian 定義
