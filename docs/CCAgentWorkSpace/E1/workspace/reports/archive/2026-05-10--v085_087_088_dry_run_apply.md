### 2.5 task spec 與 sqlx checksum 算法的偏差

task spec Step 4 給的範例：
```sql
digest(pg_read_file('/home/ncyu/BybitOpenClaw/srv/sql/migrations/V085__panel_funding_curve.sql'), 'sha384')
```

實測 `pg_read_file` 失敗：`could not open file ... for reading: No such file or directory`。原因：PG 在 docker container 內，data_directory = `/var/lib/postgresql/data`；`pg_read_file` 限制 inside data_directory 或需 superuser 絕對路徑權限，而 `trading_admin` 不是 superuser。

**實際採用 approach**：shell `sha384sum` 計 hex → SQL `decode(hex, 'hex')` 變 bytea。結果完全等價。
