- **新 helper 重複 view-exists check 與 GROUP BY query**：與 ratio helper 結構幾乎相同（兩個 fail-soft block + try/except logging）。考量 (a) RFC §3.1 / §9.3 顯示這是兩個獨立 producer surface；(b) 合併會犧牲 fail-soft 隔離（一邊 SQL 拋例外另一邊也回 0）；(c) consumer 兩 dict 應自獨立 source 拉以便 retro audit；保持兩個 helper 獨立。E2 若認為應合併（e.g. 同一 SQL `SELECT strategy_name, count(*), count(*) FILTER (WHERE attribution_chain_ok)` 共用）→ 後續 follow-up，本輪不動以最小化範圍。
