## §3 關鍵 SQL 與 verdict 邏輯

### 3.1 主 aggregate SQL（cursor block 內單一 query）

```sql
SELECT
    EXTRACT(EPOCH FROM NOW() - to_timestamp(MAX(snapshot_ts_ms)::BIGINT / 1000.0))
        AS age_seconds,
    MAX(array_length(alt_symbols, 1))::INT
        AS cohort_size_max,
    COUNT(*)::INT
        AS total_n,
    COUNT(*) FILTER (WHERE regime_tag = 'extreme')::INT
        AS extreme_n,
    AVG(ABS(btc_book_imbalance)) FILTER (WHERE btc_book_imbalance IS NOT NULL)
        AS book_imb_abs_avg,
    COUNT(*) FILTER (WHERE btc_book_imbalance IS NOT NULL)::INT
        AS book_imb_nonnull_n
FROM panel.btc_lead_lag_panel
WHERE snapshot_ts_ms > (EXTRACT(EPOCH FROM NOW() - INTERVAL '60 minutes') * 1000)::BIGINT
```

走 V088 hot-path index `idx_btc_lead_lag_panel_ts_window (snapshot_ts_ms DESC, lead_window_secs)`；1h window 約 60 row 極輕（Linux PG empirical exec time = 0.167ms）。

### 3.2 三段 verdict

```python
# 條件 1：age
if age_ms < 120_000: age_verdict = "PASS"
elif age_ms < 300_000: age_verdict = "WARN"
else: age_verdict = "FAIL"

# 條件 2：cohort
if cohort_size == 7: cohort_verdict = "PASS"
elif cohort_size < 7: cohort_verdict = "FAIL"
else: cohort_verdict = "WARN"  # > 7 producer 多寫

# 條件 3：regime extreme ratio
if ratio < 0.05: extreme_verdict = "PASS"
elif ratio < 0.20: extreme_verdict = "WARN"
else: extreme_verdict = "FAIL"

# 條件 4：book imbalance
if book_avg is None:            # 全 NULL
    book_verdict = "FAIL" if book_required else "WARN"
elif book_avg <= 1e-9:          # placeholder 0.0
    book_verdict = "FAIL" if book_required else "WARN"
else:                            # 真實 W2-IMPL-1 接線生效
    book_verdict = "PASS"

# 整體 verdict（PA §3.3）
if age == FAIL or cohort == FAIL or extreme == FAIL or fail_count >= 3:
    overall = "FAIL"   # → silent-dead Exit 1
elif warn_count > 0 or fail_count > 0:
    overall = WARN_or_FAIL_if_REQUIRED
else:
    overall = "PASS"
```
