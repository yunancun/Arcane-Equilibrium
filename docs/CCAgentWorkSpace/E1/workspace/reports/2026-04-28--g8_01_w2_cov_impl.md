## §5 regret/dream exclusion mechanism — N/A (full coverage achieved)

The task brief mentioned adding `# pragma: no cover` or module-level `omit`
for the deferred-unreachable regret/dream branches. **Decision: not used** —
because:

1. **Public API exposes them directly**: `CognitiveModulator.update(...)`
   accepts `regret_data` + `dream_data` as kwargs. Unit tests at this layer
   are testing the **modulator API contract**, not StrategistAgent
   integration. The branches are reachable in unit tests by passing the
   expected schema dicts directly.
2. **Adding `# pragma` would require editing production code**, which the
   task explicitly forbids ("不改 cognitive_modulator.py production code").
3. **Module-level `omit` config (`.coveragerc`)** would require excluding
   specific line ranges, which is brittle (line numbers shift on any
   refactor) and hides regression coverage if/when the producers are
   re-implemented (RFC Option B).
4. **Achieved 100% line cov anyway** via direct API calls. No exclusion
   needed; cov budget has 15-point headroom above the 85% bar.

This treatment is documented in the test module's MODULE_NOTE so future
readers (E2/E4/QA) understand these branches are *currently* dormant in the
production hot path (per REGRET-DREAM escalation report) but remain valid
API-contract surface.
