## 長期教訓

- 自評不可信：任何「IMPL DONE / PASS」宣稱（含前手 agent、含自己）commit 前必 grep 證 fn 真存在 + 親跑 spec'd 測試；commit message 與 report 必對齊，Partial 就寫 Partial 並列未進 commit 的檔案清單。
- PG-coupled 改動（V### migration / schema / writer）必先 ssh trade-core 對 Linux PG empirical reflection（`\d` + pg_get_constraintdef）；spec 內 SQL 非 SSOT；migration 必雙 apply 驗冪等；PG 容器=trading_postgres、user trading_admin、db trading_ai。
- spec / dispatch packet 的路徑、fn 名、section 引用、base SHA、LOC 估算常 stale 或 typo：IMPL 前 find / grep / git cat-file 驗證，以代碼真實狀態為準；偏差必 push back 且在報告明標 deviation（prompt > spec）。
- 改 struct 欄位 / fn 簽名 / enum variant / return 語意必 grep 全 production caller + 全 test 構造點；`cargo build --lib` 過 ≠ `cargo test` 編譯過（manual struct literal、exhaustive match）；gate / cutover 類改動必跑 base-vs-HEAD 全套失敗清單 diff。
- production path 嚴格 fail-closed：不留 unwrap()、normalize / parse 得 0 或非法值回 None 不送原值、缺 secret / 解析失敗=disabled 非 error；LiveDemo=live 槽憑證+demo 端點，是合法 Live 模式（5-gate 按 Live 嚴格標準），任何守衛不可誤殺。
- 測試必須 bite：修 race / gate 後做 mutation A/B（臨時還原 buggy 版證 test 真 FAIL 再還原修法）；不 monkeypatch 掉被測對象；fixture 必鏡像真實上游 boundary 不變量（synthetic fixture 讓 silent bug 沉默通過）；對抗 probe 必含 production-empirical 真實字串，全 mock 自我引用=自驗空話。
- 「語法過+已提交 ≠ 已接線」：helper 加了但 emit site 不走、Python client 存在但 dispatch arm 沒接、gate 維度沒 thread 進 producer 都是 silent 死碼；接手他人 IMPL 必 trace 全鏈接線 + 親跑 full test。
- refactor 搬移模塊前先 grep tests 的 monkeypatch / patch 命名空間（命名解析縫是 load-bearing 約束）；stub `from PKG import SUB` 須 dual patch sys.modules + package attr，finally 反序還原防跨 test 汙染。
- multi-session dirty tree 紀律：不 revert 不認識的改動；stash / linter 事件後 git status + grep 驗自己 edits 還在；cross-wave 檔 `git checkout HEAD --` 還原；meta-doc 用 `git commit --only`；worktree 內寫檔絕對路徑必指 worktree root 非主 srv tree。
- Mac 只能驗純邏輯層（無 fastapi / pytest 受限、無 PG / engine）；advisory-lock、PG runtime 語意、真連線 smoke 必 ssh trade-core 實機；Mac engine not_running 是預期非故障。
- 跨平台路徑紅線：production 代碼與 doc 均不得出現 /home/ncyu、/Users/...（placeholder 也會被 grep 命中）；用 `Path(__file__).resolve().parents[N]`（N 必實測）或 CARGO_MANIFEST_DIR / srv_root() 推算。
