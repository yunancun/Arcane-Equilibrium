## 附錄 C：去重算法與統計

### C.1 去重邏輯

```
10 份提案 × 平均 50 項/份 = ~500 項原始
去重維度：
  1. same_problem(描述 + 根因 + 影響) → merge as 1
  2. different_domain(QC 量化 vs FA 功能層) → keep separate
  3. supersede(舊版 claim 被新審計推翻) → latest version only
  4. hierarchy(P0 > P1 > P2 > MID > LOW) → keep highest level
結果：180-220 項獨立 finding（55-60% 去重）
```
