# OpenClaw / Bybit Control API & GUI 阶段性收尾清单

日期：2026-03-26
分支：`feature/openclaw-bybit-control-api-gui-v1-rc2`

---

## 1. 本清单用途

本清单用于定义：

- 当前 API / GUI 章节在“阶段性完工”意义下，还差哪些收尾项
- 哪些项目达到后，可以视为本章节已基本收口
- 哪些内容应留待主线后续，而不在本章节继续膨胀

---

## 2. GUI 章节收尾判定项

### 2.1 页面结构

- [ ] 首页主要区块已固定，不再频繁大改布局
- [ ] 运行模式控制区已固定为正式骨架
- [ ] business / income 区已固定为正式骨架
- [ ] 产品族配置区已成为独立卡片，而非散落在表格说明中

### 2.2 可读性

- [ ] 关键动作说明已经使用“人话”表达
- [ ] 双语表达不再明显干扰排版
- [ ] 关键概念提示默认折叠
- [ ] 产品族命名已含中文解释

### 2.3 交互

- [ ] 关键动作均具备二次确认
- [ ] 关键动作执行后会有明确结果反馈
- [ ] 页面刷新后能够正确更新 overview / product family / source context

### 2.4 预埋完成度

- [ ] OpenClaw 统一入口命名已预埋
- [ ] 产品族配置区已预留更多设置承接位
- [ ] observe / demo / live 等层级展示位已预留

---

## 4. 本章节不追求在此刻完成的内容

以下项目不作为当前章节“阶段性完工”阻塞项：

- live 真正开放
- 自动化真实执行开关
- 更复杂的风险参数编辑器
- GUI 的最终视觉精修
- OpenClaw 全量一体化并入
- learning / hypotheses 的最终成熟版本

这些内容留待主线后续或下一阶段章节推进。

---

## 5. 建议的本章节完成判定

当以下条件成立时，可把本章节视为：

**阶段性完工，可回归主线**

### 满足条件

1. GUI 主结构稳定
2. API 关键路由稳定可用
3. 产品族配置区已具备后续承接能力
4. OpenClaw 入口预埋完成
5. 关键说明文案已达到“人能看懂”的程度
6. 不存在会误导为 live 已开放的高风险表达

---

## 6. 当前建议

如果当前代码和页面已经满足以上大部分条件，则不应继续无限打磨 GUI/API 细节，而应：

- 做少量必要收尾
- 固定结构
- 记录状态
- 回归推进主线

---

## 7. 一句话结论

本章节的正确收尾方式不是“做到绝对完美”，而是“做到结构稳定、预埋齐全、边界清楚、可以安全回主线”。
