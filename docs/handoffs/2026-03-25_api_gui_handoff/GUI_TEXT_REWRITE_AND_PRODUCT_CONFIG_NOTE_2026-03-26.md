# GUI 文案重写与产品族配置区说明（2026-03-26）

## 已确定的改动方向

### 1. 关键动作说明改成人话

后续 GUI 文案统一采用以下原则：

- 直接说“这一步做了什么”
- 直接说“点完后你会看到什么变化”
- 不用过度依赖 `demo_reserved / demo_arm / gate / armed_but_closed` 这种内部术语来解释内部术语

示例：

- `切到 Demo Reserved` 应解释为：
  - 从“完全不走 demo 流程”变成“允许继续做 demo 判断”
  - 不是下单，不是 live，不是立刻获得执行权
- `验证 Demo 前提` 应解释为：
  - 只做检查
  - 看系统现在是否满足继续前进的条件
- `执行 Demo Arm` 应解释为：
  - 系统更接近下一步
  - 但仍未放开执行

### 2. Key Concept Hints 默认折叠

该区域不应一直展开占用主阅读区。

应改成：

- 默认折叠
- 愿意看的人自己展开
- 内容优先解释“事实 / 权限配置 / 状态推进”的区别

### 3. 产品族配置区应独立成卡片

应新增独立卡片：

- `产品族配置 / Product Family Configuration`

初始至少包含：

- spot / 现货产品配置
- margin / 保证金配置
- perp_linear / 线性永续配置
- perp_inverse / 反向永续配置
- options / 期权配置
- other_derivatives_reserved / 其他衍生品（预留）

当前阶段先放：

- enabled
- visible
- mode

后续再接：

- 配置按钮
- 专属产品族设置
- 更细颗粒度说明
