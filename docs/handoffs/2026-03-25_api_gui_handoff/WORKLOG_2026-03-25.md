## 8. 今天采用的工作方式

1. **先验证，后开发**
   - 先做合同与边界确认
   - 再做最小实现
   - 再本地验证
   - 验证通过后再扩展

2. **不直接猜真实 runtime，而是先做桥接层**
   - 先做 runtime snapshot contract
   - 先做 validator / generator / provider
   - 再让 OpenClaw runtime 去适配该 contract

3. **不把 raw JSON 当最终 GUI**
   - raw JSON 只保留为调试与审计区
   - 主界面逐步提升为人类可读控制台

4. **所有高风险动作都维持 protected boundary**
   - 当前不开放真实 live execution
   - 所有 demo / arm / mode 推进都必须 guarded / auditable

5. **发现真实 bug 时先修复再继续开发**
   - validator 枚举 bug
   - runtime bridge overview mode 映射 bug
   - 测试环境变量串扰 bug
   都已按这个原则处理
