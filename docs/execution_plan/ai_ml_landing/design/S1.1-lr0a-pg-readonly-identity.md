## 1. Non-negotiables (verbatim from plan §LR0A)

The Adapter must:

1. connect through an **allowlisted local Unix socket** or **explicitly authenticated loopback** identity;
2. scrub `psqlrc`, service files, shell startup and all ambient `PG*` routing;
3. select a **dedicated read-only PG role** and allowlist exact `SELECT`/catalog queries; reject
   generic SQL text or any write-capable role;
4. bind caller, platform, socket/endpoint, database, query ID, source/schema hashes, result digest,
   observation time and TTL;
5. redact secrets and **prove negative** write / role-escalation / search-path cases.

Direct `psql` remains denied. Production observation is bootstrap-gated to S2.0; **S1.1 does NOT
query production PG**.

## 2. Minimum-coherent design

### 2.1 Shape decision

One new self-contained governance Adapter module that mirrors the cited effect-adapter conventions
(`ADAPTER_ID`, `build_*`/`validate_*` receipt pair, canonical self-digest, `_receipt_schema()`
`lru_cache` loader, `schema_subset_errors` structural check). It is **stateless** (pure functions +
one per-call disposable connection), so no singleton-table registration is required.

It deliberately does **not** register itself into the central AIML closure-validator
(`aiml_gate_receipt_validator.py` `SCHEMA_FILES`/dispatch) nor into the governance
registry/route-compiler. Those are the shared "effect vocabulary / registry / route-compiler /
closure-validator" surfaces that **S1.2 (LR0B)** owns. In S1.1 the Adapter self-validates its own
receipt and is exercised only by its own tests, keeping the S1.1 path manifest disjoint from S1.2.

Language: Python, matching the whole AIML governance surface (`agent_governance_*.py`,
`aiml_gate_receipt_validator.py`) and the two cited adapters. CLAUDE.md "Rust-first" carve-out
("unless the local design clearly says otherwise") applies: this is governance tooling around the
`psql` permission boundary, whose entire neighborhood is Python.

### 2.2 New module: `helper_scripts/maintenance_scripts/agent_governance_pg_readonly_identity.py`

Constants (mirror `agent_governance_effects.py`):

- `ADAPTER_ID = "pg_readonly_identity_adapter_v1"`
- `RECEIPT_SCHEMA_VERSION = "pg_readonly_identity_receipt_v1"`
- `SCHEMA_PATH` → the receipt schema file (see §3 for the directory decision)
- `DIGEST_RE = ^sha256:[0-9a-f]{64}$`, `SQLSTATE_RE = ^[0-9A-Z]{5}$`
- `ENDPOINT_CLASSES = {"unix_socket_allowlisted", "authenticated_loopback"}`
- `TARGET_CLASSES = {"disposable_local", "production"}` — **S1.1 accepts only `disposable_local`**
- `FORBIDDEN_ROLE_ATTRS = ("rolsuper","rolcreaterole","rolcreatedb","rolbypassrls","rolreplication")`
  — every one must be `false` for a RO role
- `DENIAL_SQLSTATES = {"42501","25006",...}` — insufficient-privilege / read-only-txn class; the
  receipt binds the **observed** code from the real disposable server, and the validator asserts
  membership in this denial class (we do not hard-guess one code)
- `ALLOWED_QUERIES: dict[str,str]` — the only executable SQL, keyed by stable `query_id`, catalog/
  `SELECT` only. Minimal S1.1 set:
  - `pg_role_attributes_v1` → `SELECT rolname,rolsuper,rolcreaterole,rolcreatedb,rolcanlogin,rolreplication,rolbypassrls FROM pg_catalog.pg_roles WHERE rolname = current_user`
  - `pg_session_readonly_state_v1` → `SELECT current_setting('transaction_read_only'), current_setting('search_path'), current_user`
  - `pg_server_version_v1` → `SELECT version()`
- `PG_ROUTING_ENV_KEYS` — the ambient routing surface to neutralize: `PGHOST PGHOSTADDR PGPORT
  PGDATABASE PGUSER PGPASSWORD PGPASSFILE PGSERVICE PGSERVICEFILE PGSYSCONFDIR PGOPTIONS PGSSLMODE
  PGREQUIRESSL PGCLIENTENCODING PGCONNECT_TIMEOUT` (and any other `PG*`).

Public functions:

- `receipt_digest(receipt) -> str` — sha256 over every field except `self_digest` (mirrors
  `effect_receipt_digest`).
- `allowed_query_digest(query_id) -> str` — sha256 of the exact allowlisted SQL text, for binding.
- `scrub_pg_environment(base_env=None) -> tuple[dict[str,str], list[str]]` — build a **minimal clean
  env from scratch** (allowlisted keys only, e.g. `PATH`), never inheriting `os.environ` PG routing;
  return the clean env plus the sorted list of dropped/neutralized routing keys. Also isolates
  `psqlrc` by pointing `PGSYSCONFDIR` at an empty temp dir and neutralizing `PGSERVICEFILE`. Pure /
  structurally testable.
- `resolve_allowed_query(query_id) -> str` — return the exact SQL for a known `query_id` or raise;
  **never** accepts caller-supplied SQL text (the function has no free-text path at all).
- `build_readonly_connection_params(*, endpoint_class, socket_dir|loopback_host, port, database,
  role) -> dict` — assemble psycopg2 connect kwargs enforcing: socket-only (`host` = a directory path
  starting with `/`) **or** authenticated loopback (`host ∈ {127.0.0.1, ::1}` with an explicit auth
  method), `options="-c default_transaction_read_only=on -c search_path=pg_catalog"` (pins RO txn +
  search_path at the connection), and the scrubbed env. Rejects any other host form fail-closed.
- `run_readonly_probe(conn_params, *, query_ids, ...) -> ProbeResult` — opens the RO connection with
  `psycopg2` (the repo's driver, v2.9.12), verifies role attributes (all `FORBIDDEN_ROLE_ATTRS`
  false), runs each allowlisted query capturing rows + per-query result digest, then runs the three
  negative probes. Marked `# pragma: no cover` on the live branch per repo convention; driven only by
  the disposable-cluster fixture. Never shells out to `psql`.
- `probe_write_denied(cur) -> dict` — attempt a write (`CREATE TEMP TABLE …` / `INSERT`); expect a
  denial-class SQLSTATE (`42501` insufficient_privilege or `25006` read_only_sql_transaction); record
  `{attempted, observed_sqlstate, verdict:"DENIED"}`.
- `probe_role_escalation_denied(cur) -> dict` — attempt `SET ROLE <writer/superuser>`; expect a
  denial-class SQLSTATE; record verdict.
- `probe_search_path_pinned(cur) -> dict` — attempt `SET search_path TO <malicious>`, then re-read
  `current_setting('search_path')`; expect it either denied or overridden back to the connection-pinned
  `pg_catalog`; record `{attempted, effective_search_path, verdict:"PINNED"|"DENIED"}`.
- `redact_connection_fingerprint(params) -> dict` — emit a **non-secret** endpoint fingerprint (role
  name, database, endpoint_class, socket_dir/loopback_host, port) with **no** password/DSN secret; a
  secret-scan regex (reusing the `GITHUB_SECRET_LIKE_RE` style already in
  `aiml_gate_receipt_validator.py`, extended with `PGPASSWORD`/`password=`/`postgres(ql)?://…:…@`)
  runs over the fully serialized receipt and asserts zero leakage.
- `build_pg_readonly_identity_receipt(*, caller, platform, endpoint, database, role, target_class,
  probe_result, observation_time, ttl_seconds, evidence_class) -> dict` — canonical builder. Sets
  `status="PASS"` **only if** all hold: `target_class=="disposable_local"`; `evidence_class ==
  "LOCAL_REPRODUCIBLE"`; role attrs all false; every allowlisted query executed; all three negative
  probes returned a denial/pinned verdict with a captured SQLSTATE; secret-scan clean;
  `observation_time <= now < expires_at` and `ttl_seconds` within bound. Otherwise `status="FAIL"`
  with a non-empty `failure_reason`. Computes `source_sha256` (this module), `schema_sha256`
  (the receipt schema), `result_digest` (canonical sha256 over all query results), and finally
  `self_digest`.
- `validate_pg_readonly_identity_receipt(receipt, *, require_success=False) -> list[str]` — structural
  + integrity validation mirroring `validate_effect_receipt`: `schema_subset_errors` against the
  schema, exact field-set match, `adapter_id`/`schema_version` consts, digest regexes,
  endpoint_class/target_class enums, **`target_class` must be `disposable_local`** at the S1.1 gate
  (production is rejected fail-closed), role-attr negative facts present and all false, three
  negative-case records present with denial verdicts + SQLSTATE captured, secret-scan `leaked==false`,
  TTL/time ordering, and `self_digest` re-derivation.

### 2.3 How each non-negotiable is enforced

| Requirement | Enforcement point |
|---|---|
| socket-only / authenticated loopback | `build_readonly_connection_params` accepts only a `/`-prefixed socket dir or `{127.0.0.1,::1}` loopback with explicit auth; all else fail-closed. Disposable cluster runs with `listen_addresses=''` (no TCP at all). |
| scrub `psqlrc` / service / shell / `PG*` | `scrub_pg_environment` builds a clean env from scratch, drops every `PG*` routing key, isolates `PGSYSCONFDIR` (kills system/user `psqlrc`) and neutralizes `PGSERVICEFILE`. Recorded as `ambient_routing_scrubbed` in the receipt. |
| dedicated RO role + no write-capable role | `run_readonly_probe` verifies `FORBIDDEN_ROLE_ATTRS` all false via `pg_role_attributes_v1`; any true → FAIL. |
| exact SELECT/catalog allowlist, no generic SQL | `resolve_allowed_query` has no free-text path; only `ALLOWED_QUERIES` values execute; each query's `query_id`+`query_sha256` bound in the receipt. |
| bind caller/platform/endpoint/db/query-id/hashes/digest/time/TTL | receipt fields (see §4). |
| redact secrets | `redact_connection_fingerprint` + secret-scan over serialized receipt; `secret_scan.leaked==false` required for PASS. |
| prove negative write/role/search-path | three `probe_*` functions against the **real** disposable server capture actual SQLSTATE; verdicts required for PASS. |

## 4. `pg_readonly_identity_receipt_v1` schema shape

Draft 2020-12, `additionalProperties:false`, `$defs` for `digest`/`timestamp`/`sqlstate`, matching
the existing `.schema.json` style (`effect_adapter_result_v1`, `aiml_required_effect_classification_v1`).
Required top-level fields:

- `schema_version` — const `"pg_readonly_identity_receipt_v1"`
- `adapter_id` — const `"pg_readonly_identity_adapter_v1"`
- `status` — enum `["PASS","FAIL"]`
- `caller` — string ≥1 (governed caller/role id)
- `platform` — object `{os: enum[darwin,linux], arch: string, postgres_version: string}` (pins reproducibility)
- `target_class` — enum `["disposable_local","production"]` (S1.1 validator requires `disposable_local`)
- `evidence_class` — enum `["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"]` (honesty marker; PASS ⇒ LOCAL_REPRODUCIBLE)
- `endpoint` — object `{endpoint_class: enum[unix_socket_allowlisted,authenticated_loopback], socket_dir: str|null, loopback_host: str|null, port: int|null, allowlisted: const true}`
- `database` — string ≥1
- `role` — object `{name: string, attributes: {rolsuper:false, rolcreaterole:false, rolcreatedb:false, rolbypassrls:false, rolreplication:false, rolcanlogin:true}, is_read_only: const true}`
- `ambient_routing_scrubbed` — object `{dropped_env_keys: [string…], psqlrc_neutralized: const true, service_file_neutralized: const true, pgsysconfdir_isolated: const true, effective_env_has_no_pg_routing: const true}`
- `queries` — array (≥1) of `{query_id: string, query_sha256: digest, row_count: int≥0, result_digest: digest}` (each allowlisted query executed)
- `result_digest` — digest (canonical sha256 over all query results; the plan's single "result digest")
- `negative_cases` — object with three required records:
  - `write_denied` — `{attempted: string, expected_denial: const true, observed_sqlstate: sqlstate, verdict: const "DENIED"}`
  - `role_escalation_denied` — `{attempted, expected_denial: const true, observed_sqlstate, verdict: const "DENIED"}`
  - `search_path_pinned` — `{attempted, effective_search_path: string, pinned: const true, verdict: enum["PINNED","DENIED"]}`
- `source_sha256` — digest (Adapter module source)
- `schema_sha256` — digest (this schema file)
- `secret_scan` — object `{patterns_checked: [string…], leaked: const false}`
- `observation_time` — timestamp
- `expires_at` — timestamp
- `ttl_seconds` — integer > 0, bounded (proposed ≤ 3600 — see §8.5)
- `failure_reason` — `string|null`
- `self_digest` — digest

`allOf` conditional (mirrors `effect_adapter_result_v1`): if `status=="PASS"` then
`evidence_class=="LOCAL_REPRODUCIBLE"`, `target_class=="disposable_local"`, `failure_reason==null`,
all negative verdicts DENIED/PINNED; else `failure_reason` is a non-empty string.

`sqlstate` `$def`: `{type:string, pattern:"^[0-9A-Z]{5}$"}`.

Reproducibility: `source_sha256` + `schema_sha256` + `platform.postgres_version` + the deterministic
seeded fixture ⇒ the receipt (including `result_digest` and negative verdicts) is independently
re-derivable, satisfying plan §LR0A line 213 ("independently reproducible").

## 5. Disposable-proof strategy (evidence-class honesty)

**Local binary check (run 2026-07-22 on this Mac):**

```
$ which initdb pg_ctl postgres
/opt/homebrew/bin/initdb
/opt/homebrew/bin/pg_ctl
/opt/homebrew/bin/postgres
$ pg_config --version
PostgreSQL 16.14 (Homebrew)
$ python3 -c "import psycopg2" → psycopg2 2.9.12 OK
```

`initdb` / `pg_ctl` / `postgres` **and** the repo driver `psycopg2` are present ⇒ plan option (a)
(ephemeral disposable cluster) is feasible locally. There is **no** existing disposable-cluster
fixture in the repo; S1.1 introduces the first (kept test-local, no shared `conftest.py`).

### (a) `LOCAL_REPRODUCIBLE` — ephemeral disposable local PG cluster [the crux]

A pytest **session fixture** (inside the E4 local-test-adapter allowlist because the governed command
stays `pytest`; `initdb`/`pg_ctl`/`postgres` are spawned as subprocesses of the test process):

1. `initdb` a throwaway cluster into a `tempfile` dir.
2. Start via `pg_ctl` with `unix_socket_directories=<tmp>/sock`, **`listen_addresses=''`** (socket-only,
   zero TCP surface).
3. Seed a fixed tiny dataset; `CREATE ROLE aiml_ro LOGIN` granted only `SELECT` on a fixed schema, plus
   a separate `aiml_writer` role as the escalation target.
4. The Adapter connects over the unix socket as `aiml_ro` and runs role-attr verification, the
   allowlisted SELECTs, and the three negative probes.
5. Teardown: `pg_ctl stop -m immediate` + `rmtree`.

Because this is a **real `postgres` process, not a mock**, it emits **genuine** PG runtime semantics —
real SQLSTATE for write denial (`42501`/`25006`), `SET ROLE` denial, and connection-pinned
`search_path`. The CLAUDE.md caveat "Mac mocked tests cannot establish PG runtime semantics" does
**not** apply here: nothing is mocked. Gated by `shutil.which("initdb")`; where PG binaries are absent
the disposable module **skips** (never a false PASS). **This is the only path that emits a
`status=PASS`, `target_class=disposable_local`, `evidence_class=LOCAL_REPRODUCIBLE` receipt.**

Proves: socket-only connection, RO-role selection, SELECT/catalog allowlist, and the negative
write/role/search-path denials — against real PG. Does **not** prove production PG's roles/ACLs.

### (b) `STRUCTURAL_ONLY` — hermetic pure-Python tests (no server)

`scrub_pg_environment` key-dropping; `resolve_allowed_query` rejecting raw SQL and unknown ids; the
redaction/secret-scan catching an injected secret; builder/validator digest binding, exact field-set,
TTL/time-order, PASS/FAIL conditionals, and the **fail-closed rejection of `target_class="production"`
at the S1.1 gate**. No PG; runs everywhere including CI without PG. These paths must **not** emit a
runtime negative-case verdict (they may only exercise builder/validator logic).

### (c) CI-deferred

**None is strictly required for the S1.1 gate**, because (a) runs locally and yields LOCAL_REPRODUCIBLE
evidence. Optional hardening: run the same disposable-cluster proof in Linux CI with a `postgres`
service for aarch64/x86 cross-platform reproducibility — a nice-to-have, not the gate.
**Do not conflate:** production-PG negative-ACL proof is **S2.0-gated by the plan** (lines 216-221),
not "CI-deferred."

### Evidence summary (no overstatement)

| Behaviour | Evidence class | Where |
|---|---|---|
| write / role-escalation / search-path denial; RO-role selection; socket-only connect | **LOCAL_REPRODUCIBLE** (real disposable PG) | disposable test |
| `psqlrc`/`PG*` scrub; SQL allowlist; secret redaction; builder/validator digest, TTL, field-set; `production` rejection | **STRUCTURAL_ONLY** | structural test |
| production-PG roles/ACLs | not in S1.1 — **S2.0-gated** | — |
| cross-platform disposable proof | optional CI hardening | (deferred) |

S1.1 emits exactly **one** disposable `pg_readonly_identity_receipt_v1`
(`target_class=disposable_local`, `evidence_class=LOCAL_REPRODUCIBLE`). It is the **Adapter-mechanism**
proof — reproducible from pinned source + schema + PG major + deterministic fixture. It is explicitly
**not** a production observation and **not** authority to query production PG.

## 7. Boundary compliance & fail-closed posture

- **No production PG** — the validator rejects `target_class="production"` at the S1.1 gate; only
  `disposable_local` can PASS. Production observation is S2.0.
- **No direct `psql`** — the Adapter connects via `psycopg2` over the socket and never shells `psql`;
  the `permissions.py` `psql` denial is left in force. The disposable cluster is driven from within
  `pytest`, so the governed command stays `pytest` (E4 local-test-adapter allowlist) — no new
  permission grant.
- **No migration** — the disposable fixture's `CREATE ROLE`/`CREATE TABLE` is throwaway-cluster
  scaffolding on a temp dir, never a repo migration and never production. The Adapter never applies an
  AIML migration and never creates a writer (plan §LR0A line 220-221).
- **`DISPOSABLE_ONLY`** — temp-dir cluster, torn down; the only artifact is the reproducible receipt
  JSON.
- **Fail-closed defaults** — unknown/absent `query_id` → reject; any write-capable role attr → FAIL;
  any negative probe not returning a denial verdict → FAIL; any secret detected → FAIL; `production`
  target → reject; missing PG binaries → **skip** (never a false PASS).
