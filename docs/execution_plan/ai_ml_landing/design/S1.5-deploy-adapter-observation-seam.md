## 0. The overriding constraints — three frozen/cautioned surfaces, one airtight boundary

S1.5 is the **most effectful** S1 session, so the constraints are stated first.

1. **No production apply — machine-forced, not promised.** Every receipt binds `target_class` with an
   enum whose only PASS value is `disposable_local`; `production` is fail-closed (raise). Every receipt
   binds `production_apply_performed / real_service_restart / real_remote_host_mutation` as **const
   `false`**. The disposable apply is a real filesystem / real disposable-PG mutation on a **temp
   dir / throwaway cluster**, torn down after — never a systemctl call, never a real host, never a real
   migration/credential slot. This mirrors S1.1/S1.3/S1.4's schema-enforced disposable gate.

2. **Frozen S0.3 is untouched (same discipline as S1.2 §0).** `validate_program_adoption_receipt`
   **re-derives** `aiml_effect_classifier_digest()`, `classify_required_effects()` (S0.3),
   `terminal_receipt_sink_contract()`, `PROGRAM_*_PATHS`, and the frozen-contract branch on every
   adoption re-validation. **S1.5 mutates none of those.** Verified adoption-safety: the adoption gate's
   `registry_contract_errors` checks `canonical_validator_path` by **existence only** (`is_file()`,
   `agent_governance_aiml_adoption.py:130`), never by content hash — so additive edits to the central
   validator file do not break `PROGRAM_ADOPTED`.

3. **The S1.2 component-effect matrix is cautioned but is S1.5's designated property.** The matrix
   digest `aiml_component_effect_class_matrix_digest()` is deliberately **separate** from S0.3 (S1.2 §0),
   so editing the matrix does **not** touch `PROGRAM_ADOPTED`. S1.5 is the *named owner* of the six
   `PENDING_S1_5` markers. The only matrix edit S1.5 makes is flipping those six markers to
   `IMPLEMENTED_DISPOSABLE` — exactly as S1.2 set `TERMINAL_RECEIPT_APPEND` to `IMPLEMENTED_DISPOSABLE`
   when its adapter landed (§8.2). It touches **no** other matrix field and **no** S0.3 structure.

## 2. Minimum-coherent design — ONE data-driven adapter, extending existing infra

Mirroring S1.2's decisive insight (the 7-class matrix is *data-driven*, not seven hand-coded adapters),
S1.5 builds **one** self-contained module, not six parallel ones — the six classes share a single
apply/rollback/postcheck skeleton and differ only by (a) matrix-derived intent fields and (b) a
**disposable-target kind** (§4). A second "adapter" that differs only in data fails the deletion test.

New module `helper_scripts/maintenance_scripts/agent_governance_component_effects.py`, a purpose-built
sibling of `agent_governance_p0b_effects.py` (parallel filename, parallel role: a per-purpose effect
adapter with its own intent/result/postcheck/closure validators). It:

- **derives** each class's exact-intent contract from `AIML_COMPONENT_EFFECT_CLASS_MATRIX`
  (`required_intent_fields`, `adapter_id`, `actor_node_id`, `independent_postcheck_node_id`,
  `recovery_contract`) — the caller cannot supply or downgrade them (reuses the S1.2 classifier);
- **dispatches** the disposable apply/rollback/postcheck by `disposable_target_kind` (§4.1–§4.3);
- **consumes** S1.3 `agent_governance_identity_acl_contract` (for the PG-role/ACL + credential classes'
  contract validation) and S1.4 `agent_governance_runtime_candidate_spike.hash_bundle_tree` (for the
  learning-runtime bundle digest) as **read-only imports** (DRY, no edit to S1.3/S1.4);
- reuses the S1.2 WORM atomic primitive (`os.link` / `os.replace` + `chmod 0o444` immutable +
  `0o700` private store + `ROLLED_BACK_INTERRUPTED` on interruption) for the artifact-swap target;
- is **stateless** (pure functions + per-call disposable connection/temp-dir), so **no
  singleton-table registration**.

Language: **Python, stdlib-first** (`hashlib`/`json`/`os`/`shutil`/`tempfile`/`stat`), matching the whole
AIML governance surface and both cited adapters; `psycopg2` is a **lazy** import used only in the
`initdb`-gated disposable path (S1.1/S1.3 precedent). CLAUDE §七 "Rust-first" carve-out ("unless the local
design clearly says otherwise") applies — this is governance tooling in a Python-only neighborhood, the
same carve-out PM accepted for S1.1–S1.4. **Portability (CLAUDE §六):** `REPO_ROOT =
Path(__file__).resolve().parents[2]`; canonical production paths (`/opt/aiml/...`, unit names) are
**recorded as design data**, never a path the Mac spike writes to. No `/Users/ncyu` / `/home/ncyu` literal.

The four seams S1.5 wires are each an **extension of an existing surface**:

1. **Adapter mechanism** → new module (above), following `agent_governance_p0b_effects.py`.
2. **Registry** → six additive `effect_adapters` entries (§8.1) — *registry-JSON-only*, no adoption-gate edit.
3. **Effect vocabulary** → flip six matrix `adapter_binding_status` markers (§8.2).
4. **Schema + closure-validator recognition** → four schemas + four additive central-validator dispatch
   branches (§9), mirroring the S1.2 WORM branch.

## 3. The typed intent contract per class (derived, not hand-declared)

Each class's `component_effect_intent_v1` binds **exactly** the matrix `required_intent_fields` for that
class (the plan-table col-2 recovery fields), plus the invariant governance obligations. The Adapter
looks the fields up in the matrix; a caller that omits, adds, or renames a field is rejected (reuse of
`classify_component_required_effects`). Illustration for two classes:

- `PG_ROLE_ACL_MIGRATION`: `migration_id, migration_checksum, role_acl_delta, pre_state_digest,
  transactional_or_double_apply, recovery` — and the `role_acl_delta` must **conform to the S1.3
  `pg_role_topology`** (validated by importing `agent_governance_identity_acl_contract`, exactly the
  "consumer adapter binds S1.3's contract" direction S1.3 §1.1/§6 specified).
- `ENGINE_SCANNER`: `binary_digest, unit, env_digest, config_digest, stop_start_order,
  readiness_deadman_checks, prior_bundle_rollback` — with `prior_bundle_rollback` bound to the
  pre-state bundle digest (the rollback target).

Every intent additionally carries the matrix invariants (`requires_ops_preflight`,
`requires_pm_operator_approved_intent`, `requires_independent_ops_postcheck`,
`applier_is_not_sole_verifier`), `target_class` (disposable_local only), `approved_by`/`approved_at`/
`typed_confirm`/`expires_at`, an `ops_preflight` binding, and a **rollback binding** (`recovery_contract`
+ `pre_state_digest`). Absence of preflight, approval, or rollback binding is a bypass-negative (§11).

## 4. The disposable apply / rollback / postcheck mechanism (the crux)

Common lifecycle for every class (an extension of the deploy/p0b OPS-preflight→apply→OPS-postcheck shape,
generalized with an explicit **rollback** and an **exact-restoration** proof):

```
pre-state digest  →  OPS preflight  →  approved intent  →  APPLY (real, disposable)
   →  stable observation window (prove APPLIED)  →  ROLLBACK (restore prior)
   →  post-rollback digest  →  assert  pre-state-digest == post-rollback-digest   [exact restoration]
   →  INDEPENDENT postcheck by a DISTINCT actor node (re-derives digest, confirms restoration)
```

Two load-bearing invariants, both machine-checked:

- **Exact restoration** — `pre_state_digest == post_rollback_digest` (byte-identical). A `component_effect_result_v1`
  claiming `rolled_back` but with unequal digests is rejected (`rollback_not_exact` bypass-negative).
  Crucially, `APPLIED_ROLLED_BACK_EXACT` **additionally** requires the apply to have CHANGED state —
  `applied_digest` must be present and `!= pre_state_digest` — so a no-op/identity apply
  (`applied == pre == post`) cannot satisfy `pre == post` vacuously (`apply_was_a_noop` bypass-negative,
  §11-12). This guard is scoped to `APPLIED_ROLLED_BACK_EXACT`; `ROLLED_BACK_INTERRUPTED`'s
  `applied == pre` remains legitimate.
- **Applier is not sole verifier** — the `component_effect_postcheck_attestation_v1.verifier_node` must
  differ from `component_effect_result_v1.apply_actor_node`; equal ⇒ refused
  (`applier_is_sole_verifier` bypass-negative). This is the same distinctness S1.2 enforced
  (`readback_verifier_id != append_actor_id`) and p0b enforced (`ops_postcheck` distinct node). The
  postcheck is made **load-bearing**, not decorative: `restoration_confirmed` must equal
  (`reobserved_post_rollback_digest == result.post_rollback_digest`) at attestation-build time (a
  verifier that cannot re-derive the exact post digest cannot claim confirmed), and the rollup entry
  carries `reobserved_post_rollback_digest` which both `build_admitted_class_entry` and the rollup
  **validator** re-check equals `post_rollback_digest` (so the consumer's gate, not only the builder,
  enforces that the independent verifier actually confirmed the exact restoration).

The apply/rollback dispatches to one of **three disposable-target kinds**:

### 4.1 `disposable_pg` — real throwaway PG cluster (PG_ROLE_ACL_MIGRATION, CREDENTIAL_ROTATION, and CONTROLLER_WORKERS' pg-role facet)

Reuses the S1.1/S1.3 `initdb`/`pg_ctl`/`listen_addresses=''`/socket-only pattern (spawned inside `pytest`,
so the governed command stays `pytest`; `psql` denial untouched):

- **PG_ROLE_ACL_MIGRATION**: pre-state = digest of the role/ACL/grant catalog projection → apply the
  `role_acl_delta` (`CREATE ROLE` / `GRANT`, real SQL) → observe the grant present → rollback
  (`REVOKE` / `DROP ROLE`) → post-state digest **== pre-state**. Independent verifier (distinct
  connection/actor) re-reads and confirms restoration **and** that the least-privilege reader is still
  write-denied **`42501`** (real SQLSTATE, nothing mocked).
- **CREDENTIAL_ROTATION**: pre-state = the scram credential fingerprint → apply `ALTER ROLE … PASSWORD`
  A→B → observe B connects → rollback B→A → post-state **== pre-state**. Independent verifier confirms
  the **old** credential B is rejected **`28P01`** after rollback and A reconnects (reuses the S1.3
  disposable rotation proof; fingerprints are `sha256` of throwaway A/B — non-secret).

The disposable cluster pins `log_statement='none'` + `log_min_duration_statement=-1`: PostgreSQL does
**not** redact `ALTER ROLE … PASSWORD` in statement logs, so the disposable proof pins the no-plaintext-in-
statement-log invariant that S2.4's real credential slot must carry. The intent-side S1.3 conformance is
**exercised** (not merely imported): a `PG_ROLE_ACL_MIGRATION` `role_acl_delta` runs through S1.3
`assert_least_privilege_topology` (a superuser / reader-with-write role is rejected) and a
`CREDENTIAL_ROTATION` intent runs the S1.3 rotation-order check + `credential_slot_fingerprint` equality
(an unsafe rotation order or mis-derived slot fingerprint is rejected).

Gated on `shutil.which("initdb")`; where PG binaries are absent the disposable facet **skips** (never a
false PASS), and that class's `evidence_class` degrades to `STRUCTURAL_ONLY`.

### 4.2 `temp_dir_artifact` — real generation-pointer / throwaway-unit swap (ENGINE_SCANNER, LEARNING_RUNTIME, CONTROLLER_WORKERS' unit facet)

A `0o700` temp-dir "deploy root" holding a **bundle store** `bundles/<sha256>/` and an
**active-generation pointer** file (content = the active bundle's content hash), plus a throwaway
`.service`-shaped unit text file for the ENGINE_SCANNER/CONTROLLER facet (a **file**, never a real unit):

- pre-state = digest of `{active-pointer content, bundle inventory, unit-file bytes}`;
- **apply** = stage a new bundle (write files; hash with S1.4 `hash_bundle_tree`), **atomically** swap the
  pointer via `os.replace` (the atomic boundary — S1.2 WORM precedent), `chmod 0o444` the new bundle files
  (immutable-after-write); observe the pointer now names the new hash;
- **rollback** = atomically swap the pointer **back** to the prior bundle hash (the prior bundle is
  retained — the matrix `prior_bundle_rollback` / `exact_rollback` contract); an interrupted apply that
  never swapped the pointer leaves the prior generation active (`ROLLED_BACK_INTERRUPTED`, S1.2 pattern);
- post-rollback digest **== pre-state**; independent verifier re-reads the deploy root and confirms
  pointer==prior-hash + digest match.

Filesystem confinement is fail-closed: bundle-file keys (`dict[str, bytes]`) and object restore rels are
resolved through a `_within`-style guard that rejects an absolute key, a `..` escape, a symlink-escape, or
a NUL/empty key **before any write** — the exact rel the retention deleter rejects, the restore path also
rejects — and the active-generation pointer / `prior_hash` are confined to `^[0-9a-f]{64}$` so a `../`
pointer can never make `artifact_state_digest` hash an arbitrary directory.

`LEARNING_RUNTIME` additionally binds the S1.4 candidate: the bundle is a content-addressed fixed-path
runtime whose launch manifest pins the exact `<sha256>` (no mutable `latest` symlink — S1.4 §LR0C-272).
**This proves the Adapter mechanism + exact rollback; it is NOT a real systemd unit or a running
process** (that is S1.6/S2.5). Runs everywhere (pure stdlib).

### 4.3 `temp_dir_objects` — real object delete + restore (RETENTION_APPLY)

A temp dir of throwaway "objects" (files) + a throwaway "restore-capacity" copy:

- pre-state = object-set digest → **apply** = tombstone-delete the exact declared object set (real
  `os.unlink`), observe deleted → **rollback / interruption-recovery** = restore from the throwaway
  copy (real copy-back) → post-restore digest **== pre-state**; independent verifier confirms restoration
  and that the deleter identity could not exceed the tombstone set. Runs everywhere (pure stdlib).

## 5. The remote/platform observation / attestation seam

The seam is the mechanism by which an **independent observer attests** the post-apply and post-rollback
state — the plan's "independent remote/platform attestation" + "applier cannot be its sole verifier".

**S1.5 defines and disposably proves the STRUCTURE:**

- `component_effect_postcheck_attestation_v1` is produced by a **distinct `verifier_node`** that
  re-observes the disposable target (re-reads the temp dir / disposable cluster) and **independently
  re-derives** the state digest — it does not copy the applier's claimed digest. It records
  `verifier_node != apply_actor_node`, `restoration_confirmed`, `applier_is_not_sole_verifier=true`,
  and `remote_platform_attested=false`. This is the same independent-observer shape as the deploy
  `build_ops_evidence` postcheck and p0b's `ops_postcheck` / observer-v2.

**S1.5 is honest that the real remote/platform attestation is DEFERRED (mirrors S1.3 §4.1):**

- A `LOCAL_REPRODUCIBLE` disposable attestation **cannot self-authenticate its own execution** (CLAUDE
  Typed Authority Matrix: a canonical self-digest proves integrity, never who executed what). The
  distinct in-process verifier proves the *structural* independence rule, not cryptographic actor
  authenticity or a real remote observation.
- The **real** remote/platform-attested observation — a real remote host / systemd / platform emitting a
  signed running/health attestation over a transport — is **DEFERRED to S2.5** (`WATCHDOG_ROLLBACK_TEST`
  / running attestation) and the **S1.6** target-host probe. S1.5 makes **no** network contact and
  touches **no** remote host.
- **Consumer contract:** any S2.4/S2.5 consumer relying on a `LOCAL_REPRODUCIBLE` component-effect receipt
  **MUST re-run the disposable proof or obtain platform attestation** before trusting it; it may not
  trust the receipt's `evidence_class` label alone.

## 6. The honest disposable-vs-deferred split (per class)

`LOCAL_REPRODUCIBLE` = real bytes recapturable on this Mac; `STRUCTURAL_ONLY` = contract + structural
negative, no runtime bytes; `DEFERRED_S1_6/S2` = requires a real host/service/remote mutation.

| Class | Disposable target | What is REAL on this Mac (LOCAL_REPRODUCIBLE) | What DEFERS |
|---|---|---|---|
| `PG_ROLE_ACL_MIGRATION` | `disposable_pg` | real `CREATE ROLE`/`GRANT` apply, `REVOKE`/`DROP` rollback, pre==post digest, reader still `42501`-denied by distinct verifier | real **production migration/role/ACL** → S2.0/S2.4 |
| `CREDENTIAL_ROTATION` | `disposable_pg` | real `ALTER ROLE PASSWORD` A→B→A, old-cred `28P01` rejection by distinct verifier | real **production credential slot / secret manager** → S2.4 |
| `RETENTION_APPLY` | `temp_dir_objects` | real `os.unlink` delete + real restore, pre==post digest, distinct verifier | real **production object store / deleter identity** → S3.3/S2.4 |
| `ENGINE_SCANNER` | `temp_dir_artifact` | real bundle stage + atomic pointer swap + rollback, pre==post digest, `0o444` immutable, distinct verifier | real **systemctl / binary+unit swap / running attestation** → S1.6 + S2.5 |
| `CONTROLLER_WORKERS` | `temp_dir_artifact` (+ `disposable_pg` for the pg-role facet) | real pointer/queue-fence artifact swap + rollback + distinct verifier; real disposable pg-role | real **cgroup/slice/UID / running workers** → S2.4 + S1.6 |
| `LEARNING_RUNTIME` | `temp_dir_artifact` | real content-addressed bundle stage (S1.4 hash) + atomic pin swap + rollback + distinct verifier | real **runtime start/stop / native-lib loading / cgroup / PG identity** → S1.6 + S2.5 |

**Honest headline:** the facets that are *properties of PG semantics and filesystem atomicity*
(role/ACL/rotation, object delete+restore, content-addressed pointer swap with exact rollback) get **real
disposable `LOCAL_REPRODUCIBLE`** evidence — a real `postgres` process emits genuine `42501`/`28P01`, and
real `os.replace`/`os.unlink`/`chmod` emit genuine state transitions (nothing mocked; the CLAUDE "Mac
mocked tests cannot establish PG semantics" caveat does not apply). The facets that are *real host/service
acts* (systemctl start/stop, cgroup/slice/UID, running attestation, production migration/credential slots,
remote-host mutation, platform-attested telemetry) are honestly **DEFERRED to S1.6/S2.4/S2.5** — never
faked. This split is the deliverable: it prevents the classic spike failure of promoting "I proved the
disposable half" into "the component is deployed on the host."

## 7. `effect_seams_ready_receipt_v1` schema shape (a rollup; what it asserts vs S1.6)

Draft 2020-12, `additionalProperties:false`, `$defs` for `digest`/`timestamp`, frozen `RECEIPT_FIELDS`
set validated exactly, mirroring the S1.1–S1.4 receipt style. It is a **thin rollup** binding the
per-class result + postcheck receipts **by digest** (as `runtime_candidate_comparison_v1` binds the two
candidate receipts) — it does not inline them. Top-level fields:

### 7.1 Governance-wiring block (proves "not contract prose alone")

- `governance_wiring` — `{component_effect_matrix_digest (== aiml_component_effect_class_matrix_digest()),
  registered_adapter_ids: [the six], schema_files_registered: [the four new schema_versions],
  central_validator_recognizes: const true, generated_views_unchanged: const true}` — the first-class

## 11. Bypass-negative test plan (fail-closed — the load-bearing property)

"The generic deploy apply must stay DISABLED for a class without its per-class contract; a session cannot
route an effectful class as source-only; applying without preflight/approved-intent/rollback binding must
fail closed." Cases (hermetic unless marked disposable), each recorded in the receipt as `REJECTED`:

1. `source_only_route_of_effectful_class` — a component work-package touching a surface but declaring
   `effect_class=NONE`/omitting the class → `classify_component_required_effects` **raises** (reuse the
   S1.2 mechanism, re-asserted for the six deploy classes).
2. `generic_deploy_apply_enabled_without_per_class_contract` — assert `deploy_adapter_v1` status stays
   `declared_apply_disabled_until_recovery_controls_bound`; the generic adapter cannot apply a component
   class.
3. `apply_without_preflight` — apply without a valid OPS preflight binding → refuse/FAIL.
4. `apply_without_approved_intent` — apply without a PM/operator-approved intent
   (missing `approved_by`/`typed_confirm`) → refuse/FAIL.
5. `apply_without_rollback_binding` — apply without a bound `recovery_contract`+`pre_state_digest` →
   refuse/FAIL (CLAUDE §八 "generic deploy apply remains disabled … until exact rollback binding … bound").
6. `production_target` — `target_class=production` at the S1.5 gate → fail-closed **raise**.
7. `applier_is_sole_verifier` — postcheck `verifier_node == apply_actor_node` → refused.
8. `rollback_not_exact` — a result with `rolled_back=true` but `post_rollback_digest != pre_state_digest`
   → rejected (the exact-restoration crux is not fakeable).
9. `cross_class_adapter_substitution` — wrong `adapter_id` for the class → rejected (S1.2 classifier).
10. `classifier_or_matrix_digest_tamper` — `classifier_digest != aiml_component_effect_class_matrix_digest()`
    → rejected.
11. `plaintext_secret_ingress` — a secret-like value anywhere in a serialized intent/result/receipt →
    refuse to serialize (`GITHUB_SECRET_LIKE_RE`-style scan, extended per S1.1/S1.3).
12. `apply_was_a_noop` — an `APPLIED_ROLLED_BACK_EXACT` result whose `applied_digest` is absent or equals
    `pre_state_digest` (the apply never changed the active state) → rejected. Without this, an identity
    apply (`applied == pre == post`) would trivially satisfy `pre == post` and "prove" exact restoration
    vacuously. The guard is scoped to `APPLIED_ROLLED_BACK_EXACT` ONLY; `ROLLED_BACK_INTERRUPTED`'s
    `applied == pre` stays legitimate (the pointer never swapped, the prior generation stayed active).

Plus the positive **disposable** proofs (§4, one real apply/rollback/postcheck per class, distinct
verifier) and the standard receipt-integrity suite (field-set, digest re-derivation, TTL/time order,
`production` fail-closed, `production_apply_performed`/`real_service_restart` machine-false, `secret_scan`).
