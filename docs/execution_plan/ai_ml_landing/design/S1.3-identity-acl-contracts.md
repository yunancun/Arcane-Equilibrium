## 0. The overriding boundary — S1.3 is a CONTRACT, not a provisioner

The single most important framing: the plan splits **owning the identity/ACL contract** (LR0B, this
Session) from **applying it to the production host** (S2.0 observer bootstrap; S2.4
`CREDENTIAL_PG_UNIT_INSTALL`). S1.3 sits entirely on the contract side. Every "provision" verb in the
requirement is, at S1.3, a **declarative specification + a proof that a bad specification is rejected**,
never a real host mutation. This mirrors:

- **S1.1**, whose disposable `CREATE ROLE` is "throwaway-cluster scaffolding … never production", with
  production observation "bootstrap-gated to S2.0"; and
- **S1.4**, whose runtime-only seams are schema-forced `DEFERRED_S1_6` so the boundary is *machine-
  enforced*, not merely promised.

S1.3 adopts the identical discipline: real host-UID/role/ACL mutation is **machine-forced absent** from
any PASS receipt (`production_*_provisioned` consts are all `false`), and the crux deliverable — that an
over-granting contract FAILS closed — is proven with real disposable evidence where possible and honest
structural evidence otherwise. A receipt that claimed a real production identity mutation would be
faking evidence (CLAUDE §四).

## 1. What the identity/ACL slice requires, and the exact S1.3 / S2.0 / S2.4 split

Plan §LR0B-243 + §LR2-306 demand a least-privilege identity/ACL topology with seven facets. The split:

| Facet | S1.3 (this Session) | Deferred to |
|---|---|---|
| **Separate host UIDs** per component (distinct, non-root, no OCI socket, no DBus) | declarative topology + **structural** negatives | real `useradd`/UID = **S2.4** (needs root) |
| **Dedicated PG roles** per component (least-privilege, no shared writer-for-reader, no superuser) | topology + **disposable-real** role/ACL negatives (initdb) | real production roles = **S2.0/S2.4** |
| **Auth mapping** (`pg_hba`/`pg_ident` or authenticated loopback, local-only, no trust-from-anywhere) | policy contract + **structural** trust/wide-CIDR rejection; **optional** disposable password-denial | real `pg_hba.conf` install/reload = **S2.0/S2.4** |
| **Socket-dir ACLs** (private mode, not world-readable/writable) | **disposable-real** `chmod`/`stat` on temp dirs | real runtime socket dirs = **S2.4** |
| **Secret lifecycle** (rotation slot/order, old/new non-secret fingerprints, old-credential rejection, protected loading, no plaintext ingress) | rotation contract + **disposable-real** old-credential rejection (initdb) | real production credential slot = **S2.4** |
| **Negative ACL tests** (over-grant must FAIL closed) | **the crux** — hermetic + disposable | — |
| **Exact rollback** per identity/ACL change | per-change rollback contract (pre-state digest + action) | real rollback drill = **S2.4/S2.5** |

S1.3 owns the whole left column. It is the least-privilege *vocabulary of a valid identity/ACL
topology*; S2.0/S2.4 are the effect Sessions that provision it on the real host and run the real
negative postchecks (plan §LR0A-216, §LR3-327 "prove denied actions").

### 1.1 How S1.3 aligns with the S1.2 matrix — by construction, not by editing it

The S1.2 `AIML_COMPONENT_EFFECT_CLASS_MATRIX` already defines the *intent-field vocabulary* for the
classes that will consume S1.3's contract in Sprint 2. S1.3's contract mirrors those field names
**verbatim** so the two can never drift, **without touching the matrix**:

| S1.2 matrix class | Matrix `required_intent_fields` (frozen by S1.2's digest) | S1.3 contract facet that defines their valid space |
|---|---|---|
| `CREDENTIAL_ROTATION` | `secret_slot_target, role_target, old_fingerprint, new_fingerprint, rotation_order, old_credential_rejection_proof` | `secret_lifecycle.rotation` uses these **exact** keys |
| `PG_ROLE_ACL_MIGRATION` | `migration_id, migration_checksum, role_acl_delta, pre_state_digest, transactional_or_double_apply, recovery` | `pg_role_topology` (the valid `role_acl_delta` space) + `rollback[*].{pre_state_digest,recovery}` |
| `CONTROLLER_WORKERS` | `unit_slice_cgroup_uid_pgrole_set, queue_fencing_state, start_order, drain_rollback` | `host_uid_topology` + `pg_role_topology` per component **is** the `uid_pgrole_set` |

The reference direction is: a future S2.4 intent's `role_acl_delta` / `role_target` /
`unit_slice_cgroup_uid_pgrole_set` must **conform to** the S1.3 contract. That binding is enforced by
the **adapter that validates the intent** (built in S1.5/S2.4), not by the matrix. See §6 for why S1.3
must **not** modify the matrix.

## 2. Minimum-coherent design — one self-validating, disjoint contract module

Shape decision (mirrors S1.1/S1.4 exactly): **one new self-contained governance module** with the
`ADAPTER_ID` / `build_*`+`validate_*` receipt pair / canonical `self_digest` / `_receipt_schema()`
`lru_cache` loader / `schema_subset_errors` structural check / `source_sha256`+`schema_sha256` binding /
fail-closed secret scan / `target_class` disposable-gate / `ttl ∈ [1,3600]` conventions. It is
**stateless** (pure contract-checking functions + one optional disposable PG probe + real temp-dir
`chmod`/`stat`), so **no singleton-table registration**.

It **self-validates** its own receipt and is exercised only by its own tests. It does **not** register
into the central AIML closure-validator (`aiml_gate_receipt_validator.py`), the S1.2 effect matrix, the
governance registry `effect_adapters`, the route compiler, or the vocabulary. Those are the shared
surfaces the **consumer** (S1.5/S2.4) will wire when the `pg_role_acl_migration_adapter` /
`credential_rotation_adapter` exist — exactly as S1.1 left its central wiring to its consumer S1.2. See
§6 for the full wiring-vs-self-validating justification.

Language: **Python, stdlib-first** (`hashlib`/`json`/`os`/`stat`/`shutil`/`tempfile`), matching the
entire AIML governance surface and S1.1/S1.2/S1.4; `psycopg2` is a **lazy** import used only inside the
`initdb`-gated disposable test path (S1.1 precedent). CLAUDE §七 "Rust-first" carve-out ("unless the
local design clearly says otherwise") applies — this is governance tooling around the identity/ACL and
`psql` boundary, whose entire neighborhood is Python; S1.1/S1.2/S1.4 made and PM accepted the identical
carve-out. **Portability (CLAUDE §六):** `REPO_ROOT = Path(__file__).resolve().parents[2]`; the
canonical production install paths (e.g. `/opt/aiml/run/<component>.sock`, UID labels) are **recorded as
design data**, never a path/uid the Mac spike writes to or creates. No `/Users/ncyu` or `/home/ncyu`
literal.

## 3. The declarative least-privilege identity/ACL topology

The component set is taken from plan §LR2-311 + §LR3-326 + the S1.2 effect classes. Each component gets a
**distinct** non-root host UID and a **dedicated** least-privilege PG role; **no shared UID, no shared
role, no writer role for a reader, no superuser, and controller/workers get neither OCI socket nor DBus
authority.**

| Component (`host_uid_topology`) | Dedicated PG role (`pg_role_topology`) | `privilege_class` | OCI socket | DBus | Effect class it maps to |
|---|---|---|---|---|---|
| `engine_scanner` | `aiml_engine_scanner` | `scanner_capture_writer` (bounded) | no | no | `ENGINE_SCANNER` |
| `controller` | `aiml_controller` | `queue_writer` (job rows only) | **no** | **no** | `CONTROLLER_WORKERS` |
| `fit_evaluation` | `aiml_fit_evaluation` | `fit_evaluation_writer` (read dataset, write fit + eval) | **no** | **no** | `CONTROLLER_WORKERS`/`LEARNING_RUNTIME` |
| `serving` | `aiml_serving` | `serving_read_only` (registry read) | no | no | ML5 serving |
| `deleter` | `aiml_deleter` | `retention_deleter` (delete tombstoned rows) | no | no | `RETENTION_APPLY` |
| *(observation)* | `aiml_observer_ro` = **the S1.1 identity** | `observer_read_only` | no | no | LR0A / `PG_READONLY` |

Notes: the S1.1 read-only identity is **one row** in this topology (`observer_read_only`), reused
as-is; S1.3 adds the component roles the runtime needs. `serving_read_only` and `observer_read_only`
are readers and MUST NOT carry any write attribute (the "no writer-for-reader" invariant). Every role
has all of `rolsuper/rolcreaterole/rolcreatedb/rolbypassrls/rolreplication` **false** (S1.1's
`FORBIDDEN_ROLE_ATTRS`). The **auth mapping** is local-only: either `pg_hba` `ident`/`scram-sha-256`
on a local (Unix-socket) line, or an authenticated loopback identity — never `trust`, never a
non-local CIDR, never `0.0.0.0/0`. **Socket-dir ACLs** are private (mode `0700`, owner = the component
UID, not world-readable/writable). **Secret lifecycle**: each rotating credential binds a slot/role
target, an ordered rotation, non-secret sha256 fingerprints of old/new, an old-credential-rejection
proof, protected loading, and `plaintext_ingress=false`. **Rollback**: every identity/ACL change binds
a `pre_state_digest` + a concrete `rollback_action` (mirrors `PG_ROLE_ACL_MIGRATION.recovery`).

## 4. The honest disposable-vs-deferred evidence split (the crux for S1.3)

Per facet. `LOCAL_REPRODUCIBLE` = real bytes recapturable on this Mac (real disposable PG cluster /
real temp-dir `chmod`/`stat`); `STRUCTURAL_ONLY` = declarative contract + structural negative, no
runtime bytes; `DEFERRED_S2` = requires a real host mutation (root `useradd`, real `pg_hba` install,
real production credential slot) and is owned by S2.0/S2.4.

| Facet | Evidence class | How it is proven at S1.3 |
|---|---|---|
| PG role least-privilege: distinct roles, all forbidden attrs false, **no superuser**, **reader cannot write** | **LOCAL_REPRODUCIBLE** | real disposable cluster (reuse S1.1 `initdb` pattern): `CREATE ROLE` the component roles, `GRANT` least-privilege, connect as a reader role and observe write denial **`42501` insufficient_privilege** (real SQLSTATE) |
| **Writer-for-reader over-grant is rejected** | **LOCAL_REPRODUCIBLE** | a contract putting a write privilege on `serving_read_only`/`observer_read_only` → contract validator REJECTS; and on the disposable cluster, a reader granted write is proven to actually write (so the contract's refusal is meaningful, not vacuous) |
| **Credential rotation + old-credential rejection** | **LOCAL_REPRODUCIBLE** | on the disposable cluster: `ALTER ROLE … PASSWORD` A→B over a `scram-sha-256` local line, reconnect with **old** password → real denial **`28P01` invalid_password** (or `28000`); fingerprints = `sha256` of throwaway A/B (non-secret) |
| Socket-dir ACL mode (private `0700`, not world-rw) | **LOCAL_REPRODUCIBLE** | real temp dir: `mkdir` + `os.chmod(0o700)` + `os.stat` → assert `st_mode & 0o077 == 0`; a contract asserting `0777`/world-writable → REJECTED |
| Host-UID separation (distinct / non-root / **no OCI socket** / **no DBus**) | **STRUCTURAL_ONLY** | declarative: uid_labels all distinct, every `non_root=true`, controller/worker `oci_socket_access=false` & `dbus_authority=false`; over-grant contracts (shared UID, root UID, controller-with-OCI-socket/DBus) → REJECTED. Real `useradd` needs root → **DEFERRED_S2** |
| Auth mapping: **trust / from-anywhere / wide-CIDR rejected**, local-only | **STRUCTURAL_ONLY** | contract validator rejects `method=trust`, non-local CIDR, `0.0.0.0/0`. *(Actually binding `0.0.0.0` to prove it live is the insecure act we refuse — so this negative is structural by design.)* Optional hardening in the disposable password-denial above covers a real local scram line |
| Per-change rollback present | **STRUCTURAL_ONLY** | every change binds `pre_state_digest`+`rollback_action`; a change missing rollback → REJECTED |
| **Real host UID/role/ACL/pg_hba/credential mutation on the production host** | **DEFERRED_S2** | `production_uid_provisioned` / `production_role_provisioned` / `production_hba_installed` / `production_socket_provisioned` / `production_credential_rotated` are **all const `false`**; S1.3 mutates no real host |

**Honest headline:** the facets that are *properties of PG semantics and filesystem modes* (PG
role/ACL/rotation, socket-dir mode) get **real disposable LOCAL_REPRODUCIBLE** evidence — because a real
`postgres` process and a real `chmod`/`stat` emit genuine denials (`42501`/`28P01`) and mode bits,
nothing mocked (CLAUDE's "Mac mocked tests cannot establish PG runtime semantics" caveat does **not**
apply — nothing is mocked). The facets that are *host-identity acts requiring root* (UID creation, real
`pg_hba` install, real credential slots) are honestly **STRUCTURAL_ONLY contract + structural negatives
now**, with the real mutation **DEFERRED to S2.4** — never faked. This split is the deliverable: it
prevents a contract Session from quietly promoting "I asserted the topology in JSON" into "the host
identity is provisioned and proven."

### 4.1 Evidence-authenticity honesty — a `LOCAL_REPRODUCIBLE` label is not self-authenticating

One further honesty line the receipt must not blur (CLAUDE §四): the `_disposable` test genuinely runs a
real cluster and real `chmod`/`stat`, but the **receipt bytes** only *carry a label* about that run. The
per-facet `evidence_class` ceiling is derived from the more-specific advisory fields `observation_source`
/ `mode_source` / `new_credential_connected` rather than a bare `evidence_class` field — which defeats
the *laziest* forgery (flipping `evidence_class` alone, a real improvement the builder and the consumer's
re-verification both enforce). But those source fields are themselves **caller-supplied labels in the
same self-hashed receipt**: a hermetic caller could set `observation_source==live_disposable_pg` on bytes
it never actually observed, and the self-digest would still verify (a canonical self-digest proves
integrity, never who executed what). This is **not** crypto-fixable at S1.3 — only
`PLATFORM_OR_EXTERNAL_ATTESTED` evidence can authenticate execution — so S1.3 does not pretend to.
**Consumer contract:** any S1.5/S2.4 consumer that relies on a `LOCAL_REPRODUCIBLE` identity/ACL receipt
**MUST re-run the disposable proof** (a real cluster / `chmod`/`stat`) **or obtain platform attestation**
before trusting it; it may **not** trust the receipt's `evidence_class` label alone. The receipt's honest
role is a reproducible *summary + integrity seal*, not a proof of execution.

## 5. `identity_acl_contract_receipt_v1` schema shape

Draft 2020-12, `additionalProperties:false`, `$defs` for `digest`/`timestamp`/`sqlstate`, mirroring
`pg_readonly_identity_receipt_v1.schema.json`. A frozen `RECEIPT_FIELDS` set is validated exactly.
Required top-level fields:

- `schema_version` — const `"identity_acl_contract_receipt_v1"`
- `adapter_id` — const `"identity_acl_contract_adapter_v1"`
- `status` — enum `["PASS","FAIL"]`
- `caller` — string ≥1 (governed caller/role id)
- `platform` — `{os: enum[darwin,linux], arch, postgres_version}` (the disposable-PG facet pins reproducibility)
- `target_class` — enum `["disposable_local","production"]` — **S1.3 accepts only `disposable_local`; `production` rejected fail-closed** (production provisioning is S2.0/S2.4). Mirrors S1.1.
- `evidence_class` — enum `["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"]` — receipt-level **ceiling summary** (= strongest non-deferred facet actually exercised). **Advisory (§4.1):** a self-hashed receipt cannot authenticate its own execution, so a `LOCAL_REPRODUCIBLE` label alone is a self-attested summary; a consumer must re-run the disposable proof or obtain platform attestation before trusting it.
- `host_uid_topology` — array (one per component) of `{component: enum[engine_scanner,controller,fit_evaluation,serving,deleter], uid_label: string, non_root: const true, oci_socket_access: const false, dbus_authority: const false, least_privilege_caps: [string…] (each a benign allowlisted token; over-broad caps rejected), evidence_class: enum["STRUCTURAL_ONLY","DEFERRED_S2"], production_uid_provisioned: const false}` — distinctness of `uid_label` is validated across the array in Python; **each `uid_label` is additionally cross-bound to the component's canonical host UID and any `root`/`superuser`/privileged label token is rejected even when `non_root=true` (a free-form label must not ride on the boolean).**
- `pg_role_topology` — array (one per role) of `{component: string, role_name: string, privilege_class: enum[observer_read_only,scanner_capture_writer,queue_writer,fit_evaluation_writer,serving_read_only,retention_deleter], is_superuser: const false, forbidden_attrs_all_false: const true, is_reader: bool, writer_for_reader: const false, evidence_class: enum["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"], production_role_provisioned: const false}` — role_name distinctness validated in Python; component→canonical `privilege_class` cross-bound in the validator; `is_reader==true ⇒` no write privilege.
- `auth_mapping` — `{method: enum[pg_hba_ident_local,pg_hba_scram_local,authenticated_loopback], local_only: const true, trust_from_anywhere: const false, wide_cidr: const false, ident_map: const null (S1.3 removes the ident_map free-form attack surface — the honest disposable contract never uses an ident map; a bounded ident map is genuinely S2.4 scope), evidence_class: enum["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"], production_hba_installed: const false}`
- `socket_dir_acl` — array of `{component: string, socket_dir_label: string, mode: string pattern ^0[0-7]{3}$, mode_source: enum["live_chmod_stat","structural_contract"], world_readable: const false, world_writable: const false, owner_uid_label: string, group_label: string, evidence_class: enum["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"], production_socket_provisioned: const false}` — `mode & 0o077 == 0` validated in Python; an `evidence_class==LOCAL_REPRODUCIBLE` row must carry `mode_source==live_chmod_stat` (a real `chmod`/`stat`); **`owner_uid_label`/`group_label` are cross-bound to the owning component's canonical host UID (a single owner shared across components, or an owner naming root/privileged, is rejected — the same binding runs in the consumer's reconstruction gate).**
- `secret_lifecycle` — `{rotation: {secret_slot_target: string, role_target: string, old_fingerprint: digest, new_fingerprint: digest, rotation_order: [string…]≥3 (stage-new → alter/activate → revoke-old; **the alter/activate step is mandatory — a `["stage","revoke"]` order that never activates the new credential is rejected** — and revoke-before-stage is rejected), old_credential_rejection_proof: {attempted: string, observed_sqlstate: sqlstate, verdict: const "DENIED", observation_source: enum["live_disposable_pg","structural_contract"], new_credential_connected: bool}}, protected_loading: {no_plaintext_ingress: const true, loader_kind: string}, plaintext_ingress: const false, evidence_class: enum["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"], production_credential_rotated: const false}` — field names mirror the S1.2 `CREDENTIAL_ROTATION` intent verbatim (§1.1); `old_fingerprint != new_fingerprint`; a live rotation proof (`observation_source==live_disposable_pg`) must observe invalid-password `28P01` and prove the new credential connected first.
- `rollback` — array (one per identity/ACL change) of `{change_id: string, change_kind: enum[host_uid,pg_role,auth_mapping,socket_acl,secret_slot], pre_state_digest: digest, rollback_action: string≥1, recovery: enum[rollback,approved_forward_only]}` — mirrors `PG_ROLE_ACL_MIGRATION.recovery`.
- `negative_acl_cases` — array (**the ten §8 crux cases**, `minItems: 10`) of `{case_id: string, over_grant_kind: enum[writer_role_for_reader,shared_uid,root_uid,world_readable_socket,superuser_role,trust_auth_from_anywhere,wide_cidr,missing_rollback,controller_oci_socket_or_dbus,plaintext_secret_ingress], expected: const "FAIL_CLOSED", observed_verdict: const "REJECTED", evidence_class: enum["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"], reason: string≥1}` — **the crux; each records that the contract validator refused an over-grant. The consumer's validator independently reconstructs the contract, re-runs the least-privilege checker (must be empty) and re-derives these cases (kinds must match exactly) — it never rubber-stamps the claimed verdicts.**
- `source_sha256` / `schema_sha256` — digests binding this module + this schema.
- `secret_scan` — `{patterns_checked: [string…], leaked: const false}`.
- `observation_time` / `expires_at` / `ttl_seconds` (int `[1,3600]`) — same TTL policy as S1.1/S1.4.
- `failure_reason` — `string|null`.
- `self_digest` — digest over every field except itself.

**`allOf` conditional (fail-closed gate), enforced in the Python validator** (`schema_subset_errors`
processes `allOf` but not `if/then`, per S1.1/S1.2). If `status=="PASS"` then: `target_class==disposable_local`;
`failure_reason==null`; **every** `negative_acl_cases[*].observed_verdict=="REJECTED"` and the ten crux
`over_grant_kind`s are all present; **every** `production_*_provisioned==false` (S1.3 provisions nothing
real); all distinct-UID / non-root / no-OCI-socket / no-DBus / no-superuser / no-writer-for-reader /
allowlisted-caps / local-only / no-trust / no-wide-CIDR / null-or-bounded-ident-map / safe-rotation-order /
mode-`&0o077==0` / rollback-present / component-completeness invariants hold;
`secret_scan.leaked==false`; and `evidence_class` consistent with the strongest non-deferred facet
exercised. Else `failure_reason` is a non-empty string. Structural/integrity violations that cannot be
safely serialized (secret detected, an over-grant case **not** `REJECTED`, any `production_*_provisioned==true`,
`target_class=production`, a runtime rotation/denial code that is not denial-class, ttl out of range)
**raise** instead of emitting a receipt — mirrors S1.1's `build_pg_readonly_identity_receipt` and S1.4.

Reproducibility: `source_sha256` + `schema_sha256` + `platform.postgres_version` + the deterministic
disposable fixture ⇒ every LOCAL_REPRODUCIBLE facet and the `self_digest` are independently re-derivable
(plan §LR0A-213 style).

## 6. Wiring vs self-validating — recommendation + justification

**Recommendation: SELF-VALIDATING + DISJOINT** (mirror S1.1/S1.4). S1.3 does **not** wire
`identity_acl_contract_receipt_v1` into the central validator, and does **not** modify the S1.2 matrix.
Justification, strongest first:

1. **Deletion test — no consumer at the S1.3 gate.** Central registration (a `SCHEMA_FILES` entry + a
   `validate_aiml_artifact` dispatch branch delegating to S1.3's validator) only *buys* the ability for

## 8. Negative-ACL test plan (the crux, fail-closed)

The load-bearing property (plan §LR0B-245, §LR2-313): **a contract that grants too much must FAIL
closed.** A `assert_least_privilege_topology(contract) -> list[str]` checker (called by the builder
before it will emit a PASS, and directly by the tests) returns non-empty errors — and
`build_identity_acl_contract_receipt` **raises / emits FAIL** — for each over-grant. The nine §8 crux
rows below = **ten** `over_grant_kind`s (row 6 carries both `trust_auth_from_anywhere` and `wide_cidr`),
and a PASS receipt records all ten as `REJECTED` (`negative_acl_cases` `minItems: 10`). The crux cases
(hermetic unless marked disposable):

| # | `over_grant_kind` | Over-granting input | Required verdict | Evidence |
|---|---|---|---|---|
| 1 | `writer_role_for_reader` | write privilege on `serving_read_only`/`observer_read_only` (`is_reader=true` + write) | REJECTED | STRUCTURAL; **disposable** confirms a write-granted reader actually writes (refusal non-vacuous) |
| 2 | `shared_uid` | two components share one `uid_label` | REJECTED (distinctness) | STRUCTURAL |
| 3 | `root_uid` | any component `non_root=false` / uid 0 | REJECTED | STRUCTURAL |
| 4 | `world_readable_socket` | `socket_dir_acl.mode` with `&0o077 != 0` (e.g. `0777`, `0755`) | REJECTED | **disposable** `chmod`/`stat` + STRUCTURAL |
| 5 | `superuser_role` | any role `is_superuser=true` / a forbidden attr true | REJECTED | STRUCTURAL; **disposable** confirms a non-superuser reader is denied write `42501` |
| 6 | `trust_auth_from_anywhere` / `wide_cidr` | `auth_mapping.method=trust`, or non-local / `0.0.0.0/0` CIDR | REJECTED | STRUCTURAL (binding `0.0.0.0` live is the insecure act we refuse) |
| 7 | `missing_rollback` | an identity/ACL change lacking `pre_state_digest`/`rollback_action` | REJECTED | STRUCTURAL |
| 8 | `controller_oci_socket_or_dbus` | controller/worker with `oci_socket_access=true` or `dbus_authority=true` | REJECTED (plan §LR2-311) | STRUCTURAL |
| 9 | `plaintext_secret_ingress` | a plaintext credential anywhere in the contract | REJECTED (`SecretLeakageError`) | STRUCTURAL secret-scan |

The checker is additionally **exhaustive over the free-form fields** (not separate crux cases, but they
make the honest builder + the consumer's re-verification reject an over-grant): `least_privilege_caps`
outside a benign allowlist (`over_broad_capability`); **any non-null `auth_mapping.ident_map`
(`unsafe_ident_map` — S1.3 accepts only `null`)**; a `rotation_order` that revokes the old secret before
staging the new, **or that lacks an alter/activate step so the new credential is never activated**
(`unsafe_rotation_order`); **a `host_uid_topology[].uid_label` that is not the component's canonical host
UID or that names a root/privileged identity (`unsafe_uid_label`)**; and **a `socket_dir_acl[].owner_uid_label`
/`group_label` that is not the owning component's canonical host UID, is shared across components, or names
a root/privileged identity (`unsafe_socket_owner`)**. Because the consumer's validator reconstructs the
6-facet contract and re-runs this exact checker (§5), a forged receipt carrying any of these is caught by
the consumer's gate too, not only by the builder.

Plus positive disposable proofs (`..._disposable.py`): the honest-topology contract PASSes; the reader
role is genuinely write-denied (`42501`); credential rotation's **old** password is genuinely rejected
(`28P01`); the socket dir is genuinely `0700`. Plus the standard receipt-integrity suite (field-set,
digest re-derivation, TTL/time-order, `production` fail-closed, `production_*_provisioned` machine-false,
secret-scan).

## 9. Boundary compliance & fail-closed posture

- **`DISPOSABLE_ONLY` / no production provisioning** — every `production_*_provisioned` is const `false`;
  the disposable cluster + temp dirs are torn down; the sole artifact is the reproducible receipt JSON.
  Real host UID/role/ACL/`pg_hba`/credential mutation is **DEFERRED to S2.0/S2.4**.
- **No real UID/role mutation** — host-UID facet is declarative + structural (`useradd` needs root,
  S2.4); disposable-cluster `CREATE ROLE`/`ALTER ROLE` is throwaway scaffolding on a temp dir, never
  production, never a migration (S1.1 precedent).
- **No migration** — the contract references `migration_id`/`role_acl_delta` vocabulary but applies no
  migration; the disposable cluster is not a repo migration and not production.
- **`psql` stays denied** — `psycopg2` over the socket from within `pytest`; `permissions.py` untouched.
- **Frozen S0.3 + S1.2 matrix untouched** — S1.3 edits neither `aiml_gate_receipt_validator.py` nor the
  matrix/digest (§6, §7); nine authorities stay FALSE.
- **Secrets** — the contract carries only non-secret sha256 fingerprints; a `GITHUB_SECRET_LIKE_RE`-style
  scan (extended with `PGPASSWORD`/`password=`/DSN shapes, as S1.1) over the serialized receipt refuses
  to emit anything secret-like; `plaintext_ingress=false`.
- **Fail-closed defaults** — any over-grant not REJECTED → raise/FAIL; `production` target → reject; a
  rotation/denial code not in the denial class → raise; a real-provision flag true → raise; missing PG
  binaries → **skip** the disposable facet (never a false PASS); secret detected → refuse to serialize.

### 9.1 Accepted residuals (tracked, closed by S2.0/S2.4)

1. **Host-UID separation is a declarative contract at S1.3.** Distinctness/non-root/no-OCI-socket/no-DBus
   are proven *structurally*; the real `useradd`, cgroup/slice binding and the real "prove denied
   actions" (plan §LR3-327) are **S2.4**. Stated, not papered over.
2. **The `trust`/wide-CIDR negative is structural.** Proving it by actually binding `0.0.0.0` would
   itself be the insecurity; the disposable password-denial (`28P01`) covers a real *local* auth line
   instead. Full `pg_hba` install/enforcement is **S2.0/S2.4**.
3. **Auth-mapping and secret slots are throwaway.** The rotation proof uses a disposable cluster password,
   not a real secret slot/manager; the real slot rotation + old-credential rejection *on the production
   role* is **S2.4** (`CREDENTIAL_PG_UNIT_INSTALL`).
