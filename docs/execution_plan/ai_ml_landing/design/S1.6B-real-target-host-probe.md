# S1.6B — Real Target-Host Disposable Runtime Probe (trade-core, non-root)

**Status**: DESIGN (read-only; no effect executed). Supersedes the S1.6 Mac-local
stand-in for §LR0C's target-host requirement. Author: OPS design + PM decisions.
Effect class: `TARGET_HOST_DISPOSABLE_RUNTIME_PROBE` (reuses the S1.5-registered
`learning_runtime_deploy_adapter_v1`; no new adapter).

## Why (Codex PR #112 finding)

S1.6 (Mac) made a BINDING runtime choice WITHOUT running the real target-host
probe, and S1.5's "remote/platform observation seam" was contract-only. Plan
§LR0C (lines 257-275) requires: "After the LR0B remote/platform seam exists, use
a typed `TARGET_HOST_DISPOSABLE_RUNTIME_PROBE` ... Exercise start/stop, cgroup/
resource isolation, network denial, PG identity, native-library loading,
immutable dependency closure, artifact persistence, failure recovery, rollback
and complete cleanup with an independent postcheck. Only then select exactly
one." This design does the real probe on trade-core.

## 1. Docker / OCI verdict

Docker is reachable by `ncyu` without sudo (docker group) but is **rootful,
system-scope, and root-equivalent**, and would hand the probe exactly the OCI
socket authority **LR2 forbids** ("controller/workers receive neither OCI socket
nor DBus authority"). Rootless alternatives are absent. **Within the mandated
non-root/user-scope boundary, candidate A (OCI) has no admissible runtime on
trade-core** → boundary-driven non-selection, consistent with LR2 as written.
This is a REAL-evidence confirmation of S1.6's rule-based OCI non-selection.

## 3. Remote/platform observation seam (S1.5 made real)

The probe **executes on trade-core (the target host)**; a **distinct postcheck
actor reads on-host kernel/systemd/cgroup/filesystem state directly over the ssh
bridge** via the governed on-host `capture-command` (shell=false, Context-bound,
reader identity recorded). "Remote/platform attested" here = evidence whose
authenticity comes from reading trade-core state **on trade-core**, not the
applier's self-report: `systemctl --user show`, the scope cgroup files under
`/sys/fs/cgroup/.../user@1000.service/`, private-netns egress failure, on-disk
digest, post-cleanup absence. Honest trust limit: anchor = ssh host identity +
governed on-target capture. This is `PLATFORM_OR_EXTERNAL_ATTESTED` relative to
the Mac author (Mac cannot self-produce trade-core kernel facts); it upgrades
S1.5 from contract-only to target-host-observed-by-an-independent-actor. It is
**not** hardware-rooted attestation (no TPM/measured boot). applier ≠ postcheck
throughout.

## 4. Effect gating (real disposable effect on trade-core)

1. **OPS preflight** (independent of applier): host==trade-core (ssh host key);
   uid=1000 + no passwordless sudo; linger active; delegated controllers ==
   `cpu memory pids`; target root == fresh `mktemp -d` under `/run/user/1000`
   (never a production path); no production unit/socket/PG in scope; docker NOT
   invoked. Emit preflight receipt.
2. **PM/operator-approved exact intent**: `TARGET_HOST_DISPOSABLE_RUNTIME_PROBE`
   binding exact host, non-root uid, user-scope-only, two candidate ids, per-seam
   argv, throwaway root, TTL, rollback (atomic swap + rmtree), applier ≠
   postcheck. Reuses S1.5 `learning_runtime_deploy_adapter_v1`.
3. **Execution**: applier runs each candidate probe through the governed on-host
   `capture-command` (Context-bound), user-scope only.
4. **Independent OPS postcheck**: distinct node re-observes each seam + proves
   complete cleanup (no residue).

**Boundary**: non-root; user-scope transient units only (NO `systemctl`
system-scope, NO `--system`); NO docker/rootful daemon; NO production
unit/socket; prod PG on :5432 untouched; NO production data/paths; everything
under a fresh throwaway dir + user cgroup subtree + private netns; complete
teardown with independent residue check. **Fail-closed stops**: unexpected
passwordless sudo; missing delegated cpu/mem/pids; target under any production
path; any docker/system-scope; any prod-PG contact; applier==verifier; cleanup
residue; secret in any serialized field; TTL exceeded → STOP, no PASS.

  production path/unit/socket/PG in scope, applier node ≠ independent-postcheck
