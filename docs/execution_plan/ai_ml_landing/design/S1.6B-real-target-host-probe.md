## 10. Real-run findings (executed on trade-core, 2026-07-23)

The probe ran for real, non-root, on `trade-core`. Three seams needed a
primitive swap because the **as-designed** primitive is blocked by a real host
policy — each swap preserves the seam's essential property and is recorded
honestly (no forced PASS). Root cause for all three:
`apparmor_restrict_unprivileged_userns=1` (Ubuntu 24.04 default) +
`newuidmap`/`newgidmap` absent → non-root unprivileged user namespaces are
denied, so `bwrap`/`unshare -rn` fail (`uid_map` "Permission denied" for a
userns-only sandbox; with `--unshare-net` they get past uid_map but die at `lo`
bring-up, which needs root-in-userns). This is the same class of boundary as the
OCI non-selection: a real host constraint, not a harness gap.

1. **network_denial: netns → seccomp.** `bwrap --unshare-net` cannot run a child
   here. Replaced with a libseccomp (`ctypes`) `SCMP_ACT_ERRNO(ENETUNREACH)`
   filter on `connect`/`sendto`/`sendmsg`, proven by a **differential**: a
   no-filter baseline child CONNECTs to the same host:port; the filtered child is
   kernel-denied (`ENETUNREACH`). seccomp needs only `no_new_privs` (fully
   non-root) and is the real enforceable egress-denial mechanism on this host.
   → `PASSED_TARGET_HOST`.

2. **cgroup_resource_isolation: OOM read race fixed.** The original workload let
   the cgroup OOM kill the scope's **main** process, so systemd tore the scope
   down before `memory.events:oom_kill` could be read (it always read 0 → false
   DEFERRED). Fixed by `-p OOMPolicy=continue` + restructuring so a **child hog**
   is OOM-killed while **main survives** to keep the cgroup dir alive for the
   read. All three counters now fire: `oom_kill=1`, `pids.events:max=1`,
   `cpu.stat:throttled_usec>0`. → `PASSED_TARGET_HOST`.

3. **native_lib_loading: copy-system-.so → compile-unique-soname + direct load.**
   Copying an already-loaded system lib (`libz`) never maps from the bundle — the
   dynamic linker dedupes by `DT_SONAME`, so `/proc/self/maps` showed the system
   path (false DEFERRED). Fixed by compiling a **unique-soname** `.so`
   (`libaiml_probe_bundle.so`, `cc -shared`) into the bundle, then `CDLL` it and
   assert `maps` shows the **bundle** prefix + the symbol is callable (=42). bwrap
   mount-isolation is AppArmor-blocked, so the load falls back to a **direct**
   run (`sandbox_mode=direct_no_sandbox`); the maps-origin proof is real in either
   mode (unique soname rules out system dedup). Mount-isolated variant deferred
   like cpuset/io. → `PASSED_TARGET_HOST` (representativeness-flagged; full
   LR2-sealed native closure = S2.3).

**Result:** 7/8 fixed-path seams `PASSED_TARGET_HOST` on real evidence
(`start_stop`, `cgroup_resource_isolation`, `network_denial`,
`native_lib_loading`, `immutable_closure_persistence`,
`failure_rollback_cleanup`, `pg_identity` = real disposable initdb cluster →
42501). `independent_postcheck` is DEFERRED by design in the applier's own

`content_addressed_fixed_path` is **BINDING**. OCI stays
`NON_SATISFIABLE_NON_ROOT`. Complete teardown verified: zero residue (0

On-target pytest `test_agent_governance_target_host_probe_on_target.py`: 3

## 12. Exact-head adversarial hardening (2026-07-24)

Review of the authenticated-but-unmerged PR #115 head found five further P1
forge paths. They are closed at source before a new target-host effect is
allowed to be emitted:

1. **The child no longer accepts caller-created authority.** A canonical
   capsule checksum is transport integrity only. The isolated child now
   requires the existing source-pinned operator key to SSH-sign the exact typed
   intent and source head under the dedicated
   `arcane-equilibrium-aiml-s1-target-host-apply` namespace. It revalidates the
   intent self-digest and capsule-to-intent host/actor/root/launcher/expiry
   projection before opening its process-local gate. Re-sealing a capsule after
   substituting a root or launcher fails.

2. **Preflight is an observed fact, not a structurally valid record.** The
   applier capture must be the exact operator-authenticated governed observer
   command produced by `ops_preflight`. The validator replays the permit
   signature and binds complete stdout bytes to safe host facts. A generic
   governed `git rev-parse` capture cannot back the target-host receipt.

3. **Postcheck performs the residue sweep inside the captured command.** The
   declared `ops_postcheck` observer returns the exact zero-residue result in
   complete captured stdout; the closure cross-checks that projection against
   its residue evidence. A same-process clean claim, unrelated command, wrong
   node, truncated output, or generic governed capture fails.

4. **Closure time is authenticated.** S1 closure/source freshness is evaluated
   at `trusted_execution_bundle_v1.issued_at`, which is inside the operator
   SSHSIG bytes and is first checked for host-time freshness/skew. A local
   timestamp appended after signing can no longer change historical replay.

5. **Every mandatory reviewer owns its PASS.** In an effect-bearing closure,
   PA/CC/E2/E3/E4/QA fragments must cite the authenticated workflow wave whose
   `result_fragment_digests` contains that exact fragment. The target-host
   effect receipt can support only the effect/OPS nodes; it cannot substitute
   for seven independent reviewer results.

The Registry implementation-path closure and landing-attempt owned paths include
the apply child, operator authorization, semantic observers/validator, effect
validator, trust layer and finalizer. These changes do not grant any runtime,
Postgres, deploy, migration, broker, order, ML5/ML6 or direct-model authority.
The prior runtime artifact is invalidated by the new source head; a fresh
two-phase `trade-core` run and fresh closure SSHSIG are mandatory before
