## 0. The overriding boundary — why S1.4 cannot (and must not) select

Plan §LR0C makes the final choice conditional on the exact seams that **only a target-host probe can
exercise**:

> "…use a typed `TARGET_HOST_DISPOSABLE_RUNTIME_PROBE`… Exercise start/stop, **cgroup/resource
> isolation, network denial, PG identity, native-library loading**… Only then select exactly one:
> exact-image-ID OCI, only if **all seams pass**; or content-addressed fixed-path runtime, **if OCI
> cannot satisfy them**."

The selection criterion is therefore **dominated by S1.6-only seams**. It is not that S1.4 chooses to
defer the choice — the plan's own decision rule keys off runtime facts (cgroup, network denial, PG
identity, native-lib *loading*) that do not exist offline. So S1.4's honest deliverable is: prove each
candidate on the **offline-evaluable** seams, mark every runtime-only seam **DEFERRED**, and emit a
comparison whose `final_choice` is **machine-forced `null`**. Any receipt that claimed a selection at
S1.4 would be faking evidence (CLAUDE §四 "do not fake … test results / evidence").

This mirrors S1.1's disposable-only gate and S1.2's frozen-additive discipline: the boundary is
**schema-enforced**, not merely promised in prose.

## 1. What LR0C requires (offline slice), and the runtime it must eventually host

The eventual runtime (plan §LR2, lines 306-316) must be: built from **sealed inputs without network**;
**Python isolated mode** verified; **LightGBM/sklearn/ONNX/native-library origins** and **lock
closure** verified; **no `/usr/bin/python3` fallback**; host systemd owns lifecycle; **no mutable
tag/alias, relocatable renamed venv, system-Python fallback, or dual production stack** (§LR0C lines
272-273). §LR2 is explicit that it "emits sealed-build and expected-identity receipts only; it does not
attest a running unit … network or PG identity" — those belong to S2.5/LR6.

S1.4 sits *before* even LR2's sealed build. It is a **candidate spike**: for each of the two candidate
runtime *shapes*, evaluate which of those eventual properties are provable **offline on this dev Mac**,
which are **structural/design-only**, and which **defer to the S1.6 target-host probe**.

The two candidates (plan §LR0C lines 267-268):

- **(A) exact-image-ID OCI** — an OCI image pinned by exact image ID / digest.
- **(B) content-addressed fixed-path runtime** — a hash-named runtime bundle at a fixed path.

## 2. Ground truth — what THIS machine can actually do (probed 2026-07-22)

Recorded verbatim so the evidence split below is honest, not aspirational.

**Container tooling**

```
which docker   → /opt/homebrew/bin/docker      (Docker Engine Community, client 29.6.1)
which colima   → /opt/homebrew/bin/colima       (running: macOS Virtualization.Framework)
which podman / nerdctl / buildah → NOT FOUND
docker info  server → 29.2.1 REACHABLE          (context "colima", socket unix://<home>/.colima/default/docker.sock)
colima status → arch: aarch64 · runtime: docker · mountType: virtiofs
docker buildx → "unknown command: docker buildx"   ← NO buildx plugin
```

Consequence: a Linux OCI daemon **is** reachable locally, but it is a **colima linux/arm64 VM only**,
and **without `buildx` there is no linux/amd64 (qemu) cross-build**. If `trade-core` is x86_64, an
image built here is the **wrong architecture**; the target-host arch is **unknown offline**. So the OCI
candidate's local capability is real but **arch-limited and not the target host**.

**Python + lockfile tooling**

```
python3 → 3.10.1 · /usr/local/bin/python3 · arch arm64 · NOT in a venv (sys.prefix==base_prefix)
pip (bare) → NOT FOUND ; python3 -m pip → pip 21.2.4 present
uv → 0.11.26  (has --generate-hashes, --no-index, --offline / UV_OFFLINE)
python3 -I  → isolated: 1 · no_user_site: 1
PYTHONPATH=/tmp/injected python3 -I  → injected path present in sys.path: FALSE   ← -I ignores ambient env
```

Consequence: **`python3 -I` provably ignores ambient `PYTHON*` env** — the *isolated-mode /
no-system-Python-fallback mechanism* is genuinely **offline-provable on the host interpreter**. `uv`
can *generate* a hash-pinned lockfile, but **resolving** the real LightGBM/sklearn/ONNX closure needs a
package index (network); with `--offline --no-index` it can only resolve against a pre-seeded local
cache we do not have. So real-ML-closure **resolution** is a **networked sealed-build (LR2/CI)** step,
not an offline-Mac step; only the **content-addressing / hashing mechanism** is offline-provable here.

**Repo state**: clean slate — no `Dockerfile`/`Containerfile`/OCI artifact and no runtime-bundle or
sealed-build helper anywhere (`agent_governance_p0b_runtime_bindings.py`,
`p0b_psycopg_private_bundle_stage_v1.py`, `runtime_environment_probe.py` are unrelated). S1.4
introduces the first runtime-candidate harness.

## 3. Minimum-coherent design of the offline spike

### 3.1 Shape decision

One new **self-contained, self-validating** governance harness module that mirrors the S1.1 adapter
conventions exactly (`HARNESS_ID`, `build_*`/`validate_*` receipt pair, canonical `self_digest`,
`_receipt_schema()` `lru_cache` loader, `schema_subset_errors` structural check, `source_sha256` /
`schema_sha256` binding, fail-closed secret scan, `target_class` disposable-gate). It is **stateless**
(pure functions + one optional disposable subprocess probe), so **no singleton-table registration**.

Like S1.1's first cut, it **deliberately does not register into the central AIML closure-validator**
(`aiml_gate_receipt_validator.py`) nor the registry / route-compiler / vocabulary — those shared
governance surfaces are what **S1.2 owns**, and S1.4 must stay **disjoint** (protocol line 159:
"S1.1-S1.4 may overlap only where path manifests are disjoint"). S1.4 **self-validates** its own
receipts; central-validator wiring, if ever wanted, is a later concern (§8.2).

Language: **Python, stdlib-first** (`hashlib`/`json`/`subprocess`/`shutil.which`/`tempfile`), matching
the whole AIML governance surface and S1.1/S1.2. CLAUDE §七 "Rust-first" carve-out ("unless the local
design clearly says otherwise") applies: this is governance tooling around the receipt boundary, whose
entire neighborhood is Python. **Portability (CLAUDE §六):** paths derive from
`REPO_ROOT = Path(__file__).resolve().parents[2]`; disposable bundles go to `tempfile` dirs; the
fixed-path runtime's canonical install path (e.g. `/opt/aiml/runtime/<sha256>`) is **recorded as design
data**, never a path the Mac spike writes to. No `/Users/ncyu` or `/home/ncyu` literal.

### 3.2 What the spike does per candidate

For each candidate the harness produces **one** `runtime_candidate_receipt_v1` by evaluating a fixed
seam list. It runs against a **fixed tiny fixture** (a handful of pinned files / a minimal
`requirements.in` with vendored placeholder wheels), never the real ML closure, so it is deterministic
and network-free.

**(A) exact-image-ID OCI — offline evaluation**

1. **Digest-pinning mechanism (OFFLINE_PROVEN, arch-caveated):** author an image *spec* whose base is
   pinned `FROM <python-base>@sha256:<digest>` (by digest, never a mutable tag). Compute the
   **sealed-input digest** over `{Dockerfile bytes, pinned base digest, lockfile bytes, build-context
   manifest}` — deterministic, offline.
2. **Optional single throwaway build (PM-gated, §8.1):** *if* PM authorizes one base-image pull, build
   a **minimal linux/arm64** image with **no network-requiring `RUN`** (base + COPY only), capture its
   **exact image ID / RepoDigest**, and re-inspect to show the ID is stable. This proves "an exact
   image ID is obtainable and content-addressed" — but on **linux/arm64 via colima**, **not** the
   target host, and **without** the ML stack. Classified `LOCAL_REPRODUCIBLE` **with a loud platform
   caveat**; skipped (never faked) if PM declines or the daemon is down.
3. **Native-lib loading, isolation-in-container, cgroup/network/PG:** **DEFERRED S1.6** — these need the
   image *running* on the target host.

**(B) content-addressed fixed-path runtime — offline evaluation**

1. **Content-addressing determinism (OFFLINE_PROVEN):** hash the fixture bundle tree twice → identical
   `sha256`; the hash **is** the bundle's identity and its fixed path `<store>/<sha256>/`. Pure stdlib,
   fully offline, no daemon.
2. **Isolated-mode / no-system-Python-fallback (OFFLINE_PROVEN):** the launch contract invokes
   `<bundle>/bin/python3 -I` by **absolute pinned path** (no `PATH` lookup, no `/usr/bin/python3`
   fallback). The harness demonstrates via the host interpreter that `python3 -I` yields
   `isolated=1, no_user_site=1` and **ignores injected `PYTHONPATH`** (probe §2) — the mechanism is
   real and offline-reproducible.
3. **Non-relocatable / no mutable alias (OFFLINE_PROVEN, structural):** a content-hash path is immutable
   by construction; rollback re-pins a **specific prior hash** in the launch manifest, **not** a mutable
   `current`/`latest` symlink (plan §LR0C line 272 prohibition, machine-checked in the receipt).
4. **Native-lib loading on target arch/OS:** **DEFERRED S1.6** — Mac cannot load Linux `.so`; only the
   *inventory + origin hashing* is offline.

Common to both: **real dependency-closure resolution** (LightGBM/sklearn/ONNX) is `STRUCTURAL_DESIGN`
here and belongs to the **networked sealed-build (LR2)**; **reproducible image/bundle OUTPUT
bit-identity** is `DEFERRED` (nondeterministic layers/timestamps need a builder + CI).

### 3.3 The preliminary comparison (no premature choice)

One `runtime_candidate_comparison_v1` binds the two candidate receipts' `self_digest`s, records a
per-seam **offline** verdict matrix (A vs B), an **explicitly non-binding** `preliminary_lean` string,
the list of `deferred_to_s1_6` decisive seams, and a **`final_choice` const-`null`**. The schema
forbids a non-null `final_choice` at the S1.4 gate — so the S1.4↔S1.6 boundary is **fail-closed**, not
prose. The real choice is S1.6's `learning_runtime_choice_receipt_v1` (PROGRESS row S1.6).

## 4. The honest offline-vs-deferred evidence split (the crux for S1.4)

Per-seam classification. `OFFLINE_PROVEN` = real `LOCAL_REPRODUCIBLE` bytes recapturable on this Mac;
`STRUCTURAL_DESIGN` = design/contract only, no runtime bytes; `DEFERRED_S1_6` = requires the target-host
probe (or a networked sealed-build for closure resolution).

| Seam | (A) OCI — offline | (B) fixed-path — offline | Decisive at |
|---|---|---|---|
| content-addressing / sealing **determinism** | `OFFLINE_PROVEN` (sealed-input digest) | `OFFLINE_PROVEN` (hash tree twice) | S1.4 ✓ |
| sealed **build-input** digest | `OFFLINE_PROVEN` | `OFFLINE_PROVEN` | S1.4 ✓ |
| Python **isolated-mode** `-I` mechanism | `OFFLINE_PROVEN` (host interp) | `OFFLINE_PROVEN` (host interp) | S1.4 ✓ |
| **no-system-python-fallback** (abs pinned interp, no `PATH`) | `STRUCTURAL_DESIGN` (image ENTRYPOINT) | `OFFLINE_PROVEN` (abs-path launch; `-I` ignores env) | S1.4 ✓ |
| **non-relocatable / no mutable tag-alias** | `STRUCTURAL_DESIGN` (image ID immutable by construction) | `OFFLINE_PROVEN` (content-hash path immutable; no `latest` symlink) | S1.4 ✓ |
| exact **image ID / digest** pin obtainable | `LOCAL_REPRODUCIBLE` **but colima linux/arm64 only, no buildx, ≠ target host** (PM-gated build) | N/A | S1.4 (caveated) |
| native-lib **inventory / origin** hashing | `OFFLINE_PROVEN` (mechanism, fixture) | `OFFLINE_PROVEN` (mechanism, fixture) | S1.4 ✓ |
| **rollback bundle shape** | `STRUCTURAL_DESIGN` (prior image ID) | `STRUCTURAL_DESIGN` (prior hash pin) | S1.4 (design) |
| **artifact persistence** model | `STRUCTURAL_DESIGN` (external volume) | `STRUCTURAL_DESIGN` (fixed data path) | S1.4 (design) |
| **real ML dep-closure resolution** (LightGBM/sklearn/ONNX) | `STRUCTURAL_DESIGN` → **networked sealed-build (LR2)** | same | LR2 |
| reproducible **OUTPUT** bit-identity | `DEFERRED` (builder + CI) | `DEFERRED` (builder + CI) | LR2/CI |
| **native-lib loading** on target host | `DEFERRED_S1_6` | `DEFERRED_S1_6` | **S1.6** |
| **cgroup / resource isolation** | `DEFERRED_S1_6` | `DEFERRED_S1_6` | **S1.6** |
| **network denial** at runtime | `DEFERRED_S1_6` | `DEFERRED_S1_6` | **S1.6** |
| **start / stop / failure / cleanup** | `DEFERRED_S1_6` | `DEFERRED_S1_6` | **S1.6** |
| **PG identity** at runtime | `DEFERRED_S1_6` | `DEFERRED_S1_6` | **S1.6** |

**Honest headline (what we can actually prove offline on this Mac):**

1. The seams that are **fully offline-provable** (content-addressing determinism, sealed-input digest,
   isolated-mode, no-system-python-fallback, non-relocatable/no-mutable-alias, native-lib
   inventory-hashing) are **exactly the ones the content-addressed fixed-path candidate (B) satisfies
   with `OFFLINE_PROVEN` LOCAL bytes** — because they are properties of *how you hash a tree and launch
   an interpreter*, which needs no daemon and no target host.
2. The OCI candidate (A)'s **headline advantage is kernel-level isolation + single-digest identity** —
   but isolation (cgroup/network) is precisely the set that is **DEFERRED_S1_6**, and its one locally
   demonstrable win (exact image ID) is **arch-limited (colima linux/arm64, no buildx) and not the
   target host**.
3. Therefore the offline slice yields a **PRELIMINARY, non-binding lean**: *offline evidence is thinner
   for (A) than for (B), but only because (A)'s decisive strengths live in the S1.6 runtime seams.* This
   is **not** a selection. The plan's selection rule ("OCI only if all seams pass; fixed-path if OCI
   cannot") keys off the DEFERRED seams — so **S1.4 must record both candidates as viable-pending-S1.6**
   and let S1.6 decide.

This split is the deliverable: it prevents the classic failure of a spike quietly promoting "I proved
the easy half on my laptop" into "candidate X is chosen."

## 5. `runtime_candidate_receipt_v1` schema shape (one per candidate)

Draft 2020-12, `additionalProperties:false`, `$defs` for `digest`/`timestamp`, mirroring
`pg_readonly_identity_receipt_v1.schema.json`. A frozen `RECEIPT_FIELDS` set is validated exactly.
Required top-level fields:

- `schema_version` — const `"runtime_candidate_receipt_v1"`
- `harness_id` — const `"runtime_candidate_spike_v1"`
- `status` — enum `["PASS","FAIL"]`
- `caller` — string ≥1 (governed caller id)
- `platform` — `{os: enum[darwin,linux], arch: string, python_version: string,
  container_runtime: string|null, container_runtime_available: bool, buildx_available: bool}`
  (pins the Mac reality: colima linux/arm64, buildx absent)
- `candidate` — `{id: enum["exact_image_id_oci","content_addressed_fixed_path"], kind: string}`
- `target_class` — enum `["disposable_offline","target_host"]` — **S1.4 accepts only
  `disposable_offline`; `target_host` rejected fail-closed** (that is S1.6). Mirrors S1.1's
  `disposable_local` gate.
- `evidence_class` — enum `["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"]` — receipt-level **ceiling summary**
  (= the *strongest* non-deferred seam actually exercised: `LOCAL_REPRODUCIBLE` iff at least one
  non-deferred seam is `OFFLINE_PROVEN`, else `STRUCTURAL_ONLY`; consistent with the line-294 gate).
- `dependency_closure` — `{lock_tool: string, lock_input_ref: string, closure_hash: digest,
  hashed_input_count: int≥0, real_ml_closure_resolved: const false}` — offline hashing mechanism;
  **`real_ml_closure_resolved` is const `false`** at S1.4 (resolution is the networked LR2 sealed-build).
- `native_library_inventory` — array of `{name: string, origin: string, sha256: digest,
  load_verified_on_target: const false}` — inventory/origin hashing offline; loading DEFERRED.
- `isolation_mode` — `{python_isolated_mode: bool, ignores_ambient_env: bool,
  system_python_fallback_possible: bool, launch_interpreter: enum["absolute_pinned","path_lookup"],
  evidence_class: enum["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY"]}`
- `sealed_input` — `{sealed_input_digest: digest, inputs: [{ref: string, sha256: digest}],
  reproducible_output_verified: const false}` — sealed BUILD-INPUT digest offline; OUTPUT bit-identity
  deferred, so const `false`.
- `immutability` — `{identity_kind: enum["exact_image_id","content_addressed_path"],
  mutable_tag_or_alias: const false, relocatable_renamed_venv: const false}` — the §LR0C line-272
  prohibitions, **machine-forced false**.
- `artifact_persistence` — `{model: enum["external_volume","fixed_data_path"],
  inside_immutable_image_or_bundle: const false}`
- `rollback` — `{shape: string, prior_identity_pinned: bool, mutable_current_symlink: const false}`
- `oci_build` — nullable OCI-only block: `null` for candidate B; for candidate A
  `{builder: string|null, built: bool, image_id: string|null, image_platform: string|null,
  buildx_multiarch: const false, target_arch_match_verified: const false}` — honest about
  colima-arm64-only / no-buildx / target-arch-unknown.
- `seams` — array (covers the full §4 seam list) of
  `{seam_id: string, offline_evaluable: bool, verdict: enum["OFFLINE_PROVEN","STRUCTURAL_DESIGN",
  "DEFERRED_S1_6"], evidence_class: enum["LOCAL_REPRODUCIBLE","STRUCTURAL_ONLY","DEFERRED"],
  detail_digest: digest, note: string}` — **the honest split, machine-checked.**
- `source_sha256` / `schema_sha256` — digests binding this module + this schema.
- `secret_scan` — `{patterns_checked: [string…], leaked: const false}`.
- `observation_time` / `expires_at` / `ttl_seconds` (int, `[1,3600]`) — same TTL policy as S1.1.
- `failure_reason` — `string|null`.
- `self_digest` — digest over every field except itself.

**`allOf` conditional (fail-closed gate).** If `status=="PASS"` then: `target_class=="disposable_offline"`;
`failure_reason==null`; `dependency_closure.real_ml_closure_resolved==false`;
`immutability.mutable_tag_or_alias==false` **and** `relocatable_renamed_venv==false`;
`sealed_input.reproducible_output_verified==false`; **every runtime-only `seam_id`**
(`cgroup_isolation`, `network_denial`, `start_stop_failure_cleanup`, `pg_identity_runtime`,
`native_lib_loading_target`) is present with `verdict=="DEFERRED_S1_6"` (a PASS receipt that marked any
of these `OFFLINE_PROVEN` is rejected — cannot fake a runtime seam); and `evidence_class` consistent
with the strongest non-deferred verdict. Else `failure_reason` is a non-empty string. Structural/
integrity violations that cannot be safely serialized (secret detected, a runtime seam claimed offline,
`target_host` at the S1.4 gate, ttl out of range) **raise** instead of emitting a receipt (mirrors
S1.1).

**Companion `runtime_candidate_comparison_v1` (preliminary, non-binding).** Fields:
`schema_version` const; `harness_id` const; `candidate_a_digest` / `candidate_b_digest` (the two
receipts' `self_digest`, each a digest); `candidate_status` (`{a,b}`, each enum `["PASS","FAIL"]` —
records each candidate's status so a FAIL candidate stays visible and is never silently presented as
"both evaluated"; the validator can cross-check it against the supplied receipts and does **not**
hard-require PASS); `seam_matrix` (array of `{seam_id, a_verdict, b_verdict,
decisive_at: enum["S1.4","LR2","S1.6"]}`); `deferred_to_s1_6` (array of seam_ids);
`preliminary_lean` (`string|null`, explicitly documented non-binding); **`final_choice` const `null`**;
`source_sha256` / `schema_sha256` / `observation_time` / `expires_at` / `ttl_seconds` / `self_digest`.
The const-`null` `final_choice` is the schema-level guarantee that S1.4 makes no selection.

Reproducibility: `source_sha256` + `schema_sha256` + `platform` + the deterministic fixture ⇒ every
`OFFLINE_PROVEN` seam's `detail_digest` and the `self_digest` are independently re-derivable.

## 7. Boundary compliance & fail-closed posture

- **Offline / disposable only** — fixtures are `tempfile` dirs and pinned bytes; the sole artifacts are
  the reproducible receipt JSONs. No production runtime, no deploy.
- **No target-host probe** — every runtime-only seam (`cgroup_isolation`, `network_denial`,
  `start_stop_failure_cleanup`, `pg_identity_runtime`, `native_lib_loading_target`) is present with
  `verdict=DEFERRED_S1_6`, and the schema **rejects a PASS receipt that marked any of them
  `OFFLINE_PROVEN`**. `target_class=="target_host"` is rejected fail-closed at the S1.4 gate.
- **No final choice** — `runtime_candidate_comparison_v1.final_choice` is const `null`; the real choice
  is S1.6's `learning_runtime_choice_receipt_v1`.
- **Plan §LR0C line-272 prohibitions machine-forced** — `immutability.mutable_tag_or_alias`,
  `relocatable_renamed_venv`, and `rollback.mutable_current_symlink` are all const `false`; the OCI
  candidate pins by digest not tag, the fixed-path candidate pins by content hash not `latest` symlink.
- **No network** — no real ML-closure resolution here; `dependency_closure.real_ml_closure_resolved` and
  `sealed_input.reproducible_output_verified` are const `false`. Real resolution is the LR2 networked
  sealed-build.
- **No secret leakage** — a secret-scan regex over the fully serialized receipt refuses to emit any
  receipt carrying secret-like content (mirrors S1.1, extended for registry/socket paths — the colima
  socket path is redacted to `<home>` in evidence).
- **Portable (CLAUDE §六)** — no host-specific literals; target install path is design data, not written.
- **Fail-closed defaults** — unknown candidate id → reject; `target_host` → reject; any runtime seam
  claimed offline → reject; missing docker daemon → **skip** the OCI image-ID data point (never a false
  PASS); secret detected → refuse to serialize.
