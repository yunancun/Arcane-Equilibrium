## 1. What LR0B requires, and the exact S1.2/S1.3/S1.5 split

Plan §LR0B demands a typed ownership/allowlist matrix for **seven** component effect classes, each with
OPS preflight + PM/operator-approved exact intent + typed Adapter execution + stable observation-window
+ **independent** OPS postcheck ("the actor that applies an effect cannot be its sole verifier"), and
that Registry / effect vocabulary / route compiler / typed intent+result+preflight+postcheck schemas /
closure validator / generated views recognize every admitted class, deriving required effect fields
from work-package classification so a Session cannot self-declare an effectful row as source-only.

Split (delivery protocol lines 145-162):

- **S1.2 (this)** = the *contracts + vocabulary + governance recognition* for all 7 classes, **plus the
  one concrete adapter** (`terminal_receipt_sink` WORM append) proven disposably, **plus** the S1.1
  central-validator wiring. Only ONE adapter is implemented+disposably-proven here.
- **S1.3 (NOT here)** = host UID / PG role / `pg_hba`/`pg_ident` / socket ACL / secret-lifecycle
  *provisioning contracts* + negative ACL tests.
- **S1.5 (NOT here)** = the per-component *deploy Adapter implementations* + remote/platform observation
  seam + disposable apply/rollback/postcheck for the six deploy classes. S1.2 defines their
  contracts/vocabulary; S1.5 implements their adapters.

Consequence: for the six deploy classes S1.2 registers **vocabulary + required-field/recovery
contracts only** (adapter binding `PENDING_S1_5`); it does **not** create their adapter modules,
routable effect nodes, or `closure_packet_v1` bindings. Only the WORM sink is concrete.

## 4. Work-package classification → derive `required_effects`, block self-declared `NONE`

Sibling schema `aiml_component_effect_classification_v1.schema.json` (co-located with the S0.3 schemas)
+ a classifier `classify_component_required_effects(work_package, *, classified_at)` in the central
validator. This is the enforcement point for "a Session cannot self-declare an effectful row as
source-only."

- **Input**: a component work-package declaring `component_effect_class` (one of the 7), the exact
  declared intent fields, `owned_path_manifest`, `direct_interfaces`. Distinct from the S0.3-shaped
  `work_package` — hence a **sibling** schema, not a mutation of `aiml_required_effect_classification_v1`
  (whose `classified_inputs.work_package_id` is `const AIML-S0.3-GOVERNANCE-ADOPTION` and whose
  `effect_class` enum is only `{NONE, EXTERNAL_READONLY_ATTESTATION}` — both frozen by §0).
- **Derivation is from the matrix, not the caller**: the required `effect_class`, `adapter_id`,
  `actor_node_id`, `rollback_contract`, `independent_postcheck_node_id`, and `required_intent_fields`
  are looked up in `AIML_COMPONENT_EFFECT_CLASS_MATRIX`. The caller cannot supply or downgrade them —
  exactly as `classify_required_effects` derives S0.3's effect from `AIML_EFFECT_CLASSIFIER_RULES`.
- **NONE-block (fail-closed)**: if a work-package touches an effectful component surface (declares a
  `component_effect_class` in the 7, or its `owned_path_manifest`/`direct_interfaces` intersect a
  component surface) but declares `effect_class=NONE` / omits the class / supplies a wrong `adapter_id`,
  the classifier **raises** (never emits a `NONE` classification). This mirrors the existing
  `test_required_effect_classification_rejects_caller_downgrade_to_none` guard, extended to the 7
  classes.
- **Output**: an `aiml_component_effect_classification_v1` artifact with `classifier_digest ==
  aiml_component_effect_class_matrix_digest()`, `classification_id`/`self_digest` self-hashed, and a
  single `required_effects` entry the future effectful Session (S1.5/S2.x) binds via its
  `session_attempt_v1.effect_classification_digest`.

`validate_aiml_artifact` gains a branch for `aiml_component_effect_classification_v1` that
**recomputes** the classification from `classified_inputs` and compares (rejecting a hand-forged
`required_effects` or a mismatched `classifier_digest`) — structurally identical to the existing
`aiml_required_effect_classification_v1` branch (L1251-1273), pointed at the sibling classifier.

Boundary: S1.2 defines and validates the component classification **artifact**; it does **not**
generalize `session_attempt_v1` / `_s0_3_work_package_errors` to *emit* one for an arbitrary S1.x
session (that generalization lands when a real component-effect Session runs, S1.5+). This keeps the
frozen S0.3 `session_attempt` path (§0) untouched.

not solely via schema `allOf`/`if`/`then` — `schema_subset_errors` processes `allOf` clauses but does
**not** evaluate `if`/`then`/`else` (verified in `agent_governance_schema.py`), exactly as the S1.1
receipt does its PASS gates in Python.

### 5.2 Adapter functions

- `build_terminal_receipt_append_intent(...) -> intent` + `terminal_receipt_intent_digest()`;
  `idempotency_key = digest(landing_scope_id | terminal_state | terminal_payload_digest)` derived (not
  caller-supplied); `typed_confirm = "terminal-append:{landing_scope_id}:{terminal_state}:{intent_id}"`.
- `apply_terminal_receipt_append(intent, *, store_dir, append_actor_id) -> result` — the disposable
  WORM-emulation apply (§5.3).
- `readback_terminal_receipt(result, *, store_dir, readback_verifier_id) -> ack` — the **independent**
  readback ACK by a **distinct** actor.
- `validate_terminal_receipt_append_intent / _result / _readback_ack(..., now=...)` — structural +
  integrity + digest binding + `target_class=disposable_local_worm_emulation` gate + actor-distinctness
  + idempotency + freshness.

### 5.3 Disposable WORM-emulation proof (no real external service)

Store = a temp directory acting as a **content-addressed, immutable-after-commit** append log. Apply is
record-level transactional:

1. Serialize the terminal payload; `persisted_payload_digest = sha256(payload)`.
2. Write to `store_dir/tmp/<uuid>` (uncommitted).
3. **Atomic commit** via `os.link(tmp, store_dir/records/<idempotency_key>.record)` — `os.link` raises
   `FileExistsError` if the key already exists ⇒ this **is** the idempotency dedup (`IDEMPOTENT_DEDUP`,
   no duplicate record). `os.replace`/`link` is the atomic boundary; an interrupted apply leaves only an
   orphan `tmp/<uuid>` (never a committed record).
4. On successful link, `chmod(record, 0o444)` ⇒ **immutable-after-write**; `unlink(tmp)`.

Proof properties (all real OS semantics, nothing mocked):

| Property | How proven |
|---|---|
| **append idempotent** | second apply, same `idempotency_key` → `os.link` `FileExistsError` → `IDEMPOTENT_DEDUP`; `records/` contains exactly one file |
| **payload hash binds** | readback recomputes `sha256(record bytes)` and requires `== intent.payload_binding.terminal_payload_digest == result.persisted_payload_digest` |
| **distinct-actor readback ACK** | `readback_terminal_receipt(readback_verifier_id ≠ append_actor_id)`; if equal → `same_actor_violation=true`, refused (mirrors frozen contract `same_actor_allowed=false`) |
| **immutable-after-write (prove-negative)** | attempt to rewrite a committed `0o444` record → `PermissionError` observed and recorded |
| **rollback / interruption recovery** | simulate crash between step 2 and 3 (write `tmp` only) → no committed record; `append_status=ROLLED_BACK_INTERRUPTED`; the same `idempotency_key` remains free ⇒ a retry re-appends cleanly (WORM never *mutates* a committed record — "rollback" == the interrupted apply produced none) |
| **external target rejected** | `target_class=external_worm` → fail-closed reject at build/validate |

Teardown removes the temp store. This is explicitly the **disposable proof of the sink mechanism**; the
real external destination binding (S3 Object-Lock / append-only WORM bucket, etc.) is **S8.6**, which
consumes this contract. S1.2 makes no network contact.

### 5.4 Scope decision — no live routing / no `closure_packet` binding in S1.2

The WORM sink's S1.2 proof is **self-validated** by its own tests + the central AIML validator (via
`SCHEMA_FILES` dispatch), exactly as S1.1's `pg_readonly` adapter self-validates. S1.2 therefore does
**not**:

- inject a routable `terminal_receipt_sink` effect node into `agent_governance_routing.route_task`
  (the WORM append is not a `deploy`/service-restart; it has no `operations_needed` trigger), or
- add a `closure_packet_v1` effect-binding dispatch (`validate_effect_evidence` /
  `validate_deploy_effect_binding`) in `agent_governance_effects.py` / `agent_governance_closure.py`.

Both are for **live consumption** and land with S8.6 (terminal validators). Doing them now would create
a closure-admissible runtime-effect path, contradicting `DISPOSABLE_ONLY`. This is the single biggest
scope judgement — flagged for PM in §11.

## 6. S1.1 central-validator wiring (`pg_readonly_identity_receipt_v1`) — CC review note D2

S1.1 deliberately left central registration to S1.2 (its design §2.1, §8.2). In
`aiml_gate_receipt_validator.py`:

1. **`SCHEMA_FILES` +=** `"pg_readonly_identity_receipt_v1": "pg_readonly_identity_receipt_v1.schema.json"`
   (schema already present in `SCHEMA_DIR`).
2. **`validate_aiml_artifact` branch** for `pg_readonly_identity_receipt_v1` **delegates** to
   `agent_governance_pg_readonly_identity.validate_pg_readonly_identity_receipt(artifact,
   require_success=<param>, now=now)` — never relies on the generic `schema_subset_errors` alone
   (which ignores the schema's `if/then` PASS conditional).

This satisfies CC note **D2** exactly:
- **(a) passes `now` for freshness** — the S1.1 validator already accepts `now` and, per its docstring,
  "consumers such as the S1.2 central validator MUST pass `now`"; a receipt observed outside
  `observation_time ≤ now < expires_at` is rejected.
- **(b) disposable-real/attested only, never structurally-hand-built** — the S1.1 validator recomputes
  `source_sha256()` and `schema_sha256()` from the on-disk adapter/schema bytes and requires an exact
  match, requires `target_class=disposable_local` (production fail-closed), and for a `PASS` requires
  the genuine runtime facts a stub cannot fabricate: `session_read_only=on`, write-denial SQLSTATE
  `25006` (read_only_sql_transaction, not a mere `42501`), and a real `25006`-denied persistent
  `search_path` hijack. A structural stub (`evidence_class=STRUCTURAL_ONLY`, forged codes, or wrong
  source/schema binding) fails.

Honesty caveat (preserved, not papered over): a `LOCAL_REPRODUCIBLE` receipt cannot self-authenticate
its own execution (CLAUDE.md Typed Authority Matrix). The central validator's gate is
structural+integrity+freshness+disposable-real-markers; final runtime authenticity of the disposable
proof remains a trusted-host replay (out of S1.2 scope). The wiring threads `require_success` so a
future PG-observation consumer (S2.x) can demand a fresh **PASS** disposable-real receipt.

### 10.1 Accepted residuals / S8.6 follow-ups

The following are **accepted P3 residuals** of the S1.2 disposable adapter — recorded here so they are
tracked, not silently carried. Each is closed or generalized by S8.6 live consumption, not by S1.2:

1. **Actor separation is opaque-string equality, not authenticated identity.** Independence
   (`readback_verifier_id != append_actor_id`; `applier_is_not_sole_verifier`) is enforced by comparing
   opaque identity strings (`verifier == append_actor`). A genuine **authenticated** verifier↔append-actor
   identity binding (`route_task` / closure identities) is an **S8.6 seam**; S1.2 proves the *structural*
   independence rule, not cryptographic actor authenticity.
2. **A standalone `APPENDED` result validates structurally but is inert without its paired intent+ACK.**
   When no intent is supplied at the central gate the result validates *structurally only* (no
   cross-binding is available); it is **not**, by itself, proof of a committed, independently-verified
   append. Consumers MUST NOT treat a lone result as commit/independence proof — the paired intent
   (approval) and the distinct-actor readback ACK are load-bearing (mirrors the readback-ACK
   "independence cannot be verified without its paired result" central-gate rule).
3. **Secret scan covers structured secret shapes only.** `_contains_secret_like` / `WORM_SECRET_LIKE_RE`
   detect credential-assignment / token / auth-scheme *shapes*; a bare high-entropy or opaquely-encoded
   value carrying no secret shape is **not** detected. This is acceptable because the payload contract is
   **digests / fingerprints only** (non-secret by construction) and the scan is defense-in-depth, not a
   general DLP. Stated explicitly as a limitation.
4. **`O_NOFOLLOW` guards the final path component only.** The `records/<key>.record` read/commit refuses
   to follow a symlinked *final component*; a symlinked `records/` **directory** would be written through.
   That requires same-uid write into the (now `0o700`, adapter-owned) store — an attacker already inside
   the trust boundary — so it is out of scope for the disposable proof and subsumed by the S8.6
   real-destination ACL contract.
