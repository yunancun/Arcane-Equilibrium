# S2.2A — LR1 Scoped Compatibility Identity (Design)

**Status**: SOURCE-LANDED (source-only, `NONE`-effect). Session S2.2A / LR1.
Branch `agent/aiml-s2-2a-lr1-compat`. No production PG / runtime / engine /
broker / order / network / systemd effect. No `TODO.md` / `PROGRESS.md` /
`memory/` / `requirements-ml.txt` mutation. Git owned by PM.

This design replaces whole-repo-HEAD liveness for the ALR scanner-snapshot
event consumer with a positive `learning_runtime_digest` computed over only the
learning-relevant source surface, and threads that identity through three bind
points (preflight, spawn, finalize).

---

## 1. The defect

`alr_event_consumer.py::run_event_consumer` used to call
`verify_runtime_source_head(source_head, ...)` before any DB use. That helper
runs `git rev-parse HEAD` and does an **exact string compare** against the
pinned `source_head`, raising `source_head_mismatch` and refusing the **entire**
event consumer on any difference.

Because the consumer drives all Scanner-snapshot ingestion, **any docs-only
commit moves HEAD and stops ingestion.** Whole-repo HEAD is far too broad a
liveness key: it couples raw evidence capture to every unrelated change
(README, unrelated modules, other lanes).

## 2. The manifest — `learning_runtime_manifest_v1`

New SSOT module `program_code/ml_training/learning_runtime_manifest.py` builds a
positive, frozen-allowlist manifest. Identity helpers are **reused** from
`aiml_gate_receipt_validator.py` (`canonical_digest`, `artifact_self_digest`);
no new hashing abstraction is introduced.

```
schema_version = "learning_runtime_manifest_v1"
generated_at_utc                      # wall clock; NOT hashed into identity
repo_source_head    (40-hex)          # TELEMETRY ONLY; never in any component digest
capture_contract  { digest, inputs:{<root-rel path>:<sha256hex>},
                    snapshot_feature_schema_version }
training_contract { digest, components:{
                      learning_code_digest, migration_fingerprints{V151..V160},
                      feature_contract_digest, label_contract_digest,
                      action_policy_digest, dependency_lock_digest,
                      runtime_config_digest } }
boundary = "source_only_none_effect"
self_digest                           # == learning_runtime_digest
```

Per-file content hash = `sha256(path.read_bytes())`, paths root-relative and
sorted. Component/identity digests use `canonical_digest` (`sha256:` + canonical
JSON). **`self_digest` (the `learning_runtime_digest`) is computed over only
`{schema_version, boundary, capture_contract.digest, training_contract.digest}`**
— deliberately excluding `generated_at_utc` and `repo_source_head`, so it is
stable across docs-only commits and across regenerations. This is the key that
makes "docs-only never stops ingestion": docs-only moves only `repo_source_head`
(telemetry), never the component digests, so `self_digest` is unchanged.

    `EDGE_P3_FEATURE_SCHEMA_VERSION` (imported from `parquet_etl.py`).

    required fields (`label_schema_hash`, `label_config_hash`,

    `regime_oos_label_contract.py` file sha256. *Deviation*: whole-file hash of
    the regime contract (robustly covers its `SCHEMA_VERSION`) is used instead
    of importing/executing that research module at runtime preflight.

    `alr_candidate_policy._POLICY_KEYS`).

    LR2/S2.3 and is **not** built here.
  - `runtime_config_digest` = `canonical_digest` over the policy template
    sha256 + the arbiter runtime constants read from the template
    (`algorithm_version`, `tie_break_version`, `q18_scale`, `thresholds`,
    `cooldown_seconds`, `unknown_portfolio_penalty`). systemd unit/env is **not
    in-repo** (an S2.4 effect) and is deliberately not covered.

## 3. The three bind points

All three bind the **same** immutable identity (identical `self_digest`).

1. **Preflight** — `alr_event_consumer.py::run_event_consumer` (via
   `_preflight_source_compatibility`). Builds the actual checkout manifest,
   evaluates compatibility against the operator-supplied
   `--expected-learning-runtime-digest` (env
   `ALR_EXPECTED_LEARNING_RUNTIME_DIGEST`), **stops capture only if
   `capture_status != COMPATIBLE`** (build failure / missing input), and
   propagates `fit_quarantined = (fit_status != COMPATIBLE)` into
   `event_consumer_loop`. `verify_runtime_source_head` is **kept but demoted**
   to a non-fatal telemetry helper (`_telemetry_source_head_match`); its exact
   contract and its direct test stay green.
2. **Spawn** — `alr_challenger_training_contract.py::build_alr_challenger_training_contract`.
   `learning_runtime_digest` is a required `code_manifest` field; when the new
   `expected_learning_runtime_digest` pin is supplied it must equal it, else
   `learning_runtime_digest_mismatch` refuses spawn.
3. **Finalize** — `alr_challenger_fit_capture_attestation.py`
   (`_normalize_actual_training_inputs`). `learning_runtime_digest` now travels
   through `expected_training_inputs` (added in the result contract, mirroring
   `source_head`); finalize asserts the attested-code value equals the expected
   value, else `actual_learning_runtime_digest_mismatch`. (It is also bound
   transitively by the existing `code_manifest_material` byte-check and
   `code_manifest_hash`.)

## 4. Quarantine semantics

- **Capture** continues unless the capture surface is itself incompatible
  (`INCOMPATIBLE`) or the manifest cannot be built (`INDETERMINATE`,
  fail-closed). Docs-only ⇒ capture keeps running.
- **Fit / promotion** is quarantined (`QUARANTINE`) whenever the training
  contract digest drifts from the reviewed pin, or the manifest is
  indeterminate. Quarantine fences the fit-triggering candidate-projection
  transition in the consumer loop, and refuses spawn / finalize.
- **Deny-by-default**: any build error — missing allowlisted input, symlink,
  non-regular file, unreadable — ⇒ `INDETERMINATE` on **both** capture and fit.

## 5. Receipt

`source_compatibility_receipt_v1` (schema
`program_code/ml_training/schemas/aiml_gate_receipts/source_compatibility_receipt_v1.schema.json`,
registered in `aiml_gate_receipt_validator.SCHEMA_FILES`). The validator adds
identity cross-checks: receipt `self_digest`, `learning_runtime_digest ==
manifest.self_digest`, and `capture/training_contract_digest` +
`migration_fingerprints` all bind the embedded manifest. The emitted
`SOURCE_READY` instance is
`docs/execution_plan/ai_ml_landing/receipts/S2.2A-source-compatibility-receipt-v1.json`,
generated by running `build_source_compatibility_receipt()` against this
checkout (every digest is real) and passing `validate_aiml_artifact`.

## 7. Scope guards

- **No lockfile here.** `dependency_lock_digest` hashes the `requirements-ml.txt`
  spec text; the sealed lock is LR2/S2.3.
- **runtime_config = in-repo only.** systemd unit/env is an S2.4 effect and is
  not covered.
- **V151..V160 exact** (span asserted; no gaps).
- **Source-only, `NONE`-effect.** S2.2B issues the runtime DONE (operational
  wiring: unit env `ALR_EXPECTED_LEARNING_RUNTIME_DIGEST`, real ingestion
  observation). This Session lands source + tests + receipt only.
