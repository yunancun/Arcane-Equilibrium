## 0. The overriding tension — and how S1.6 resolves it honestly

**S1.6's effect is a TARGET-HOST probe, but Mac cannot run a real target-host (Linux) runtime probe.** The
plan (§LR0C 261-265) wants a `TARGET_HOST_DISPOSABLE_RUNTIME_PROBE` "in an isolated unit/container for each
candidate" exercising cgroup/resource isolation, network denial, PG identity, native-library loading,
start/stop, failure recovery, rollback and cleanup — **then** select exactly one. Mac has none of the
decisive kernel machinery (S1.4 §2 ground-truth: colima linux/arm64 VM only, **no buildx**, target-host arch
unknown, cannot load Linux `.so`, cannot own a target-host systemd slice).

The honest resolution has three moves, each schema-enforced (not merely promised):

1. **Run what CAN run disposably** — an ISOLATED disposable runtime probe for BOTH candidates using S1.5's
   `LEARNING_RUNTIME` `temp_dir_artifact` disposable apply/rollback/postcheck lifecycle on a throwaway local
   target (a `0o700` temp deploy root + content-addressed bundle store + atomic active-generation pointer).
   This yields **real `LOCAL_REPRODUCIBLE`** evidence for the lifecycle seams (§3, §4).
2. **Record the honest disposable-vs-target-host-deferred split** — every kernel/host-only seam
   (cgroup/resource isolation, network denial at the kernel, native-lib LOADING on the target, true
   target-host start, networked ML-closure resolution) is marked `DEFERRED_TARGET_HOST`; a receipt that
   marked any of them proven-on-Mac is **rejected** (§5 — the exact fail-closed guard S1.4 used for its
   runtime-only seams).
3. **Make the final choice under the plan's own asymmetric rule** — OCI is selectable **only if all seams
   pass** (§6); on Mac its decisive isolation seams are `DEFERRED_TARGET_HOST` (unprovable) *and* LR2
   normatively forbids the OCI socket/DBus model. Under the rule "**fixed-path if OCI cannot satisfy
   them**", the choice is **content-addressed fixed-path**, recorded with the exact rule-based reason. The
   choice is **BINDING** on the *selection* (which runtime LR2/S2.3 seals and LR0 installs), while it
   attests **nothing running** and honestly records `target_host_probe_performed=false`,
   `production_running_attested=false` (§6.3).

This mirrors every prior S1 session's discipline: S1.1/S1.3/S1.4/S1.5 all make the disposable gate and the
const-false boundary flags **machine-forced**, never prose.

## 2. Minimum-coherent design — ONE probe harness, reusing S1.5's lifecycle as the probe effect actor

Mirroring S1.5's decisive insight (one data-driven adapter, not six), S1.6 builds **one** self-contained,
self-validating harness that **reuses the already-registered `learning_runtime_deploy_adapter_v1`** (S1.5,
registry `status=declared_disposable_apply_rollback_postcheck_implemented`, matrix
`adapter_binding_status=IMPLEMENTED_DISPOSABLE`) as the **probe effect actor for both candidates**. The
`LEARNING_RUNTIME` matrix contract is a near-exact fit for a runtime probe (verified from the central
validator):

```
required_intent_fields = [runtime_identity, dependency_manifest_digest,
                          mount_network_socket_secret_surface, exact_rollback]
recovery_contract      = exact_rollback
disposable_target_kind = temp_dir_artifact
adapter_id             = learning_runtime_deploy_adapter_v1
```

- `runtime_identity` ← the candidate identity (exact image ID **or** content-addressed path hash);
- `dependency_manifest_digest` ← the immutable dependency-closure hash (`hash_bundle_tree`, S1.4);
- `mount_network_socket_secret_surface` ← the isolation-surface descriptor (**this is where the OCI-socket
  concern is recorded** — for OCI, an OCI/DBus socket surface LR2 forbids; for fixed-path, none);
- `exact_rollback` ← the rollback contract (atomic pointer swap back to the prior content hash).

New module `helper_scripts/maintenance_scripts/agent_governance_runtime_choice_probe.py`, a purpose-built
sibling of `agent_governance_runtime_candidate_spike.py` (S1.4) and `agent_governance_component_effects.py`
(S1.5). It:

- **reuses** S1.5's lifecycle as the probe effect actor — `build_component_effect_intent(effect_class=
  "LEARNING_RUNTIME", …)`, `artifact_deploy_root_init` → `artifact_apply` (start) →
  `artifact_apply_interrupted` (failure) → `artifact_rollback` (stop) → `artifact_state_digest`
  (pre==post exact restoration), `build_component_effect_result`, `build_postcheck_attestation` (distinct
  verifier) — **all read-only imports**; it hand-codes **no** apply/rollback/postcheck (deletion test:
  a second lifecycle that differs only by candidate data fails it);
- **reuses** S1.4's `CANDIDATE_OCI`/`CANDIDATE_FIXED_PATH` ids, `hash_bundle_tree`, and binds the two S1.4
  candidate receipts + the const-`null` comparison **by digest** (read-only);
- **reuses** S1.1's disposable RO-PG identity probes for the disposable PG-identity seam (`initdb`-gated;
  skips where PG binaries absent — never a false PASS);
- **emits** the `learning_runtime_choice_receipt_v1` (its own builder + validator);
- is **stateless** (pure functions + a per-call disposable temp dir), so **no singleton-table
  registration**.

Language: **Python, stdlib-first** (`hashlib`/`json`/`os`/`shutil`/`tempfile`), matching the whole AIML
governance surface and S1.1/S1.4/S1.5; the CLAUDE §七 "Rust-first" carve-out ("unless the local design
clearly says otherwise") applies — governance tooling in a Python-only neighborhood, the same carve-out PM
accepted for S1.1–S1.5. **Portability (CLAUDE §六):** `REPO_ROOT = Path(__file__).resolve().parents[2]`; the
throwaway deploy root is a `tempfile` dir; canonical production paths (`/opt/aiml/runtime/<sha256>`, unit
names) are **recorded as design data**, never written. No `/Users/ncyu` / `/home/ncyu` literal.
**New code comments default Chinese** (CLAUDE §七), matching the S1.4/S1.5 modules.

## 3. The isolated disposable runtime probe (per candidate, reusing S1.5's lifecycle)

For **each** candidate the harness runs one probe = one S1.5 `LEARNING_RUNTIME` disposable lifecycle against
a fresh throwaway deploy root, mapping the plan's probe verbs onto the content-addressed generation
transitions:

```
runtime_identity + closure hash → build LEARNING_RUNTIME intent (approved, ops-preflight, exact_rollback)
  artifact_deploy_root_init(prior bundle)          → pre-state digest (the "installed prior runtime")
  START      = artifact_apply(new bundle)          → atomic pointer swap → applied_digest (new generation active)
  observe    = artifact_state_digest == applied     (stable observation window; APPLIED)
  FAILURE    = artifact_apply_interrupted(bundle)   → pointer never swapped → prior generation stays active
                                                      (ROLLED_BACK_INTERRUPTED: failure-recovery proof)
  STOP/ROLLBACK = artifact_rollback(prior_hash)     → atomic swap back → post_rollback_digest
  assert  pre_state_digest == post_rollback_digest  (exact restoration, machine-checked by S1.5)
  INDEPENDENT postcheck by a DISTINCT verifier_node → re-reads the deploy root, re-derives post-rollback digest
  CLEANUP    = teardown of the throwaway deploy root, THEN a distinct cleanup verifier confirms the path is
               gone (complete cleanup) — done AFTER the postcheck's read; if it is not gone the probe RAISES,
               so complete_cleanup is DISPOSABLE_PROVEN only when the teardown is confirmed
```

Seam → mechanism, per candidate:

| Probe seam (plan §LR0C) | Disposable mechanism (S1.5/S1.4/S1.1 reuse) | Honest scope |
|---|---|---|
| **start / stop** | `artifact_apply` (activate a content-addressed generation) / `artifact_rollback` (deactivate) | the **lifecycle mechanism** — a real filesystem state transition, **not** an OS process |
| **failure recovery** | `artifact_apply_interrupted` → prior generation stays active (`ROLLED_BACK_INTERRUPTED`) | real interruption-recovery |
| **rollback** | `artifact_rollback` → `pre == post` (exact restoration) | real, machine-checked |
| **complete cleanup** | teardown of the `0o700` temp deploy root + distinct-verifier confirmation it is gone | real |
| **immutable dependency closure** | `hash_bundle_tree` (S1.4) + `chmod 0o444` immutable-after-write bundle | real closure hashing (not networked resolution) |
| **artifact persistence** | bundle store `bundles/<sha256>/` + atomic `active_generation` pointer surviving the swap | real |
| **PG identity** | S1.1 disposable RO-PG identity probes on a throwaway cluster (`initdb`-gated) | **disposable** PG identity only |

**The representativeness asymmetry (honest, load-bearing).** The `temp_dir_artifact` lifecycle **is the
content-addressed fixed-path candidate's native runtime shape** → for candidate B the proven seams are
**directly representative**. For the OCI candidate, Mac cannot run a target-host container, so the same
content-addressed lifecycle is a **stand-in** — it exercises "a content-addressed generation lifecycle",
**not** OCI's kernel/daemon runtime. The receipt records this per candidate as
`representativeness ∈ {native_shape, content_addressed_standin}` so the probe never pretends the OCI run was
a real OCI runtime. **The probe therefore does not "prefer" B on evidence** — both candidates run the same
lifecycle; the choice rests on §6's rule + the LR2 normative constraint, not on a disposable probe that
discriminates them.

## 4. The honest disposable-vs-target-host-deferred seam split (the crux for S1.6)

`DISPOSABLE_PROVEN` = real `LOCAL_REPRODUCIBLE` bytes recapturable on this Mac via the S1.5 lifecycle;
`STRUCTURAL_ONLY` = contract/design, no runtime bytes; `DEFERRED_TARGET_HOST` = requires the real Linux
target-host probe (or a networked sealed-build for closure resolution).

| Seam | (A) OCI | (B) fixed-path | Decisive at |
|---|---|---|---|
| start / stop **lifecycle** | `DISPOSABLE_PROVEN` (stand-in) | `DISPOSABLE_PROVEN` (native shape) | S1.6 ✓ |
| failure recovery | `DISPOSABLE_PROVEN` (stand-in) | `DISPOSABLE_PROVEN` (native) | S1.6 ✓ |
| rollback (exact restoration) | `DISPOSABLE_PROVEN` (stand-in) | `DISPOSABLE_PROVEN` (native) | S1.6 ✓ |
| complete cleanup | `DISPOSABLE_PROVEN` | `DISPOSABLE_PROVEN` | S1.6 ✓ |
| immutable dependency closure (hashing) | `DISPOSABLE_PROVEN` | `DISPOSABLE_PROVEN` | S1.6 ✓ |
| artifact persistence model | `DISPOSABLE_PROVEN` | `DISPOSABLE_PROVEN` | S1.6 ✓ |
| **disposable** PG identity | `DISPOSABLE_PROVEN` (initdb-gated) | `DISPOSABLE_PROVEN` (initdb-gated) | S1.6 ✓ |
| **cgroup / resource isolation** (kernel) | `DEFERRED_TARGET_HOST` | `DEFERRED_TARGET_HOST` | **S2.5** |
| **network denial at the kernel** | `DEFERRED_TARGET_HOST` | `DEFERRED_TARGET_HOST` | **S2.5** |
| **native-library LOADING on target** | `DEFERRED_TARGET_HOST` | `DEFERRED_TARGET_HOST` | **S2.5** |
| **true target-host start** (real unit/PID) | `DEFERRED_TARGET_HOST` | `DEFERRED_TARGET_HOST` | **S2.5** |
| **target-host PG identity** (real role on trade-core) | `DEFERRED_TARGET_HOST` | `DEFERRED_TARGET_HOST` | **S2.5** |
| networked ML-closure **resolution** | `DEFERRED_TARGET_HOST` → LR2 | same | **S2.3** |
| exact **image ID** obtainable | arch-limited: colima linux/arm64, **no buildx**, ≠ target (S1.4) | N/A | S1.4/S1.6 (caveated) |
| **no OCI socket / DBus** (LR2 306-316) | **normatively FORBIDDEN for controller/workers** | satisfied (no socket surface) | **normative** |

**Honest headline.** The lifecycle/closure/artifact/disposable-PG seams are **`DISPOSABLE_PROVEN` for both
candidates** (real S1.5 filesystem transitions + real S1.1 `42501` denial). The **decisive** isolation seams
(cgroup, network denial, native-lib loading, true start, target-host PG identity) are **`DEFERRED_TARGET_HOST`
for both** — never faked. The one seam that decides between the candidates **now** is **normative, not a
Mac runtime fact**: LR2 forbids controller/workers an OCI socket/DBus, which the OCI runtime model needs and
the fixed-path model does not. This split is the deliverable: it prevents the classic spike failure of
promoting "I ran a temp-dir lifecycle on my laptop" into "the runtime is proven on the target host."

### 5.1 `allOf` PASS gate, enforced in the Python validator (the selection rule, machine-encoded)

`schema_subset_errors` processes `allOf` but not `if/then` (S1.1–S1.5 precedent), so the crux lives in the
Python validator. `status=="PASS"` ⇒:

- `target_class=="disposable_local"`; `failure_reason==null`;
- **both candidates probed** — exactly two `candidate_probes`, ids `{exact_image_id_oci,
  content_addressed_fixed_path}`, each with a bound `lifecycle_result_digest` + `postcheck_attestation_digest`;
- **exact restoration per candidate** — `pre_state_digest == post_rollback_digest`;
- **non-no-op lifecycle per candidate** — `applied_digest != pre_state_digest` (the validator ports S1.5's
  no-op defense: a synthetic probe whose apply changed nothing cannot certify a PASS);
- **`LOCAL_REPRODUCIBLE` per candidate** — each PASS `candidate_probe` and each of its proven seams must be
  `evidence_class == LOCAL_REPRODUCIBLE`; a `STRUCTURAL_ONLY` probe MUST NOT validate as a PASS (the validator
  closes the forge where the builder FAILs it but a label-trusting validator passed it);
- **disposable PG-identity seam bound to 42501** — whenever `disposable_pg_identity` appears under
  `disposable_seams_proven`, the seam must carry `evidence_digest == canonical_digest({observed_sqlstate:
  "42501", verdict: "DENIED"})` (the validator ports the builder's `_validate_pg_identity_evidence`; the seam
  cannot be claimed without bound 42501 evidence);
- **distinct verifier per candidate** — `postcheck_verifier_node != apply_actor_node`;
- **every `target_host_deferred_seam` is present with `verdict=="DEFERRED_TARGET_HOST"`** and **no**
  target-host-only seam appears under `disposable_seams_proven` (a receipt that marked cgroup / network /
  native-lib-loading / true-start / target-host-PG as disposably-proven is **rejected** — the S1.4
  runtime-seam guard, re-applied);
- **the selection rule** — `oci_selectable == (target_host_probe_performed AND every OCI
  target_host_deferred_seam passed)`; since `target_host_probe_performed==false` on Mac ⇒ `oci_selectable`
  must be `false` ⇒ **`final_choice` must equal `content_addressed_fixed_path`** (OCI cannot be selected
  without all seams passing, which requires a target-host probe);
- `production_running_attested==false`; every `boundary.*` const holds; `unselected_path_removal.
  forecloses_downstream==true`;
- `dependency_receipts` bind the S1.4 A/B/comparison + S1.5 receipt digests; `component_effect_matrix_digest`
  equals the live central digest; `secret_scan.leaked==false`;
- **freshness is mandatory for a PASS acceptance** — `validate_...(require_success=True)` requires a non-null
  `now` and rejects a stale receipt (an omitted `now` would silently skip the freshness check, letting an
  expired receipt pass a PASS acceptance).

Else `failure_reason` is a non-empty string. **Unsafe states raise** (never emit a receipt), mirroring
S1.1/S1.4/S1.5: a target-host seam claimed disposably-proven, `final_choice==exact_image_id_oci` while
`target_host_probe_performed==false`, `production_running_attested==true`, a `production`/`target_host`
target at the Mac gate, a non-exact candidate restoration, an applier==verifier, a secret in any serialized
field, or `ttl` out of range.

**Why encode the rule as `oci_selectable`.** Making `oci_selectable` a derived, gated boolean means the
schema itself carries the plan's asymmetric rule: a hypothetical future Linux probe that *did* run a
target-host probe and *did* pass all OCI seams could set `target_host_probe_performed=true` and select OCI —
but on the Mac deliverable that path is fail-closed to fixed-path. The rule is data, not prose. The
affirmative branch's would-be verdict constant (`PASSED_TARGET_HOST`) is **intentionally unreachable in this
v1 (Mac) receipt version** — v1's schema pins every deferred seam's verdict to `DEFERRED_TARGET_HOST`, so
`oci_selectable` is structurally `false`; the constant is reserved for a future Linux target-host receipt
version (S2.5). Likewise `binding=PROVISIONAL_PENDING_LINUX` is present but **unused**, because `BINDING` is
the PM-confirmed choice (§6.3, §11-Q1).

### 5.2 Acceptance contract — the validator is load-bearing, but PASS probe evidence is a self-attested summary

The standalone validator no longer rubber-stamps the `candidate_probes` labels: it re-verifies the builder's
guarantees from the receipt bytes it *can* re-derive — `applied_digest != pre_state_digest` (no-op defense),
`evidence_class == LOCAL_REPRODUCIBLE` for a PASS probe and its proven seams, the `disposable_pg_identity`
seam's bound 42501 `evidence_digest`, exact restoration, distinct verifier, the closed seam-set, the
selection rule, the const boundary, and (for a PASS acceptance) freshness.

What the validator **cannot** re-derive from receipt bytes alone is honestly bounded, mirroring S1.3's
evidence-honesty consumer contract and CLAUDE.md ("a canonical self-digest proves integrity only, never
authenticity"):

- `lifecycle_result_digest` / `postcheck_attestation_digest` are hashes of the S1.5 result/attestation
  objects, which are **not carried in the receipt**; the validator can only format-check them. A
  `LOCAL_REPRODUCIBLE` PASS's per-candidate probe evidence is therefore a **self-attested summary**: a
  consumer relying on this receipt as the S1-exit runtime probe **MUST re-run the builder or obtain
  trusted-host attestation** — it must not treat the receipt bytes alone as authenticity.
- the bound S1.4 A/B/comparison digests are only **format-checked** by the standalone validator (it cannot
  fetch them). The **trusted-host re-validation MUST re-fetch the S1.4 comparison by digest and re-assert
  `final_choice is null`** (this does not block S1.6, whose own `final_choice` is independently forced to the
  fixed-path by the machine-encoded rule, and whose `supersedes_comparison_null` records the binding).

This is the same posture as S1.5: the harness self-validates for integrity/structure, but a PASS is
authenticated only by an out-of-band trusted-host capability (the CLAUDE.md Closure rule).

## 6. The final choice — fixed-path, BINDING, defensible under the plan's own rule

### 6.1 The rule is asymmetric

Plan §LR0C 267-268: "exact-image-ID OCI, **only if all seams pass**; or content-addressed fixed-path
runtime, **if OCI cannot satisfy them**." OCI must clear a **positive** bar (all seams pass); fixed-path is
the **specified fallback**. On the available *and attainable* Mac evidence, OCI's decisive isolation seams
are `DEFERRED_TARGET_HOST` (unprovable — no target-host container runtime, no buildx, arch≠target), so OCI's
"all seams pass" precondition **is not met**. Under the rule, the fallback applies.

### 6.2 LR2 decides against OCI on normative grounds, independent of any probe

Plan §LR2 311: "Host systemd owns lifecycle; **controller/workers receive neither OCI socket nor DBus
authority**." An OCI-daemon runtime model requires exactly the socket authority LR2 forbids for the
workers. This is a **`normative_policy`** fact (accepted plan), decidable **now**, not a runtime
observation — so even a *successful* future OCI target-host probe would still violate LR2's least-privilege
constraint. Fixed-path (an absolute-pinned `<bundle>/bin/python3 -I` launch, no socket surface) is the shape
LR2 already blesses. S1.3 already encoded this: its `controller-oci-dbus` over-grant is a rejected topology.

### 6.3 Recommendation: BINDING selection, target-host EVIDENCE deferred

**The selection is BINDING** (fixed-path is what LR2/S2.3 seals and LR0 installs; the OCI path is
foreclosed). It is binding because it rests on grounds decidable **now** without a target-host probe:
(1) the asymmetric rule (OCI's positive precondition is unattainable in the available evidence);
(2) LR2's normative no-OCI-socket/DBus constraint; (3) fixed-path's positive offline (S1.4:
`no_system_python_fallback` + `non_relocatable` `OFFLINE_PROVEN`) **and** disposable-lifecycle (S1.6:
start/stop/failure/rollback/cleanup) evidence.

What remains DEFERRED is **not the choice** but the target-host **evidence for the chosen runtime**: the real
cgroup/network/native-load/running attestation of the *fixed-path* runtime binds on Linux at **S2.3** (sealed
build) + **S2.5** (running attestation). The receipt records `target_host_probe_performed=false` and
`production_running_attested=false` honestly. The only thing that could reopen the choice is a **normative**
change (a future ADR relaxing LR2's no-OCI-socket constraint) — not a mere Linux probe result. This is
exactly the shape the plan endorses: "**Runtime choice is not a running attestation**" — a binding choice
without a running proof.

The alternative — `binding=PROVISIONAL_PENDING_LINUX` — would defer the *choice* itself to an OPS-run Linux
target-host probe. It is more conservative but (a) S2.3 depends on S1.6 and needs an actionable selection to
seal a build, (b) the plan lists the final choice receipt as an S1.6 deliverable that reaches
`EFFECT_SEAMS_READY`, and (c) the decisive discriminator (LR2 no-OCI-socket) is normative and would not
change under any probe. **Recommended: `BINDING`.** Flagged for PM as §9-Q1 (the one genuine judgment call).

### 6.4 "Remove the unselected production path"

Plan §LR0C 271-272. **There is no AIML OCI production artifact to delete**: S1.4 was design-only (its
`OCI_DOCKERFILE_SPEC` is a Python constant / design bytes at `agent_governance_runtime_candidate_spike.py:220`,
and `/opt/aiml/runtime` appears only *inside* that constant, never as a written file — verified). The one
committed `Dockerfile` in the repo (`program_code/exchange_connectors/bybit_connector/control_api_v1/
Dockerfile`) is the unrelated Bybit connector control API, **not** the AIML learning runtime, and is out of
S1.6's scope. So the "removal" is recorded, not a filesystem delete:
`unselected_production_artifact_present=false`, `production_path_removed=true` (vacuously), and — the
load-bearing part — `forecloses_downstream=true`: **S2.3/LR2 seals ONLY the fixed-path runtime**, and no
downstream session may create an OCI Dockerfile/build/install path. The plan's "no mutable tag/alias,
relocatable renamed venv, system-Python fallback or dual production stack" (§LR0C 272-273) is preserved:
selecting exactly one shape forecloses the dual stack.

## 9. Bypass-negatives (fail-closed — the airtight boundary the S1 exit requires)

Each REALLY triggers the rejection path (no rubber stamp; the S1.5 `run_bypass_negative` "vacuous rejection"
discipline applies), recorded in the receipt as `REJECTED`:

1. `target_host_running_attestation_claimed` — a receipt with `production_running_attested=true` (or any
   `boundary.*` running flag true) → **raise/reject**. *Can't claim a target-host running attestation.*
2. `oci_selected_without_target_host_probe` — `final_choice==exact_image_id_oci` while
   `target_host_probe_performed==false` → reject. *Can't select OCI without all seams passing (needs the
   target-host probe).*
3. `oci_selected_without_all_seams_passing` — even with a target-host probe present, `final_choice==oci`
   while any OCI `target_host_deferred_seam` is unproven/failed → reject.
4. `selection_without_both_candidates_probed` — `final_choice` set but fewer than two `candidate_probes`, or
   a candidate missing its `lifecycle_result_digest`/`postcheck_attestation_digest` → reject. *Can't select
   without both candidates probed.*
5. `deferred_seam_claimed_disposably_proven` — a target-host-only seam (cgroup / network-denial /
   native-lib-loading / true-start / target-host-PG) marked `DISPOSABLE_PROVEN` → reject (the S1.4
   runtime-seam guard, re-applied).
6. `disposable_probe_not_exact_restoration` — a candidate probe whose `pre != post` (rollback not exact) →
   reject (reuse S1.5 `NonExactRollbackError`).
7. `applier_is_sole_verifier` — a candidate probe with `apply_actor_node == postcheck_verifier_node` →
   reject (reuse S1.5 `ApplierIsSoleVerifierError`).
8. `production_or_target_host_target_on_mac` — `target_class ∈ {production, target_host}` at the Mac gate →
   fail-closed reject.
9. `comparison_final_choice_mutated` — the bound S1.4 comparison digest does not match a comparison whose
   `final_choice` is null (someone tried to feed a mutated non-null comparison, or edited S1.4) → reject.
   *Protects the S1.4 const-null boundary (§7.1).*
10. `unselected_path_not_foreclosed` — `final_choice==fixed_path` but `unselected_path_removal.
    forecloses_downstream != true` → reject (must foreclose the OCI path).
11. `matrix_digest_tamper` — `dependency_receipts.component_effect_matrix_digest != live central digest` →
    reject (reuse the S1.5 tamper guard).
12. `plaintext_secret_ingress` — a secret-like value anywhere in a serialized receipt → refuse to serialize
    (`SECRET_LIKE_RE`-style scan, S1.1/S1.4/S1.5 style).

Plus the positive **disposable** proofs (§3, one real start/stop/failure/rollback/cleanup lifecycle per
candidate, distinct verifier) and the standard receipt-integrity suite (field-set, digest re-derivation,
TTL/time order, boundary consts machine-false, `secret_scan`).

## 10. Boundary compliance & fail-closed posture

- **`TARGET_HOST_DISPOSABLE_RUNTIME_PROBE`, disposable-local only on Mac** — the probe is a real temp-dir
  lifecycle, torn down; `target_class ∈ {production, target_host}` fail-closed; `target_host_probe_performed`
  honestly `false`; the receipt claims **no** target-host running runtime.
- **No production running claim** — `production_running_attested`, `real_target_host_probe_on_mac`,
  `real_process_started`, `native_lib_loaded_on_target`, `kernel_isolation_exercised`, `network_contact` all
  const `false`. "Runtime choice is not a running attestation" (plan) — the receipt selects a runtime and
  forecloses the loser; it does not claim the runtime is running.
- **Every decisive target-host seam is DEFERRED_TARGET_HOST** — cgroup / network-denial / native-lib-loading
  / true-start / target-host-PG are present with the deferred verdict, and the schema/validator **reject** a
  receipt that marked any of them disposably-proven (the S1.4 runtime-seam guard).
- **Final choice defensible under the plan's own rule** — OCI selectable only if all seams pass; unmet on
  Mac + LR2 forbids the OCI socket ⇒ fixed-path, recorded reason (§6). `oci_selectable` const `false` on the
  Mac deliverable; the rule is machine-encoded (§5.1).
- **S1.4 untouched; comparison stays const-null** — the choice lives in the new receipt, which binds the
  comparison by digest and does not edit it (§7.1).
- **No network / no remote contact** — the probe makes no network call and touches no remote host.
- **Nine authorities stay FALSE** — S1.6 is a disposable probe + a decision artifact; it grants no runtime /
  build / PG / deploy / broker / order authority.
- **Portable (CLAUDE §六)** — no host-specific literals; canonical target paths are design data, not written.
- **Fail-closed defaults** — target-host seam claimed disposable → reject; OCI selected without a target-host
  probe → reject; fewer than both candidates probed → reject; non-exact restoration → reject; applier ==
  verifier → reject; mutated S1.4 comparison → reject; secret detected → refuse to serialize; missing PG
  binaries → **skip** the disposable PG-identity facet (never a false PASS).

### 10.1 Accepted residuals (tracked; closed by S2.3 / S2.5)

1. **The disposable probe proves the lifecycle MECHANISM, not a real target-host runtime.** The
   `temp_dir_artifact` transitions are real filesystem acts; a real systemd unit / running process / kernel
   cgroup / network namespace / native `.so` load / real role identity on trade-core is
   `DEFERRED_TARGET_HOST` → S2.5. For the OCI candidate the disposable lifecycle is a `content_addressed_standin`,
   not a container run (§3).
2. **Independence is opaque-string distinctness, not authenticated identity** (same as S1.5 §13.1-1).
   `verifier_node != apply_actor_node` compares identity strings; a cryptographically authenticated
   verifier↔applier binding is an S2.5 seam.
3. **The BINDING choice is a selection, not a running proof.** It is binding on which runtime LR2/S2.3 seals
   and LR0 installs; it is falsifiable only by a normative change (an ADR relaxing LR2's no-OCI-socket
   constraint), not by a Linux probe. The chosen fixed-path runtime's sealed build (S2.3) and running
   attestation (S2.5) still bind on Linux.
4. **A `LOCAL_REPRODUCIBLE` PASS's probe evidence is a self-attested summary (integrity, not authenticity).**
   The validator re-verifies everything derivable from receipt bytes (§5.2), but `lifecycle_result_digest` /
   `postcheck_attestation_digest` hash S1.5 result objects absent from the receipt, and the bound S1.4
   A/B/comparison digests are only format-checked (the standalone validator cannot fetch them). A consumer
   relying on this receipt as the S1-exit runtime probe MUST re-run the builder or obtain trusted-host
   attestation, and the trusted-host re-validation MUST re-fetch the S1.4 comparison by digest and re-assert
   `final_choice is null`. This does not block S1.6: its own `final_choice` is independently forced to the
   fixed-path by the machine-encoded rule (§5.1) and `supersedes_comparison_null` records the binding.
