## 0. Operating Boundary

AEG-S0 exists because ADR-0047 and AMD-2026-05-31-01 changed the next safe
step from "start backfill" to "freeze evidence contracts first".

Hard boundaries:

- E1 remains blocked until these contracts receive the required role review.
- Bull-heavy data may be used only with explicit labels.
- Bybit public market endpoints are raw state inputs, not prediction.
- News, X, Reddit, and market commentary are secondary side evidence only.
- Paper is not a promotion evidence lane.
- `panel.*` short-retention surfaces are not 18mo alpha-history.
- `market.market_tickers.index_price` and `market.market_tickers.mark_price`
  remain excluded from historical basis evidence until the known persistence
  bug is fixed or bypassed.

## 1. AEG-S0-W0-S1 Evidence Storage Contract

### 1.1 Purpose

Every Alpha-Edge run must emit an artifact set that makes the run reconstructable
without relying on chat state, current survivors, mutable scanner state, or
unlabeled market-regime assumptions.

The contract is artifact-first. AEG-S1 may later add database tables, but S0 does
not create or reserve a migration number.

### 1.2 Canonical Artifact Root

Future runners should resolve the artifact root from runtime configuration:

```text
${OPENCLAW_DATA_DIR:-/tmp/openclaw}/alpha_history_runs/<run_id>/
```

The default path mirrors existing runtime convention, but production code must
not hard-code machine-specific paths. Reports copied into git should link the
runtime artifact path and include the manifest digest, not paste large generated
payloads into TODO.

### 1.3 Required Artifact Set

Each run must write these files:

| Artifact | Required | Purpose |
|---|---:|---|
| `manifest.json` | yes | Stable identity, run inputs, code/runtime provenance, classifier version, endpoint list, cost model. |
| `coverage.csv` or `coverage.parquet` | yes | Per endpoint/symbol/timeframe expected-vs-observed coverage and failures. |
| `universe.csv` | yes | Point-in-time symbol universe with active, delisted, delivering, closed, and exclusion reasons. |
| `regime_labels.parquet` | yes for scoring | Main regime and overlay flags generated by the frozen classifier. |
| `breadth_ladder.parquet` | yes for verdict | Core25, scanner-active, top-liquidity 40-50, and full survivorship/cohort diagnostics. |
| `side_evidence.json` | optional | Narrative/event context, explicitly secondary and excluded from promotion gates. |
| `execution_realism.json` | yes for verdict | Fees, slippage, maker-fill/capacity assumptions, order availability, latency assumptions. |
| `verdict_matrix.parquet` | yes for verdict | Candidate x regime x breadth x freshness x survivorship x execution-realism matrix. |

```json
  "engine_binary_sha256": "optional-for-runtime-runs",

  "timezone": "UTC",
  "universe_id": "stable-id-or-digest",
  "universe_sources": [],
  "symbol_count": 0,
  "endpoint_contract_version": "aeg.bybit_endpoint_contract.v0.1",
  "endpoints": [],
  "source_tables": [],
  "retention_snapshot": [],
  "provenance_mode": "artifact_manifest|db_ledger|timeframe_namespace",
  "cost_model_version": "string",
  "cost_model": {},

```

Rules:

- `run_id` must be globally unique and present in every artifact.
- `git_sha` must be the source checkout used to generate the run.
- `closed_bar_cutoff_utc` must be <= the earliest scoring timestamp; current or
  partial candles are excluded from evidence.
- `universe_sources` must include `market.symbol_universe_snapshots` when a
  survivorship-aware run is claimed.
- `source_tables` must distinguish raw long-history tables from `panel.*`
  short-retention derived surfaces.
- `provenance_mode` cannot be an empty string. If rows are co-mingled in an
  existing table, the manifest must say how the run is reconstructed.

### 1.5 Coverage Contract

`coverage` rows must be keyed by:

```text
run_id, endpoint_id, storage_surface, symbol, timeframe_or_period,
window_start_utc, window_end_utc
```

Required metrics:

- `expected_rows`
- `observed_rows`
- `coverage_pct`
- `min_ts_utc`
- `max_ts_utc`
- `gap_count`
- `failed_page_count`
- `nonzero_retcode_count`
- `timeout_count`
- `partial_result` boolean
- `first_error_code`
- `first_error_msg`

Fail-closed rules:

- Bybit timeout or nonzero `retCode` cannot fabricate rows.
- `retCode=10006` may be retried with explicit backoff and must be counted.
- A partial symbol must remain partial in coverage; no padding.
- Backfilled klines must not fabricate `tick_count`; use `NULL` if the endpoint
  does not provide the value.

### 1.6 Universe Contract

The universe artifact must be point-in-time and must reject current-survivor-only
shortcuts.

Required columns:

```text
run_id, universe_id, asof_utc, exchange, category, symbol, status,
listed_at_utc, delisted_at_utc, is_delisted_at_asof, source_uri,
source_snapshot_ts_utc, included, inclusion_reason, exclusion_reason
```

Minimum sources:

- `market.symbol_universe_snapshots` for historical Bybit symbol status.
- Current scanner-active universe for operational overlap.
- Bybit instruments-info for current and historical metadata where available.

Acceptance:

- If the analytical window contains delisted/closed symbols, the universe must
  include at least one `is_delisted_at_asof=true` or `status IN ('Delivering',
  'Closed')` symbol, or it must explicitly prove none exist in-window.
- A universe containing only currently trading scanner symbols fails.
- Listing and delisting timestamps must be used to mask each symbol to its
  tradable lifetime; missing early history cannot be padded.

### 1.7 Storage Surface Rules

Allowed long-history surfaces after their specific gates:

- `market.klines` for OHLCV after retention/provenance decision.
- `market.funding_rates`, `market.open_interest`, and
  `market.long_short_ratio` only after retention or dedicated research storage
  is decided for the target window.
- New dedicated research hypertables only after MIT migration design, Linux PG
  dry-run, double-apply, and E2/E4 review.

Explicit exclusions:

- `panel.funding_rates_panel`, `panel.oi_delta_panel`,
  `panel.basis_panel`, and sibling `panel.*` tables are 14d derived surfaces.
  They may support recent diagnostics, not 18mo alpha-history proof.
- `market.market_tickers.index_price` and `market.market_tickers.mark_price`
  are excluded from historical basis until `P3-MARKET-TICKERS-INDEX-MARK-DEAD-PERSISTENCE`
  is fixed or a documented bypass is chosen.
- `market.news_signals` and future narrative stores are side evidence only.

### 1.8 AEG-S1 Storage Gate

Before any E1 backfill:

1. PM/operator must choose retention or dedicated-storage path.
2. MIT must size PG growth and reflect live Timescale policies on Linux.
3. If a new table or retention mutation is used, V### migration rules apply.
4. BB must approve endpoint semantics and rate discipline.
5. Coverage and manifest artifacts must be implemented before alpha scoring.

## 2. AEG-S0-W0-S2 Regime Classifier Freeze

### 2.1 Version

Classifier version: `aeg_regime_v0.1.0`.

Any threshold change, feature addition, or label-priority change creates a new
classifier version and invalidates direct comparison with older verdicts unless
both versions are reported side by side.

### 2.2 Inputs

Primary input:

- `market.klines` daily bars (`timeframe='1d'`) when the run has accepted deep
  daily history.

Fallback input:

- `market.klines` 4h bars resampled to daily close only when daily bars are not
  available for a symbol/window. The fallback must be labeled.

Supplemental inputs:

- funding, OI, long/short ratio, premium/index/mark klines, and orderbook data
  may add overlay flags after their endpoint/storage contracts pass. They do
  not override the primary price-regime label.

Excluded inputs:

- current-day partial candles,
- future funding settlements,
- narrative sentiment,
- exchange-provided "trend" interpretations,
- candidate PnL or strategy-specific outcome variables.

### 2.3 Point-In-Time Rule

For a signal timestamp `t`, every feature must satisfy:

```text
feature_ts <= t - one_complete_bar
```

Implementation rule:

- pandas-style runners use `.shift(1)` after rolling features.
- SQL runners use joins where `feature_bar_close_ts_ms <= signal_ts_ms -
  bar_ms`.
- Any `rolling(N).max/min/mean/std` used in scoring must have a shifted
  counterpart or the run fails leak-free review.

### 2.4 Required Features

Computed per symbol, and also for BTCUSDT as the market anchor when present:

| Feature | Definition |
|---|---|
| `ret_30d` | log close return over 30 closed daily bars. |
| `ret_90d` | log close return over 90 closed daily bars. |
| `rv_30d` | realized volatility of daily log returns over 30 bars. |
| `rv_90d` | realized volatility of daily log returns over 90 bars. |
| `trend_z_30` | `ret_30d / max(rv_30d, epsilon)`. |
| `ma_50` | 50-bar SMA on shifted closes. |
| `ma_200` | 200-bar SMA on shifted closes when enough context exists. |
| `efficiency_30` | `abs(close[t-1] - close[t-31]) / sum(abs(diff(close)))` over 30 bars. |
| `direction_flip_30` | share of adjacent daily returns whose sign flips over 30 bars. |
| `rv_30d_percentile_365` | percentile of `rv_30d` against prior 365 closed bars when available. |

Minimum context:

- Full classifier: 200 closed daily bars.
- Reduced classifier: 90 closed daily bars.
- Insufficient context: fewer than 90 bars; report `insufficient_context=true`
  and block promotion use for that slice unless PM/QC explicitly scopes a
  short-history diagnostic.

### 2.5 Main Regime Rules

Rules are evaluated in this order:

1. `bull`
   - `ret_90d >= +0.15`, and
   - `trend_z_30 >= +0.8`, and
   - `close[t-1] > ma_50`, and
   - when `ma_200` exists, `ma_50 >= ma_200`.
2. `bear`
   - `ret_90d <= -0.15`, and
   - `trend_z_30 <= -0.8`, and
   - `close[t-1] < ma_50`, and
   - when `ma_200` exists, `ma_50 <= ma_200`.
3. `high-vol`
   - `rv_30d_percentile_365 >= 0.80` when available, or
   - reduced context: `rv_30d >= 1.5 * rv_90d`,
   - and neither `bull` nor `bear` already matched.
4. `chop`
   - `efficiency_30 < 0.25`, and
   - `direction_flip_30 >= 0.45`.
5. `range`
   - default non-directional regime when no prior rule matches.

The BTCUSDT anchor is used for market-level overlays, not to override a
symbol-level label. Candidate reports must include both the symbol regime and
the market-anchor regime when both are available.

### 2.6 Overlay Flags

Every scoring window must report these flags where applicable:

| Flag | Trigger |
|---|---|
| `bull_heavy` | More than 60% of scored bars are `bull`. |
| `bear_heavy` | More than 60% of scored bars are `bear`. |
| `range_or_chop_heavy` | More than 60% of scored bars are `range` or `chop`. |
| `high_vol_heavy` | More than 40% of scored bars are `high-vol` or have high-vol overlay. |
| `2024_dominated` | More than 50% of independent observations are in calendar year 2024. |
| `stale_year_dominated` | More than 50% of independent observations are older than 365 days from run end. |
| `recent_window_weak` | Full-window positive result but rolling recent 90d net <= 0. |
| `low_breadth` | Fewer than the required symbols/cohorts are available for the claimed breadth. |
| `insufficient_context` | Classifier minimum context is not met. |
| `funding_extreme` | Funding percentile is >= 95th or <= 5th after storage gate. |
| `oi_expansion` | OI growth percentile is >= 80th after storage gate. |

Promotion rule:

- A bull-only, stale-only, survivor-only, or narrative-only positive is never a
  `durable-alpha candidate`.
- Such results must become `regime-bet / learning-only`, `stale-data artifact`,
  `breadth-limited`, or `insufficient evidence`.

### 2.7 Classifier Acceptance

Before alpha scoring:

- QC approves label thresholds and confirms they were fixed before candidate
  scoring.
- MIT verifies point-in-time feature joins and `.shift(1)` usage.
- PA verifies the classifier does not couple to a specific candidate's PnL.
- PM records the classifier version in TODO or the relevant report.

## 3. AEG-S0-W0-S3 Bybit Endpoint Contract

### 3.1 Endpoint Matrix

| Endpoint | AEG use | Current local posture | Storage gate |
|---|---|---|---|
| `GET /v5/market/kline` | Primary OHLCV history for trend/momentum. | Rust reference exists; Python replay client allowlists kline but uses lower page size. | `market.klines` retention/provenance decision before >12mo backfill. |
| `GET /v5/market/mark-price-kline` | Execution realism, liquidation-distance sensitivity. | Rust reference exists; Python replay client lacks endpoint. | New storage or artifact-only decision required before history use. |
| `GET /v5/market/index-price-kline` | Basis/regime inputs when basis history is needed. | Rust reference exists; Python replay client lacks endpoint. | Do not rely on dead `market_tickers` historical mark/index; store separately or artifact-only. |
| `GET /v5/market/premium-index-price-kline` | Funding/premium regime features. | Rust reference exists; Python replay client lacks endpoint. | Storage decision required. |
| `GET /v5/market/funding/history` | Funding realized carry and funding-extreme overlays. | Rust reference exists. | `market.funding_rates` is 180d retention; 18mo use needs retention/storage decision. |
| `GET /v5/market/open-interest` | OI trend/participation overlays. | Rust reference exists; request key must be `intervalTime`. | `market.open_interest` is 180d retention; 18mo use needs retention/storage decision. |
| `GET /v5/market/account-ratio` | Long/short sentiment overlay, not primary alpha proof. | Rust reference exists; official period docs have known drift history. | `market.long_short_ratio` is 180d retention; 18mo use needs retention/storage decision. |
| `GET /v5/market/tickers` | Current liquidity, best bid/ask, funding cap/interval fields. | Rust/Python local clients exist. | Latest snapshot only; not historical proof unless locally persisted. |
| `GET /v5/market/orderbook` | Execution realism and capacity checks. | Rust/Python local clients exist. | Latest snapshot only; historical proof needs local capture. |
| `GET /v5/market/instruments-info` | PIT universe, launch/delist metadata, funding cap/interval. | Rust/Python local clients exist; pagination mandatory. | `market.symbol_universe_snapshots` is the PIT source; direct endpoint can refresh/verify. |
| `GET /v5/market/historical-volatility` | Optional option-IV side context. | Rust reference exists. | Not required for linear-perp core; use only after BB/MIT says coverage is relevant. |

### 3.2 Official Semantics To Preserve

Verified against official Bybit docs on 2026-05-31:

- Kline endpoints return reverse-sorted candle arrays and support `limit` up to
  1000 for futures-style klines.
- Funding history uses `startTime`/`endTime`, `limit <= 200`, and passing only
  `startTime` is an error.
- Open interest uses `intervalTime`, `limit <= 200`, and cursor pagination.
- Long/short ratio uses `/v5/market/account-ratio`, `period`, `limit <= 500`,
  and cursor pagination.
- Instruments-info default page size is not enough for linear; cursor pagination
  or `limit=1000` is mandatory.
- Tickers expose current `indexPrice`, `markPrice`, `fundingIntervalHour`, and
  `fundingCap`, but this is a latest snapshot, not history.
- REST orderbook is snapshot format and does not include RPI orders.

### 3.3 Rate And Failure Discipline

Minimum policy:

- Use a dedicated public-data client isolated from live order/auth clients.
- Default backfill throttle should be gentle, e.g. 2-5 requests/sec, regardless
  of higher exchange limits.
- Read `X-Bapi-Limit*` headers when the client surface exposes them.
- Retry only documented transient/rate cases with bounded exponential backoff.
- Any final nonzero `retCode`, timeout, parse failure, or cursor non-advance
  must become a coverage failure, not fabricated data.
- Pagination must have max-page guards and non-advance cursor guards.

### 3.4 Client Gaps Before E1

Before implementation, E1/BB must choose and verify one client path:

1. Extend the isolated Python replay public client to include the AEG endpoint
   allowlist and per-endpoint rate policy, or
2. Use Rust `MarketDataClient`/engine-side public client path where endpoints
   already exist, with a runner that cannot touch order/auth state.

Known local gaps to address or explicitly accept:

- Python replay client currently allowlists only kline/tickers/orderbook.
- Python replay client uses `_KLINE_LIMIT=200`; official kline limit allows up
  to 1000, but lower page size is acceptable only if runtime estimates account
  for it.
- Rust instruments parser currently does not persist funding caps into
  `SymbolSpec`; if a strategy needs funding cap, parse
  `fundingInterval`/`fundingIntervalHour`/`fundingCap` or equivalent fields
  explicitly and update the Bybit reference in the same commit.
- `market_tickers` mark/index columns are not trusted as historical source.

### 3.5 BB Review Gate

BB review must verify:

- endpoint path and parameter names,
- pagination semantics,
- maximum page size,
- retCode and timeout fail-closed behavior,
- endpoint availability for delisted/closed symbols in the target window,
- docs reference update for every new or changed endpoint use,
- no order, position, wallet, auth, or private endpoint is introduced.
