## 1. AEG-S0-W0-S1 Evidence Storage Contract

### 1.1 Purpose

Every Alpha-Edge run must emit an artifact set that makes the run reconstructable
without relying on chat state, current survivors, mutable scanner state, or
unlabeled market-regime assumptions.

The contract is artifact-first. AEG-S1 may later add database tables, but S0 does
not create or reserve a migration number.

### 1.2 Canonical Artifact Root

Future runners should resolve the artifact root from runtime configuration:

```text
${OPENCLAW_DATA_DIR:-/tmp/openclaw}/alpha_history_runs/<run_id>/
```

The default path mirrors existing runtime convention, but production code must
not hard-code machine-specific paths. Reports copied into git should link the
runtime artifact path and include the manifest digest, not paste large generated
payloads into TODO.

### 1.3 Required Artifact Set

Each run must write these files:

| Artifact | Required | Purpose |
|---|---:|---|
| `manifest.json` | yes | Stable identity, run inputs, code/runtime provenance, classifier version, endpoint list, cost model, child artifact digests. |
| `artifact_index.json` | yes | Per artifact `path`, `sha256`, `byte_size`, `row_count`, and `schema_version`; may be embedded in `manifest.json` if identical. |
| `coverage.csv` or `coverage.parquet` | yes | Per endpoint/symbol/timeframe expected-vs-observed coverage and failures. |
| `universe.csv` | yes | Point-in-time symbol universe with active, delisted, delivering, closed, and exclusion reasons. |
| `regime_labels.parquet` | yes for scoring | Main regime and overlay flags generated by the frozen classifier. |
| `feature_lineage.parquet` | yes for scoring | Per feature source timestamp, join rule, lag, and leak-violation audit. |
| `breadth_ladder.parquet` | yes for verdict | Core25, scanner-active, top-liquidity 40-50, and full survivorship/cohort diagnostics. |
| `side_evidence.json` | optional | Narrative/event context, explicitly secondary and excluded from promotion gates. |
| `execution_realism.json` | yes for verdict | Fees, slippage, maker-fill/capacity assumptions, order availability, latency assumptions. |
| `verdict_matrix.parquet` | yes for verdict | Candidate x regime x breadth x freshness x survivorship x execution-realism matrix. |

### 1.5 Coverage Contract

`coverage` rows must be keyed by:

```text
run_id, endpoint_id, storage_surface, symbol, timeframe_or_period,
window_start_utc, window_end_utc, effective_window_start_utc,
effective_window_end_utc
```

Required metrics:

- `expected_rows`
- `expected_rows_basis`
- `observed_rows`
- `coverage_pct`
- `coverage_threshold_pct`
- `coverage_gate_status`
- `min_ts_utc`
- `max_ts_utc`
- `gap_count`
- `failed_page_count`
- `nonzero_retcode_count`
- `timeout_count`
- `partial_result` boolean
- `first_error_code`
- `first_error_msg`
- `excluded_from_promotion` boolean
- `exclusion_reason`

Fail-closed rules:

- Bybit timeout or nonzero `retCode` cannot fabricate rows.
- `retCode=10006` may be retried with explicit backoff and must be counted.
- A partial symbol must remain partial in coverage; no padding.
- Backfilled klines must not fabricate `tick_count`; use `NULL` if the endpoint
  does not provide the value.
- `expected_rows` must be computed after listed/delisted lifetime masking. The
  `expected_rows_basis` field must state whether the cadence is fixed bar,
  funding interval, cursor response, or endpoint-defined settlement.

Coverage hard gates:

| Surface | Minimum gate before promotion use |
|---|---|
| Primary OHLCV regime/scoring bars | `coverage_pct >= 0.98` per symbol/effective lifetime and no missing row inside any signal lookback. Missing lookback rows exclude those signal timestamps. |
| Funding/OI/long-short overlays | `coverage_pct >= 0.95` for the overlay window. Below gate means the overlay is unavailable and cannot support promotion. |
| PIT universe source | 100% of included symbols have a lifetime mask or an explicit `unknown_lifetime` exclusion. |
| Feature lineage | `leak_violation_count = 0`; otherwise the affected slice is excluded. |

Any slice below its gate must be excluded from `verdict_matrix.parquet` PASS
rows and labeled `insufficient evidence`, `breadth-limited`, or the narrower
failure label. It may remain in diagnostics only if `excluded_from_promotion`
is true.

## 2. AEG-S0-W0-S2 Regime Classifier Freeze

### 2.1 Version

Classifier version: `aeg_regime_v0.1.0`.

Any threshold change, feature addition, or label-priority change creates a new
classifier version and invalidates direct comparison with older verdicts unless
both versions are reported side by side.

### 2.2 Inputs

Primary input:

- `market.klines` daily bars (`timeframe='1d'`) when the run has accepted deep
  daily history.

Fallback input:

- `market.klines` 4h bars resampled to daily close only when daily bars are not
  available for a symbol/window. The fallback must be labeled and requires all
  six UTC 4h bars for the daily bucket; otherwise that daily bucket is missing.

Supplemental inputs:

- funding, OI, long/short ratio, premium/index/mark klines, and orderbook data
  may add overlay flags after their endpoint/storage contracts pass. They do
  not override the primary price-regime label.

Excluded inputs:

- current-day partial candles,
- future funding settlements,
- narrative sentiment,
- exchange-provided "trend" interpretations,
- candidate PnL or strategy-specific outcome variables.

### 2.3 Point-In-Time Rule

For a signal timestamp `t`, every feature must satisfy:

```text
feature_ts <= t - one_complete_bar
```

Implementation rule:

- pandas-style runners use `.shift(1)` after rolling features.
- SQL runners use joins where `feature_bar_close_ts_ms <= signal_ts_ms -
  bar_ms`.
- Any `rolling(N).max/min/mean/std` used in scoring must have a shifted
  counterpart or the run fails leak-free review.

### 2.4 Deterministic Feature Conventions

These conventions are part of `aeg_regime_v0.1.0`:

- `epsilon = 1e-12`.
- Daily return is natural log close-to-close return on closed UTC daily bars.
- Realized volatility uses sample standard deviation (`ddof=1`) of daily log
  returns; fewer than 2 returns is insufficient context.
- Percentile rank uses the prior window only, excluding the current value:
  `count(prior <= value) / count(prior)`.
- `direction_flip_30` counts adjacent nonzero return sign changes. Zero-return
  pairs are excluded from the denominator; if fewer than 10 valid adjacent
  pairs remain, report insufficient context for this feature.
- 4h-to-daily fallback buckets are UTC days, require six closed 4h bars, and
  use the last closed 4h close as the daily close. Partial buckets are missing.

### 2.5 Feature Lineage Artifact

`feature_lineage.parquet` is mandatory before scoring. Required columns:

```text
run_id, classifier_version, signal_ts_utc, symbol, feature_name, source_table,
source_endpoint, source_ts_utc, bar_close_ts_utc, feature_bar_ms, lookback_bars,
join_rule_version, lag_ms, leak_violation_count
```

Acceptance:

- `lag_ms` must be >= one complete source bar for every scoring feature.
- `leak_violation_count` must be 0 for every promoted slice.
- The artifact must cover both symbol-level features and BTCUSDT market-anchor
  features when the anchor is present.
- A run that cannot produce feature lineage may be diagnostic only; it cannot
  emit promotion verdicts.

### 2.6 Required Features

Computed per symbol, and also for BTCUSDT as the market anchor when present:

| Feature | Definition |
|---|---|
| `ret_30d` | log close return over 30 closed daily bars. |
| `ret_90d` | log close return over 90 closed daily bars. |
| `rv_30d` | realized volatility of daily log returns over 30 bars. |
| `rv_90d` | realized volatility of daily log returns over 90 bars. |
| `trend_z_30` | `ret_30d / max(rv_30d, epsilon)`. |
| `ma_50` | 50-bar SMA on shifted closes. |
| `ma_200` | 200-bar SMA on shifted closes when enough context exists. |
| `efficiency_30` | `abs(close[t-1] - close[t-31]) / sum(abs(diff(close)))` over 30 bars. |
| `direction_flip_30` | share of adjacent daily returns whose sign flips over 30 bars. |
| `rv_30d_percentile_365` | percentile of `rv_30d` against prior 365 closed bars when available. |

Minimum context:

- Full classifier: 200 closed daily bars.
- Reduced classifier: 90 closed daily bars.
- Insufficient context: fewer than 90 bars; report `insufficient_context=true`
  and block promotion use for that slice unless PM/QC explicitly scopes a
  short-history diagnostic.

### 2.7 Main Regime Rules

Rules are evaluated in this order:

1. `bull`
   - `ret_90d >= +0.15`, and
   - `trend_z_30 >= +0.8`, and
   - `close[t-1] > ma_50`, and
   - when `ma_200` exists, `ma_50 >= ma_200`.
2. `bear`
   - `ret_90d <= -0.15`, and
   - `trend_z_30 <= -0.8`, and
   - `close[t-1] < ma_50`, and
   - when `ma_200` exists, `ma_50 <= ma_200`.
3. `high-vol`
   - `rv_30d_percentile_365 >= 0.80` when available, or
   - reduced context: `rv_30d >= 1.5 * rv_90d`,
   - and neither `bull` nor `bear` already matched.
4. `chop`
   - `efficiency_30 < 0.25`, and
   - `direction_flip_30 >= 0.45`.
5. `range`
   - default non-directional regime when no prior rule matches.

The BTCUSDT anchor is used for market-level overlays, not to override a
symbol-level label. Candidate reports must include both the symbol regime and
the market-anchor regime when both are available.

`high-vol` also has an independent overlay. If bull or bear matches first but
the high-vol trigger is true, the main regime stays `bull` or `bear` and
`high_vol_overlay=true`.

### 2.8 Overlay Flags

Every scoring window must report these flags where applicable:

| Flag | Trigger |
|---|---|
| `bull_heavy` | More than 60% of scored bars are `bull`. |
| `bear_heavy` | More than 60% of scored bars are `bear`. |
| `range_or_chop_heavy` | More than 60% of scored bars are `range` or `chop`. |
| `high_vol_overlay` | `rv_30d_percentile_365 >= 0.80` when available, or reduced context `rv_30d >= 1.5 * rv_90d`, regardless of main regime. |
| `high_vol_heavy` | More than 40% of scored bars are `high-vol` or have `high_vol_overlay=true`. |
| `2024_dominated` | More than 50% of independent observations are in calendar year 2024. |
| `stale_year_dominated` | More than 50% of independent observations are older than 365 days from run end. |
| `recent_window_weak` | Full-window positive result but rolling recent 90d net <= 0. |
| `low_breadth` | Fewer than the required symbols/cohorts are available for the claimed breadth. |
| `insufficient_context` | Classifier minimum context is not met. |
| `funding_extreme` | Funding percentile is >= 95th or <= 5th after storage gate. |
| `oi_expansion` | OI growth percentile is >= 80th after storage gate. |

Promotion rule:

- A bull-only, stale-only, survivor-only, or narrative-only positive is never a
  `durable-alpha candidate`.
- Such results must become `regime-bet / learning-only`, `stale-data artifact`,
  `breadth-limited`, or `insufficient evidence`.

### 2.9 Verdict Matrix And Promotion Gates

`verdict_matrix.parquet` minimum columns:

```text
run_id, candidate_id, strategy_family, parameter_cell_id, symbol, cohort_id,
regime, market_anchor_regime, overlay_flags, breadth_cohort, freshness_bucket,
survivorship_mode, execution_realism_mode, coverage_gate_status,
feature_lineage_status, gross_bps, cost_bps, net_bps, net_to_cost_ratio,
is_sharpe, oos_sharpe, psr_0, dsr_k, pbo, multiple_test_family, k_trials,
n_independent, sample_unit, recent_90d_net_bps, recent_180d_net_bps,
non_bull_independent_pass, final_label, reject_reasons
```

Global minimum gates for `durable-alpha candidate`:

1. `coverage_gate_status=PASS` and `feature_lineage_status=PASS`.
2. Cost model fixed before scoring; `net_bps > 0` and
   `net_to_cost_ratio >= 2.0`.
3. `n_independent >= 30` after clustering overlapping symbol/time samples.
4. `psr_0 >= 0.95`.
5. `dsr_k >= 0.95`, where `k_trials` is the full tested parameter/candidate
   family for the run.
6. `pbo < 0.50` when CSCV/PBO is computable for the family; if not computable,
   the report must say why and QC must explicitly accept the substitute.
7. `oos_sharpe >= 0.5 * is_sharpe`; `oos_sharpe < 0.3 * is_sharpe` is a kill.
8. Freshness: rolling recent 90d net must be > 0. If 180d data exists, rolling
   recent 180d net must also be > 0. Otherwise label `stale-data artifact` or
   `insufficient evidence`.
9. Regime robustness: at least one non-bull slice (`bear`, `range`, `chop`, or
   `high-vol` main regime or overlay) must independently pass gates 1-7. If
   non-bull sample is insufficient, label `regime-bet / learning-only` or
   `insufficient evidence`.
10. Survivorship: current-survivor-only universes fail. The verdict must pass a
    PIT universe / delisted-or-proved-none check.
11. Narrative side evidence cannot change a failing mathematical verdict into a
    passing verdict.

`n_independent` counting rule:

- Multi-day time-series strategies count non-overlapping holding-period windows
  after clustering highly correlated symbols at the same timestamp.
- Cross-sectional strategies count one independent rebalance timestamp after
  market-beta / BTC-beta clustering; the number of symbols in the same rebalance
  is breadth, not independent time evidence.
- The chosen `sample_unit` must be written in the verdict matrix.

### 2.10 Classifier Acceptance

Before alpha scoring:

- QC approves label thresholds and confirms they were fixed before candidate
  scoring.
- MIT verifies point-in-time feature joins and `.shift(1)` usage.
- PA verifies the classifier does not couple to a specific candidate's PnL.
- PM records the classifier version in TODO or the relevant report.
- QC/MIT approve the verdict matrix schema and promotion hard gates above.

### 3.2 Official Semantics To Preserve

Verified against official Bybit docs on 2026-05-31:

- Kline endpoints return reverse-sorted candle arrays and support `limit` up to
  1000 for futures-style klines.
- Mark/index/premium price kline endpoints return price-only candles
  `[start, open, high, low, close]`; they must not reuse standard OHLCV
  `KlineBar` parsers that expect volume and turnover.
- Funding history uses `startTime`/`endTime`, `limit <= 200`, and passing only
  `startTime` is an error. It has no cursor; historical paging must use
  `endTime` / oldest timestamp non-advance guards.
- Open interest uses `intervalTime`, `limit <= 200`, and cursor pagination.
- Long/short ratio uses `/v5/market/account-ratio`, `period`, `limit <= 500`,
  and cursor pagination.
- Instruments-info default page size is not enough for linear; cursor pagination
  or `limit=1000` is mandatory.
- Tickers expose current `indexPrice`, `markPrice`, `fundingIntervalHour`, and
  `fundingCap`, but this is a latest snapshot, not history.
- REST orderbook is snapshot format and does not include RPI orders.
- Instruments-info returns only `Trading` linear/inverse symbols by default;
  PIT listing/delisting checks must query status intentionally and/or use
  `market.symbol_universe_snapshots`.

### 3.3 Rate And Failure Discipline

Minimum policy:

- Use a dedicated public-data client isolated from live order/auth clients.
- Default backfill throttle should be gentle, e.g. 2-5 requests/sec, regardless
  of higher exchange limits.
- Read `X-Bapi-Limit*` headers when the client surface exposes them.
- Retry only documented transient/rate cases with bounded exponential backoff.
- Any final nonzero `retCode`, timeout, parse failure, or cursor non-advance
  must become a coverage failure, not fabricated data.
- Pagination must have max-page guards and non-advance cursor guards.

### 3.4 Client Gaps Before E1

Before implementation, E1/BB must choose and verify one client path:

1. Extend the isolated Python replay public client to include the AEG endpoint
   allowlist, strict parsers, per-endpoint rate policy, and page guards, or
2. Build a Rust public-only facade/allowlist. The current Rust
   `MarketDataClient` is not automatically acceptable because it wraps the
   authenticated REST client surface; it can be used only after BB verifies it
   cannot touch order/auth/private endpoints.

Known local gaps to address or explicitly accept:

- Python replay client currently allowlists only kline/tickers/orderbook.
- Python replay client uses `_KLINE_LIMIT=200`; official kline limit allows up
  to 1000, but lower page size is acceptable only if runtime estimates account
  for it.
- Python replay client default rates are higher than the AEG backfill throttle;
  AEG runners must force 2-5 requests/sec unless BB approves otherwise.
- No verified local client implements mark/index/premium price kline. These
  endpoints require a price-kline parser/schema with no volume/turnover fields.
- Rust OI and long-short methods lack historical start/end/cursor pagination
  today; they cannot be used for 18mo backfill until fixed or bypassed.
- Rust parsers that default missing/failed numeric fields to `0.0` are
  unacceptable for evidence runs. AEG parsers must be strict and turn parse
  failure into coverage failure.
- Rust instruments parser currently does not persist funding caps into
  `SymbolSpec`; it also omits status/launch/delivery fields needed for PIT
  universe checks. If a strategy needs these fields, parse the official
  status/launchTime/deliveryTime/fundingInterval/upperFundingRate/lowerFundingRate
  fields explicitly and update the Bybit reference in the same commit.
- `market_tickers` mark/index columns are not trusted as historical source.
