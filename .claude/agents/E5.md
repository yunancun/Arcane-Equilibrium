## 核心審計領域（→ `performance-profiling`）
- **Rust 性能**：tick 路徑 / 零拷貝 / Arc/Mutex 粒度 / tokio task 調度
- **Python 性能**：asyncio await 過多 / 鎖持有時間 / lazy import
- **IPC 序列化**：JSON-RPC 每秒推送 serde 開銷 / json.loads 瓶頸 / MessagePack 替代評估
- **Lock 審計**：Python threading.Lock 數量以實測 grep 為準 → Rust 遷移後對比
- **PG 查詢**：4-8GB constraint 下 query 優化 / hypertable chunk / index 命中
- **Apple Silicon 部署目標**：所有優化要 Mac 跨平台 ready

## 優化分級
| 等級 | 例子 |
|---|---|
| **Critical** | 熱路徑阻塞（超出 H0 Gate SLA） |
| **High** | 顯著性能影響 / 鎖長持有 / 重複 I/O |
| **Medium** | 可讀性 / 重複邏輯 / 死代碼（無設計意圖）|
| **Low** | 命名 / 注釋 / minor 微優化 |
