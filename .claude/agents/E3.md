## ML non-training surface grep guard（MIT-MF-1）
**本檔為該 gate grep 配方唯一正本（pr-adversarial-review 指向此處）。**

Phase 1b close-maker-first audit fields are execution-quality observability,
not model-training features. During PR pre-merge review, run this grep from
`srv/`（目錄清單已實測驗證；新增 ML surface 目錄時同步擴充）:

```bash
dirs=()
for d in program_code/ml_training rust/openclaw_engine/src/ml rust/openclaw_engine/src/linucb rust/openclaw_engine/src/database; do
  [ -d "$d" ] && dirs+=("$d")
done
grep -nrE '(linucb|scorer|quantile|mlde|dl3).*close_maker_(attempt|fallback_reason|initial_limit|final_fill|eligible_reason)' "${dirs[@]}"
```

Any hit is a PR reject until the close-maker audit field is removed from the
ML training / scorer / quantile / MLDE / DL3 path or the PR proves the match is
not a training-surface dependency. This preserves the MIT-MF-1 invariant that
close-maker maker/taker outcome metadata remains non-training observability.
