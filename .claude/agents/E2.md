## §5 Multi-session race check（P0-GOV-MULTI-SESSION-RACE-SOP-1 Phase 2 enforce 2026-05-16）

E2 PR review 跑 5 條 race check（對應 SOP 8 條精簡）：

- [ ] **5a 提交前 fetch + sibling window check**：`git fetch --prune origin` + `git log --since="2h ago" origin/main` — 看時間窗內有無 sibling push 衝突；git 狀態異常（origin/main 領先 HEAD 等）：報告開頭標註，以現有 HEAD 繼續審查；不做任何 git 寫操作
- [ ] **5b sub-agent IMPL DONE 前 status clean**：`git status --porcelain` 確認 unstaged 全屬本任務 scope；任何外洩檔（隔壁 IMPL leftover / stash leak）→ RETURN E1
- [ ] **5c 看到 unknown WIP 禁 revert**：`git status -s` / `git stash list` 出現非本 review session 改動 → 一律先 `git log --oneline -20` + `git reflog` + `git stash show -p stash@{N}` 而非 `git checkout -- <path>` / `git stash drop` / `git clean -fd`
- [ ] **5d Sign-off report commit 前 path clean**：sign-off report commit 對應檔 `git status --porcelain <path>` 必 empty 或僅本 commit 對應檔
- [ ] **5e PR review 期間 sibling 推 origin → 重 fetch 重 review**：review 進行中 sibling push 進 origin/main → 不忽略；重 `git fetch` + 評估 sibling commit 是否衝突本 PR 的 file scope（必跑 `git diff <pr-base>..<pr-head> --stat` vs `git log origin/main -1 --stat` 看 overlap）

**任一 ❌ → RETURN E1（不 merge）**。
