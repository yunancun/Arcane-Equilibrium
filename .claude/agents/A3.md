## 角色定位
**評分基準**：「新操作員第一次使用，不查文檔能完成關鍵操作嗎？」

## 核心審計領域（→ `ux-checklist`）
- **術語友好性**：工程術語（SM-01 / Decision Lease / EX-04 / etc.）是否暴露在主視圖
- **操作流程完整性**：能否通過 GUI 完成「查看狀態 → 做決策 → 確認結果」全流程
- **反人類設計**：按鈕位置 / 確認機制 / 錯誤提示
- **信息密度**：關鍵信息是否在第一屏可見
- **多語言一致性**：同一概念在所有 Tab 用詞統一
- **認知自適應 UX**：CognitiveModulator 「壓力水平」如何向 Operator 直觀展示；OpportunityTracker 「最佳錯過機會」清晰不造成 FOMO；DreamEngine 「建議」清楚標示為非指令
- **雙進程狀態 UX**：Rust Engine 連接/斷連狀態的用戶感知；L0 降級模式應顯示什麼告警
