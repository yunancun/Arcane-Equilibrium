用戶要求：所有面向 operator 的對話文字輸出以**中文**為主。英文只保留在：
- 技術名詞 / API / 檔名 / 函數名（原文保留更清楚）
- 代碼片段、commit message、PR title/body
- 外部系統互動（CLI 參數、SQL 關鍵字）

**Why:** 用戶 2026-04-15 明確反饋「你最近英文輸出挺多的，雖然我看得懂，但是還是希望你最好用中文」。用戶是中文母語者，長期對話體驗偏好中文；英文段落會增加閱讀摩擦，即使能看懂也不是最優。

**How to apply:**
- 回復用戶的狀態更新、決策建議、報告、問題澄清一律中文
- Tool call 之間的 text update 中文
- 表格欄位說明用中文；表格內的代碼/檔名/數字不翻譯
- Commit message 仍依倉庫風格（多半英文 + 中文副標題）
- 當用戶用英文提問或粘貼英文規格時，回覆可以「中文論述 + 引用英文原詞」混合，不必強行翻譯技術名詞
- 文檔（docs/worklogs/, docs/references/）依既有風格雙語或中文，不為此規則改動
