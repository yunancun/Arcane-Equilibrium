主會話 = PM + Conductor 合一。不再為 PM 角色單獨啟動 sub-agent。

**Why:** 主會話天然持有完整對話歷史 + 記憶系統，是最適合做 PM 決策的角色。單獨起 PM agent 需要重新餵大量上下文，效率低且產生角色重疊。用戶於 2026-04-03 確認方案 B。

**How to apply:**
- **我自己做的**（有上下文優勢）：PM 規劃、優先級判斷、任務拆分、PA 技術方案、結果綜合、進度追蹤
- **派 sub-agent 做的**（無狀態可完成）：
  - FA：規格審計（制衡我的規劃判斷）
  - E1 × N：並行開發（寫代碼，最多 5 個）
  - E2：代碼審查（強制，不可跳過）
  - E4：測試回歸（強制，不可跳過）
  - E5：優化審查（大板塊後強制）
  - E3/CC/A3/QC 等：按需啟動
- **工作流**：
  1. 我讀 TODO + 路線圖 → 判斷優先級 → 拆任務
  2. FA agent 驗證計劃（制衡）
  3. E1 agents 並行開發
  4. E2 agent 審查 → E4 agent 測試 → 我確認完成
- **原則**：有對話上下文優勢的角色我做，無狀態能完成的角色派 agent
