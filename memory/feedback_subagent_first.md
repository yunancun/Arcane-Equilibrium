name: 收到指令時強制先評估 sub-agent 拆分
description: 每次收到用戶 prompt 時，必須先思考是否應拆分為多個 sub-agent 後台並行處理，而非主會話串行執行
type: feedback
---

收到用戶的任何任務指令時，**強制先思考**：這個任務能否拆成多個獨立的 sub-agent 後台並行處理？

**Why:** 用戶明確要求最大化並行效率。主會話應作為 PM+Conductor 角色，負責拆分、派發、整合，而非自己串行執行所有工作。串行處理浪費時間，尤其是研究型任務（PM/PA/FA 分析）和獨立的 E1 執行任務。

**How to apply:**
1. 收到 prompt 後，第一步是分析任務可分解性：哪些部分互相獨立？哪些有依賴？
2. 獨立部分 → 立即拆成 sub-agent 用 `run_in_background: true` 後台並行啟動
3. 有依賴的部分 → 等上游 agent 完成後再啟動下游
4. 主會話只做：拆分決策、進度追蹤、結果整合、用戶溝通
5. 即使只有 2 個獨立任務，也應該並行而非串行
6. 例外：極簡單的單步操作（讀一個文件、改一行代碼）不需要拆分
