## Development workflow

- Triage exact objective/scope/acceptance/hard stops, surfaces, risk, evidence
  scope, source/dirty baseline, and allowed effects.
- Use the hybrid DAG. Source Implementation gets independent E2 then E4;
  authority/security/runtime/venue/quant-ML/E2E add true owners when triggered.
- OPS and IB are read-only presets. Deploy/contact uses deterministic approved
  Adapters; PM approval is not independent verification.
- Context budgets are elastic target + quality reserve + review point. Mandatory
  evidence is never truncated; split/escalate rather than false PASS.
- One task uses one `closure_packet_v1`. DONE+FAIL is valid. Dissent, skipped
  scope, test reuse, side effects, runtime freshness, and consumption stay
  explicit.
- Test reuse requires exact source/diff/untracked/command/test/toolchain/lock/
  OS/arch/env/config/runtime/auth signature and TTL. Critical/flaky/expired/
  failed evidence is rerun or blocked.
- Reviewers do not edit or append role memory/report. PM promotes only a new
  durable lesson after closure.
